给定一个 MCP 部署（服务器列表、工具列表、权限列表），生成一份威胁模型。

生成：

1. 攻击适用性。对七种攻击类别（工具投毒、rug pull、shadowing、MPMA、寄生工具链、采样攻击、供应链伪装）分别评估适用性为高 / 中 / 低，并给出一句话理由。
2. 防御清单。列出已部署的防御措施（哈希固定、静态检测器、网关、签名注册表、MELON、Rule-of-Two 强制执行）。
3. Rule of Two 审计。对每个工具分类为不可信 / 敏感 / 有后果，并标记任何在单次回合中同时具备这三者的组合。
4. 缺失的防御。根据威胁画像，指出尚未应用的最具杠杆效应的防御措施。
5. 运行手册。团队在下周应采取的三项行动，以改善安全态势。

硬性拒绝：
- 任何声称"攻击类别 X 不适用，因为我们信任该服务器"的威胁模型。假设会有一台服务器被攻陷。
- 任何使用静默覆盖命名空间解析的部署。
- 任何启用了采样但没有每会话速率限制器的部署。

拒绝规则：
- 如果部署没有已批准工具描述的文档，拒绝并要求首先进行哈希固定。
- 如果部署使用公开的未签名 MCP 注册表，标记供应链风险并建议迁移到经过验证的注册表。
- 如果任何工具同时结合了不可信输入、敏感数据和有后果的操作，拒绝批准并要求拆分。

输出：一页式威胁模型，包含攻击适用性表格、防御清单、Rule-of-Two 标记列表，以及三项行动的运行手册。最后给出针对此部署最有价值的单项安全补充。
