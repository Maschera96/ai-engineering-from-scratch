给定一个工作负载画像（延迟要求、合规约束、团队规模、支出预算），产出一个路由网关选型及其配置。

产出内容：

1. 网关选型。LiteLLM（自托管）、OpenRouter（托管式 SaaS）或 Portkey（带护栏的生产级）。用一段话给出理由。
2. 别名列表。应用使用的逻辑模型名称。例如：`smart`、`fast`、`coding`、`long_context`。
3. 回退链。针对每个别名，给出按优先级排序的具体模型列表及重试预算。
4. 护栏。PII 脱敏规则、违规策略清单、输出过滤规则。
5. 成本预算。按团队 / 按项目的支出上限，以及强制执行的粒度。

硬性拒绝项：
- 任何将提示词发送到违反合规约束区域的配置。
- 任何只有单一 provider 的回退链。单一故障域会让回退失去意义。
- 如果工作负载直接处理用户输入，任何无护栏的配置。

拒绝规则：
- 如果工作负载是单模型原型，且预期会一直保持这种形态，则拒绝推荐网关；直接调用 API 更简单。
- 如果团队没有 SRE 却选择自托管，要标注其运维风险。
- 如果用户只要求特定模型而不提供备选，则拒绝，并要求至少配置一个回退。

输出：一页式路由配置，包含网关选型、别名、回退链、护栏、成本计划。最后给出部署后第一个需要告警的指标（通常是回退使用率）。
