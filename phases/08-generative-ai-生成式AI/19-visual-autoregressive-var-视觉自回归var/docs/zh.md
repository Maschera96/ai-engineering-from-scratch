# 视觉自回归建模 (VAR)：下一代预测

> Diffusion 模型及时迭代采样（去噪步骤）。 VAR 按比例迭代采样 - 它预测 1x1 标记，然后是 2x2，然后是 4x4，直到最终分辨率，每个比例都以前一个比例为条件。 2024 年的论文表明，VAR 与图像生成的 GPT 式缩放法则相匹配，并在相同的计算预算下击败了 DiT。本课构建核心机制。

**类型：** 建造
**语言：** Python（搭配PyTorch）
**先修：** 第 7 阶段第 03 课（多头注意力），第 8 阶段第 06 课 (DDPM)
**时间：** 〜90分钟

## 问题

自回归生成主导了语言建模，因为它的扩展可预测：更多的计算、更多的参数、更低的困惑、更好的输出。 2024 年之前，图像生成有两次主要的 AR 尝试：PixelRNN/PixelCNN（逐像素）和 DALL-E 1 / Parti / MuseGAN（VQ-VAE 代码上的逐个词元）。

两者都面临着代序问题。像素和词元排列在 2D 网格中，但 AR 模型必须以 1D 光栅顺序访问它们。早期的角像素不知道图像最终会变成什么。生成质量比文本上的 GPT 更差，并且在匹配计算时从未达到扩散模型质量。

VAR 通过改变正在生成的内容来解决生成顺序问题。 VAR 不是在空间中逐个预测图像标记，而是以不断增加的分辨率预测整个图像。第 1 步：预测 1x1 标记（整体图像“摘要”）。第 2 步：预测 2x2 的 token 网格（更粗略的特征）。第 3 步：预测 4x4 网格。步骤K：预测最终的(H/8)x(W/8)网格。

每个尺度都涉及所有先前的尺度（因果关系按“尺度顺序”）并在其自己的尺度内平行。顺序问题消失了：尺度 k 的整个图像在一次变压器传递中生成。

## 概念

### VQ-VAE 多尺度分词器

VAR 需要一个**多尺度离散分词器**。对于图像 x，它会生成一系列分辨率逐渐提高的标记网格：

```
x -> encoder -> latent f
f -> tokenize at 1x1: token grid z_1 of shape (1, 1)
f -> tokenize at 2x2: token grid z_2 of shape (2, 2)
...
f -> tokenize at (H/p)x(W/p): token grid z_K of shape (H/p, W/p)
```

每个z_k使用相同的码本（典型大小4096-16384）。每个尺度的标记化不是独立的，它经过训练，以便对每个尺度的残差求和来重建 f：

```
f ≈ upsample(embed(z_1), target_size) + ... + upsample(embed(z_K), target_size)
```

这是**剩余 VQ** 变体。尺度 k 捕获了尺度 1..k-1 错过的内容。解码器获取所有尺度嵌入的总和并生成图像。

多尺度 VQ 分词器训练一次（如 VQGAN），然后冻结。所有生成工作都是由顶部的自回归模型完成的。

### 下一代预测

生成模型是一个转换器，它可以查看所有先前尺度的标记并预测下一个尺度的标记。

输入序列结构：
```
[START, z_1 tokens, z_2 tokens, z_3 tokens, ..., z_K tokens]
```

位置嵌入对尺度索引和尺度内的空间位置进行编码。注意力在尺度顺序上是因果关系的：尺度 k 处的标记，位置 (i, j) 可以关注尺度 1..k 处的所有标记，以及尺度 k 本身的标记，无论使用何种尺度内顺序，这些标记本身都较早出现（VAR 使用固定位置注意力，没有尺度内因果关系，尺度内的所有位置都是并行预测的）。

训练损失：在每个尺度 k 上，在给定所有先前尺度标记的情况下预测标记 z_k。离散 VQ 码的交叉熵损失。与 GPT 的结构相同，只是“序列”现在是按比例结构的。

### 一代

推理：
```
generate z_1 = sample from p(z_1)                    # 1 token
generate z_2 = sample from p(z_2 | z_1)              # 4 tokens in parallel
generate z_3 = sample from p(z_3 | z_1, z_2)         # 16 tokens in parallel
...
decode: f = sum of embed-and-upsample scales 1..K
image = VAE_decoder(f)
```

对于 K = 10 尺度，生成是 10 次变压器前向传递。每次传递都会并行生成其整个范围 - 范围内没有每个标记的自回归。对于 256x256 图像，这大约是 10 次通过，而 DiT 是 28-50 次。

### 为什么 Next-Scale 胜过 Next-Token

三个结构性胜利：
1. **从粗到细与自然图像统计数据一致。** 人类视觉感知和图像数据集都表现出与尺度相关的规律性：低频结构稳定且可预测；高频细节取决于低频内容。下一个规模的预测利用了这一点。
2. **规模内并行生成。** 与 GPT 式词元 AR 不同，VAR 一步生成某一规模的所有词元。有效生成长度是对数尺度而不是线性的。
3. **没有生成顺序偏差。** 规模 k 的词元看到规模 k-1 的所有词元；不存在强制早期词元在后期上下文可用之前提交的“左侧”或“上方”偏差。

### 缩放定律

田等人。证明 VAR 在 ImageNet 上遵循 FID 的幂律缩放曲线，就像 GPT 对于困惑度所做的那样。加倍参数或可靠地计算可将误差减半。这是第一个像语言模型一样清晰地展示这种缩放行为的图像生成模型。结果是 VAR 规模的预测变得可以通过计算来预测，而不是根据每个架构的经验猜测。

### 与 Diffusion 的关系

VAR 和扩散共享相同的数据压缩故事：都将生成问题分解为一系列更简单的子问题。

- Diffusion：逐渐添加噪音，学会撤消一步。
- VAR：逐渐添加分辨率，学会预测下一个尺度。

它们是解决问题的不同轴心。两者都产生易于处理的条件分布。根据经验，VAR 的推理速度更快（通过次数更少，在一定范围内全部并行），并且在类条件 ImageNet 上匹配或击败 DiT。文本条件VAR（VARclip、HART）是一个活跃的研究方向。

## 构建它

在`code/main.py`中你将：
1. 在合成“图像”数据（2D 高斯环）上构建一个微型**多尺度 VQ 分词器**。
2. 训练**VAR 式转换器**来预测词元的下一个尺度。
3. 通过调用 Transformer 4 次（4 个尺度）进行采样并解码。
4. 验证按比例排序的训练是否使生成在比例内并行。

这是一个玩具实现。关键是要看到尺度结构的注意力掩模和尺度内并行生成的实际工作情况。

## 交付它

本课程产生 `outputs/skill-var-tokenizer-designer.md`，一种设计多尺度分词器的技能：尺度数量、尺度比率、码本大小、残差共享、解码器架构。

## 练习

1. **规模计数消融。** 使用 4、6、8、10 个规模训练 VAR。测量重建质量与自回归遍数的关系。更多尺度 = 更精细的残差 = 更好的质量但更多的通过。

2. **码本大小。** 使用码本大小 512、4096、16384 训练分词器。较大的码本可提供更好的重建，但预测更困难。找到膝盖。

3. **规模内并行检查。** 对于经过训练的 VAR，明确测量注意力模式。在尺度 k 内，模型是​​否关注跨尺度位置而不关注尺度内位置？验证掩码实施。

4. **VAR 与 DiT 缩放。** 对于相同的 ImageNet 类条件任务，以匹配的参数预算（例如 33M、130M、458M）训练 VAR 和 DiT。绘制 FID 与计算图。 VAR 在每个尺寸上都应该领先于 DiT，以小尺寸再现论文的结果。

5. **文本调节。** 扩展 VAR 以将文本嵌入（CLIP 池化）作为通过 adaLN 的额外调节输入。这是 HART 配方。 FID 对文本对齐采样有何改进？

## 关键术语

| 学期 | 人们怎么说 | 它实际上意味着什么 |
|------|----------------|----------------------|
| VAR | “视觉自回归” | 通过 VQ 词元网格金字塔的下一尺度预测生成图像 |
| 下一个规模的预测 | “先粗略预测，再精细预测” | 该模型以增加的分辨率尺度预测标记，并以所有先前的尺度为条件 |
| 多尺度 VQ 分词器 | “剩余VQ” | VQ-VAE 产生 K 个分辨率不断增加的词元网格，解码器对所有尺度进行求和 |
| 尺度 k | “金字塔层k” | K 个分辨率级别之一，从 k=1 时的 1x1 到 k=K 时的 (H/p)x(W/p) |
| 规模内并行 | “每级一名前锋” | 尺度 k 上的所有标记都是在一次 Transformer Pass 中预测的，而不是自回归的 |
| 跨尺度因果关系 | “按比例排序的注意力” | 尺度 k 的词元可以关注所有尺度 1..k，但不能关注尺度 k+1..K |
| 残余矢量量化 | “附加标记化” | 每个尺度的标记对较低尺度留下的残差进行编码；解码器对所有尺度嵌入进行求和 |
| VAR标定法则 | “图像GPT缩放” | FID 在计算中遵循可预测的幂律，就像语言模型的复杂性一样 |
| 哈特 | “混合VAR+文本” | 文本条件 VAR 变体将 MaskGIT 风格的迭代解码与 VAR 的尺度结构相结合 |
| 尺度位置嵌入 | “（比例、行、列）三重” | 位置编码携带尺度索引和尺度内的空间坐标 |
