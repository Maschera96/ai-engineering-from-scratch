# GANs — 生成器与鉴别器

> Goodfellow 在 2014 年的技巧是完全跳过密度。两个网络。一个人做假货。一个人抓住他们。他们一直战斗到假货与真货无法区分为止。它不应该工作。但通常情况并非如此。当它发生时，这些样本仍然是窄域文献中最清晰的。

**类型：** 建造
**语言：** Python
**先修：** 阶段 3·02（反向传播）、阶段 3·08（优化器）、阶段 8·02 (VAE)
**时间：** 〜75分钟

## 问题

VAE 会产生模糊样本，因为它们的 MSE 解码器损失对于“平均”图像来说是贝叶斯最优的，并且许多可信数字的平均值是模糊数字。您想要一种奖励“合理性”的损失，而不是与任何一个目标的像素级接近度。不存在合理性的封闭形式。你必须学习它。

Goodfellow 的想法是：训练一个分类器 `D(x)` 来区分真实图像和赝品。训练发电机 `G(z)` 来愚弄 `D`。 `G` 的损失信号是 `D` 当前认为使某些东西看起来真实的任何信号。该信号随着 `G` 的改进而更新，追逐移动目标。如果两个网络收敛，`G` 就已经了解了数据分布，而无需写下 `log p(x)`。

这就是对抗性训练。数学是一个极小极大游戏：

```
min_G max_D  E_real[log D(x)] + E_fake[log(1 - D(G(z)))]
```

2026 年，GAN 不再是 SOTA 发生器（扩散和流量匹配吞噬了这一王冠）。但 StyleGAN 2/3 仍然是有史以来最清晰的面部模型，GAN 鉴别器在扩散训练中用作“感知损失”，而对抗性训练则支持快速的 1 步蒸馏（SDXL-Turbo、SD3-Turbo、LCM），让您可以进行实时扩散。

## 概念

![GAN 训练：极小极大中的生成器和判别器](../assets/gan.svg)

**生成器 `G(z)`。** 将噪声向量 `z ~ N(0, I)` 映射到样本 `x̂`。解码器形状的网络（密集或转置卷积）。

**判别器 `D(x)`。** 将样本映射到标量概率（或分数）。真→1，假→0。

**损失。** 两次交替更新：

- **火车 `D`：** `loss_D = -[ log D(x) + log(1 - D(G(z))) ]`。 real=1，fake=0 上的二元交叉熵。
- **列车 `G`：** `loss_G = -log D(G(z))`。这是 Goodfellow 使用的“非饱和”形式（当 `D` 有信心时，原始 `log(1 - D(G(z)))` 会饱和并杀死梯度）。

**训练循环。** `D` 一步，`G` 一步。重复。

**为什么有效。** 如果 `G` 完美匹配 `p_data`，那么 `D` 就不能比机会做得更好，并且到处输出 0.5； `G` 不再获得梯度。平衡。

**为什么它会崩溃。** 模式崩溃（`G` 发现一种模式 `D` 无法分类并永远铸造它），梯度消失（`D` 学得太快，`log D` 饱和），训练不稳定（学习率、批量大小等）。

## 使 GAN 发挥作用的变体

| 年 | 创新 | 使固定 |

## 构建它

`code/main.py` 在一维数据上训练一个微小的 GAN：两个高斯的混合。生成器和鉴别器是单隐藏层 MLP。我们手动实现前向、后向和极小极大循环。目标是看到两种关键失效模式（模式崩溃+梯度消失）的发生。

### 步骤1：非饱和损失

当 D 以高置信度将 G 的赝品分类为赝品时，普通 Goodfellow 损失 `log(1 - D(G(z)))` 变为 0。此时 G 的梯度基本上为零，G 无法改进。非饱和形式 `-log D(G(z))` 具有相反的渐近线：当 D 有信心时它会爆炸，给 G 一个强烈的信号。

```python
def g_loss(d_fake):
    # maximize log D(G(z))  <=>  minimize -log D(G(z))
    return -sum(math.log(max(p, 1e-8)) for p in d_fake) / len(d_fake)
```

### 第 2 步：每个生成器步骤一个判别器步骤

```python
for step in range(steps):
    # train D
    real_batch = sample_real(batch_size)
    fake_batch = [G(z) for z in sample_noise(batch_size)]
    update_D(real_batch, fake_batch)

    # train G
    fake_batch = [G(z) for z in sample_noise(batch_size)]  # fresh fakes
    update_G(fake_batch)
```

G 是新鲜的假货，否则梯度是陈旧的。

### 第 3 步：观察模式崩溃

```python
if step % 200 == 0:
    samples = [G(z) for z in sample_noise(500)]
    mode_a = sum(1 for s in samples if s < 0)
    mode_b = 500 - mode_a
    if min(mode_a, mode_b) < 50:
        print("  [!] mode collapse: one mode is starved")
```

典型症状：两个实模之一停止生成。鉴别器停止纠正它，因为它从未被视为假货。

## 常见陷阱

- **判别器太强了。** 将 D 的学习率降低 2-5 倍，或者添加 instance/layer 噪声。如果D达到>95%的准确度，G就死了。
- **生成器记住一种模式。**向 D 输入添加噪声，使用小批量鉴别器层，或切换到 WGAN-GP。
- **批次范数泄漏统计数据。** 流经同一 BN 层的真实批次 + 假批次混合了它们的统计数据。请改用实例范数或谱范数。
- **初始得分游戏。** FID 和 IS 在低样本数下有噪音。评估时使用 ≥10k 样本。
- **一次性采样对于条件任务来说是一个谎言。** 您仍然需要 CFG 尺度、截断技巧和重新采样才能获得可用的输出。

## 使用它

2026 GAN 堆栈：

| 情况 | 挑选 |
|-----------|------|
| 照片真实的人脸，固定姿势 | StyleGAN3（最锐、最小） |
| 动漫/风格化面孔 | StyleGAN-XL或Stable Diffusion LoRA |
| 图像到图像的翻译 | Pix2Pix / CycleGAN（第 8 期·04）或 ControlNet（第 8·08 期） |
| 快速 1 步文本到图像 | 扩散的对抗性蒸馏（SDXL-Turbo、SD3-Turbo） |
| 扩散训练器内的知觉损失 | 图像裁剪上的小型 GAN 鉴别器 |
| 任何多模式、开放式的事物 | 不要使用扩散或流量匹配 |

GAN 很锋利但很窄。一旦您的域打开（照片、任意文本提示、视频），请切换到扩散。对抗性技巧作为一个组件（感知损失、蒸馏）存在，而不是一个独立的生成器。

## 关键术语

| 学期 | 人们怎么说 | 它实际上意味着什么 |
|------|-----------------|-----------------------|
| 发电机 | “G” | 噪声样本网络，`G: z → x̂`。 |
| 鉴别器 | “D” | 分类器 `D: x → [0, 1]`，真品与假品。 |
| 极小极大 | “游戏” | `min_G max_D` 为联合目标。 |
| 非饱和损耗 | “修复” | G 使用 `-log D(G(z))` 而不是 `log(1 - D(G(z)))`。 |
| 模式崩溃 | “G记住了一件事” | 尽管数据多种多样，但生成器产生的不同输出很少。 |
| WGAN | “瓦瑟斯坦” | 将 BCE 替换为 Earth-Mover 距离 + 梯度惩罚；更平滑的渐变。 |
| 谱范数 | “利普希茨把戏” | 限制 D 的权重范数以限制其斜率；稳定训练。 |
| StyleGAN | “有效的那个” | 测绘网络+AdaIN；仍然是 2026 年，面部方面最好的。 |

## 制作笔记：一次性推理是GAN的持久优势

GAN 不再在开放域生成的样本质量上获胜，但它们仍然在推理成本上获胜。在产生推理文献词汇中，GAN 具有：

- **无预填充，无解码阶段。** 单个 `G(z)` 前向传递。 TTFT ≈ 总延迟。
- **没有 KV 缓存压力。** 唯一的状态是权重。批量大小受激活内存限制，而不是缓存。
- **简单的连续批处理。** 由于每个请求都采用相同的固定 FLOP，因此服务器目标占用率的静态批处理通常是最佳的。无需飞行调度程序。

这就是为什么 GAN 蒸馏（SDXL-Turbo、SD3-Turbo、ADD、LCM）是 2026 年快速文本到图像的主导技术：它将 20-50 步扩散管道折叠为 1-4 GAN 式前向传递，同时保持扩散基础的分布。对抗性损失作为训练时间旋钮而存在，用于将慢速生成器变成快速生成器。
