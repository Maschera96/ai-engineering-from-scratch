# Diffusion 模型 — DDPM 从头开始

> Ho, Jain, Abbeel (2020) 给了这个领域一个无法放弃的秘诀。用超过一千个小步骤的噪音来破坏数据。训练一个神经网络来预测噪声。在推理时反转该过程。如今，每个主流图像、视频、3D 和音乐模型都在这个循环上运行，可能还带有流匹配或一致性技巧。

**类型：** 建造
**语言：** Python
**先修：** 阶段 3·02（反向传播），阶段 8·02 (VAE)
**时间：** 〜75分钟

## 问题

您想要 `p_data(x)` 的采样器。 GAN 玩的是经常发散的极小极大游戏。 VAE 从高斯解码器产生模糊样本。你真正想要的是一个训练目标，它是 (a) 单一稳定损失（没有鞍点，没有极小极大），(b) `log p(x)` 的下限（这样你就有可能性），以及 (c) 与 SOTA 质量匹配的样本。

索尔-迪克斯坦等人。 （2015）有一个理论答案：定义一个逐渐添加高斯噪声的马尔可夫链 `q(x_t | x_{t-1})`，并训练一个反向链 `p_θ(x_{t-1} | x_t)` 去噪。 Ho、Jain、Abbeel（2020）表明，损失可以简化为一行，预测噪声，并清理了数学。在 2020 年，这是一个好奇心。 2021 年，它生产出了最先进的样品。 2022年变成了Stable Diffusion。 2026年它是基板。

## 概念

![DDPM：前向噪声、反向噪声](../assets/ddpm.svg)

**正向过程`q`。** 以`T`小步长添加高斯噪声。封闭形式，数学易于处理的原因，是累积步骤也是高斯的：

```
q(x_t | x_0) = N( sqrt(α̅_t) · x_0,  (1 - α̅_t) · I )
```

其中 `α̅_t = ∏_{s=1..t} (1 - β_s)` 表示 `β_t` 的时间表。在 T=1000 步内从 1e-4 到 0.02 线性选取 `β_t`，`x_T` 大约为 `N(0, I)`。

**反向处理 `p_θ`。** 学习神经网络 `ε_θ(x_t, t)` 来预测添加的噪声。给定 `x_t`，通过以下方式降噪：

```
x_{t-1} = (1 / sqrt(α_t)) · ( x_t - (β_t / sqrt(1 - α̅_t)) · ε_θ(x_t, t) )  +  σ_t · z
```

其中 `σ_t` 是 `sqrt(β_t)` 或学习方差。这个表达式很丑陋，但它只是代数，在给定后验 `q(x_{t-1} | x_t, x_0)` 的情况下求解 `x_{t-1}`，并用其噪声预测估计值替换 `x_0`。

**训练损失。**

```
L_simple = E_{x_0, t, ε} [ || ε - ε_θ( sqrt(α̅_t) · x_0 + sqrt(1 - α̅_t) · ε,  t ) ||² ]
```

从数据中采样 `x_0`，随机选择一个 `t`，采样 `ε ~ N(0, I)`，通过封闭形式一次性计算出噪声 `x_t`，并对噪声进行回归。一次损失，没有极小极大，没有 KL，没有重新参数化技巧。

**采样。**启动`x_T ~ N(0, I)`。迭代从 `t = T` 到 `1` 的相反步骤。完毕。

## 为什么它有效

三个直觉：

1. **去噪简单；生成很难。**在 `t=T`，数据是纯粹的噪声，网络必须解决一个微不足道的问题。在 `t=0` 中，网络只需清理几个像素。在中间 `t` 处，问题很困难，但网络有许多梯度从每个噪声级别流过相同的权重。

2. **变相的分数匹配。** Vincent (2011) 证明预测噪声相当于估计 `∇_x log q(x_t | x_0)`，即*分数*。反向 SDE 使用该分数沿着密度梯度向上行走，引导随机向高概率区域行走。

3. **ELBO 简化为简单的 MSE。** 完整变分下界每个时间步都有一个 KL 项。通过 DDPM 的参数化，这些 KL 项在具有特定系数的噪声预测上简化为 MSE； Ho 降低了系数（称之为“简单”损失）并且质量*提高了*。

## 构建它

`code/main.py` 实现一维 DDPM。数据是两种模式的混合。 “网络”是一个微小的 MLP，它采用 `(x_t, t)` 并输出预测噪声。训练是单线损失。采样迭代反向链。

### 第 1 步：远期时间表（封闭式）

```python
betas = [1e-4 + (0.02 - 1e-4) * t / (T - 1) for t in range(T)]
alphas = [1 - b for b in betas]
alpha_bars = []
cum = 1.0
for a in alphas:
    cum *= a
    alpha_bars.append(cum)
```

### 第 2 步：一次性采样 `x_t`

```python
def forward_sample(x0, t, alpha_bars, rng):
    a_bar = alpha_bars[t]
    eps = rng.gauss(0, 1)
    x_t = math.sqrt(a_bar) * x0 + math.sqrt(1 - a_bar) * eps
    return x_t, eps
```

### 第 3 步：一个训练步骤

```python
def train_step(x0, model, alpha_bars, rng):
    t = rng.randrange(T)
    x_t, eps = forward_sample(x0, t, alpha_bars, rng)
    eps_hat = model_forward(model, x_t, t)
    loss = (eps - eps_hat) ** 2
    return loss, gradient_step(model, ...)
```

### 第四步：反向采样

```python
def sample(model, alpha_bars, T, rng):
    x = rng.gauss(0, 1)
    for t in range(T - 1, -1, -1):
        eps_hat = model_forward(model, x, t)
        beta_t = 1 - alphas[t]
        x = (x - beta_t / math.sqrt(1 - alpha_bars[t]) * eps_hat) / math.sqrt(alphas[t])
        if t > 0:
            x += math.sqrt(beta_t) * rng.gauss(0, 1)
    return x
```

对于具有 40 个时间步长和 24 单元 MLP 的一维问题，它在大约 200 个时期内学习两种模式混合。

## 时间调节

网络需要知道它在哪个时间步去噪。两个标准选项：

- **正弦嵌入。** 如 Transformer 位置编码。 `embed(t) = [sin(t/ω_0), cos(t/ω_0), sin(t/ω_1), ...]`。通过MLP，广播到网络中。
- **电影/组规范调节。**项目嵌入到每个块的每通道 scale/bias (FiLM)。

我们的玩具代码使用正弦 → concat。生产 U-Net 使用 FiLM。

## 常见陷阱

- **时间表非常重要。** 线性 `β` 是 DDPM 默认值，但余弦时间表（Nichol & Dhariwal，2021）为相同的计算提供了更好的 FID。如果质量停滞不前，请切换时间表。
- **时间步嵌入很脆弱。** 将原始 `t` 作为浮点数传递适用于玩具一维，但不适用于图像；始终使用正确的嵌入。
- **V 预测与 ε 预测。** 对于狭窄的范围（非常小或非常大的 t），`ε` 的信噪比很差。 V-预测（`v = α·ε - σ·x`）更稳定； SDXL、SD3、Flux 使用它。
- **无分类器指导。** 在推理时，计算条件和无条件 `ε`，然后使用 `w ≈ 3-7` 计算 `ε_cfg = (1 + w) · ε_cond - w · ε_uncond`。第 08 课中介绍过。
- **1000 个步骤很多。** 生产使用 DDIM（20-50 个步骤）、DPM-Solver（10-20 个步骤）或蒸馏（1-4 个步骤）。参见第 12 课。

## 关键术语

| 学期 | 人们怎么说 | 它实际上意味着什么 |
|------|-----------------|-----------------------|
| 转发过程 | “添加噪音” | 固定马尔可夫链`q(x_t \| x_{t-1})` 会破坏数据。 |
| 逆向过程 | “去噪” | 学习链`p_θ(x_{t-1} \| x_t)` 重建数据。 |
| β时间表 | 《噪音阶梯》 | 每步方差；线性、余弦或 S 形。 |
| α̅ | “阿尔法酒吧” | 累计积`∏(1 - β)`；从 `x_0` 给出闭合形式 `x_t`. |
| 简单损失 | “噪声的 MSE” | `\|\|ε - ε_θ(x_t, t)\|\|²`;所有变分推导都崩溃于此。 |
| ε-预测 | “预测噪音” | 输出是添加的噪声；标准DDPM。 |
| V-预测 | “预测速度” | 输出为`α·ε - σ·x`；更好的调节能力。 |
| DDPM | “纸” | 何等人。 2020；线性 β，1000 步，U-Net。 |
| DDIM | “确定性采样器” | 非马尔可夫采样器，20-50 个步骤，相同的训练目标。 |
| 无分类器指导 | “CFG” | 混合有条件和无条件噪声预测以放大调节。 |

## 制作笔记：扩散推理是一个步数问题

DDPM 论文运行 T=1000 个反向步骤。没有人在生产中运送它。每个真正的推理堆栈都会选择三种策略之一，并且每种策略都清晰地映射到“延迟来自哪里”的生产框架：

1. **更快的采样器，相同的型号。** DDIM（20-50 步）、DPM-Solver++ (10-20)、UniPC (8-16)。反向回路的直接替换；训练后的 `ε_θ` 权重保持不变。将延迟缩短 20-50 倍。
2. **蒸馏。** 用更少的步骤训练学生匹配老师：渐进蒸馏（2 → 1）、一致性模型（任意 → 1-4）、LCM、SDXL-Turbo、SD3-Turbo。将延迟时间再缩短 5-10 倍，需要重新训练。
3. **缓存和编译。** `torch.compile(unet, mode="reduce-overhead")`，TensorRT-LLM 的扩散后端，ZXQ3QXZ/SDPA 注意力，bf16 权重。将每步延迟缩短约 2 倍。与 (1) 和 (2) 堆叠。

对于生产扩散服务器，预算对话与 LLM 的生产文献中描述的相同：延迟为 `num_steps × step_cost + VAE_decode`，吞吐量为 `batch_size × (num_steps × step_cost)^-1`。 TTFT小（一步）； TPOT 等效时间是完整响应时间，因为从用户的角度来看，图像生成是“一次性”的。
