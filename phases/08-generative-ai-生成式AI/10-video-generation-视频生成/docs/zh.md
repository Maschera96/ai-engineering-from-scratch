# 视频生成

> 图像是一个二维张量。视频是 3D 视频。原理是一样的；计算难度提高了 10-100 倍。 OpenAI 的 Sora（2024 年 2 月）证明了这是可能的。到 2026 年，Veo 2、Kling 1.5、Runway Gen-3、Pika 2.0 和 WAN 2.2 将以 1080p 的速度发布文本制作视频 - 而开放权重堆栈（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）则落后 12 个月。

**类型：** 建造
**语言：** Python
**先修：** 阶段 8·07（潜在 Diffusion）、阶段 7·09（ViT）、阶段 8·06（DDPM）
**时间：** 约45分钟

## 问题

24fps 的 10 秒 1080p 视频是 240 帧 1920×1080×3 像素。每个剪辑大约有 1.5 GB 的原始数据。像素空间扩散是不可行的。您需要：

1. **时空压缩。** VAE 将视频（而不是帧）编码为一系列时空补丁。
2. **时间连贯性。** 框架需要在几秒钟内共享内容、照明和对象标识。网络必须对运动进行建模。
3. **计算预算。** 对于相同的模型大小，视频训练比图像训练贵 10-100 倍。
4. **调节。** 文本、图像（第一帧）、音频或其他视频。大多数生产型号都接受这四种。

解决这个问题的架构是应用于时空补丁的 **Diffusion Transformer (DiT)**，在巨大的（提示、标题、视频）数据集上进行训练。与第 06 课相同的扩散损失。

## 概念

![视频扩散：patchify、DiT、解码](../assets/video-generation.svg)

### 修补

使用 3D VAE（学习的时空压缩）对视频进行编码。潜在形状为`[T_latent, H_latent, W_latent, C_latent]`。分成大小为 `[t_p, h_p, w_p]` 的补丁。对于 Sora 样式模型，`t_p = 1`（每帧补丁）或 `t_p = 2`（每两帧）。 10 秒的 1080p 视频可压缩至约 20,000-100,000 个补丁。

### 时空DiT

变压器处理平坦的补丁序列。每个补丁都有一个 3D 位置嵌入（时间 + y + x）。注意力通常被分解：

- 每帧补丁内的**空间注意力**。
- 同一空间位置跨帧的**时间注意力**。
- **全 3D 注意力** 价格贵 16-100 倍；仅在低分辨率或研究中使用。

### 文本调节

与大型文本编码器的交叉注意力（Sora 为 T5-XXL，CogVideoX-5B 使用 T5-XXL）。长提示很重要，Sora 的训练集有 GPT 生成的密集字幕，平均每个剪辑有 200 个标记。

### 训练

时空潜伏的标准扩散损失（ε 或 v 预测）。数据：网络视频 + 约 1 亿精选剪辑 + 合成文本字幕。计算：即使是小型研究运行也需要 10,000 多个 GPU 小时； Sora-规模为100,000+。

## 构建它

`code/main.py` 模拟核心时空 DiT 思想：修补一个小型合成视频，添加每个补丁位置嵌入，并通过对补丁的 Transformer 式关注对整个序列进行降噪。没有麻木；纯Python。我们表明，当相邻帧块共享降噪器和位置嵌入时，即使在一维中也会出现时间一致性。

### 第 1 步：修补合成的一维“视频”

```python
def make_video(T_frames=8, rng=None):
    # a "video" is a sequence of 1-D values following a smooth trajectory
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 步骤2：每帧的位置嵌入

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 第 3 步：降噪器查看整个序列

我们的微型网络不是独立地对每个帧进行去噪，而是连接所有帧值+它们的位置嵌入，并联合预测所有帧的噪声。

### 步骤4：时间相干性测试

训练后，对视频进行采样。测量帧间增量。如果模型已经学习了时间结构，则增量保持小于独立采样每个帧。

## 常见陷阱

- **独立的每帧采样 = 闪烁。** 如果您分别在每个帧上运行图像扩散，则输出会闪烁，因为每个帧的噪声是独立的。视频扩散通过注意力或共享噪声耦合帧来解决这个问题。
- **朴素 3D 注意力 = OOM。** 10 秒 1080p 潜在的完整 3D 注意力需要数千亿次操作。分解为空间+时间。
- **数据字幕比大小更重要。** Sora 相对于之前的工作的主要升级是对大约 10 倍详细的字幕进行培训（GPT-4 重新标记的剪辑）。 OpenAI的技术报告对此有明确说明。
- **第一帧调节。** 大多数生产型号也接受图像作为第一帧。这是“图像转视频”模式；训练包括这个变体。
- **物理漂移。** 长剪辑（>10 秒）会积累微妙的不一致。滑动窗口生成+关键帧锚定有帮助。

## 制作说明：视频潜伏是一个内存带宽问题

24 fps 的 10 秒 1080p 剪辑为 240 帧 × 1920 × 1080 × 3 ≈ 1.5 GB 原始像素。经过 4 倍视频 VAE 压缩 (`2 × spatial × 2 × temporal`) 后，每个请求的潜在值约为 100 MB。通过时空 DiT 在批次 1 中运行 30 个步骤，您将通过 HBM 移动约 3 GB/step — 瓶颈是内存带宽，而不是 FLOP。

三个产生式旋钮，全部直接来自产生式推理文献推理章节：

- **跨 DiT 的 TP。** 文本到视频模型通常≥10B 参数。 4 个 H100 中 TP=4 是标准；对于 405B 级型号，PP=2 × TP=2。每步的延迟随着 TP 大致呈线性下降，直至达到 all-reduce 墙。
- **帧批处理 = 连续批处理。** 在生成时，视频在概念上是由注意力链接的一批帧。连续批处理（动态调度）适用：如果模型架构允许滑动窗口生成，则在返回帧 `t-1` 时开始渲染帧 `t+1`。
- **剪辑级预填充缓存。** 对于图像到视频，第一帧调节类似于 LLM 的提示预填充：计算一次，在时间解码器通道中重复使用。这实际上是视频的 KV 缓存。
