# 生成模型，分类法和历史

> 每个图像模型、文本模型、视频模型和 3D 模型都适合五个桶之一。如果选错了桶，你将与数学作斗争数周。选择正确的一个，该领域过去 12 年的进展就会清晰地浮现在您的脑海中。

**类型：** 学习
**语言：** Python
**先修：** 第 2 阶段（机器学习基础知识）、第 3 阶段（深度学习核心）、第 7·14 阶段 (Transformers)
**时间：** 约45分钟

## 问题

生成模型只完成一项工作：给定从某个未知分布 `p_data(x)` 中提取的训练样本，输出看起来来自同一分布的新样本。面孔、句子、MIDI 文件、蛋白质结构，如果你眯着眼睛看，这些都是同样的问题。

问题是 `p_data` 生活在一个具有数百万维的空间中（512x512 RGB 图像的维数约为 786k），样本位于该空间内的薄流形上，而您可能只有 10M 个示例。暴力破解密度是没有希望的。每个生成模型都是一种折衷方案，用一个困难问题换取一个稍微不那么困难的问题。

过去十二年里，有五个家庭幸存下来。了解每个家庭做出的妥协会告诉你为什么它在某些任务上获胜而在另一些任务上失败。

## 概念

![生成模型的五个系列，按模型分类](../assets/taxonomy.svg)

**1.明确的密度，易于处理。** 将 `log p(x)` 写为您可以实际评估的总和。自回归模型（PixelCNN、WaveNet、GPT）因式分解 `p(x) = ∏ p(x_i | x_<i)`。标准化流（RealNVP、Glow）将 `p(x)` 构建为简单基数的可逆变换。优点：准确的可能性，干净的训练损失。缺点：自回归推理是顺序的（长序列很慢），流程需要可逆架构（架构限制）。

**2.显式密度，近似值。** 从下方绑定 `log p(x)` (ELBO) 并优化绑定。 VAEs（Kingma 2013）使用具有变分后验的编码器-解码器。 Diffusion 模型（DDPM，Ho 2020）训练隐式优化加权 ELBO 的降噪器。 Diffusion 是 2026 年图像、视频和 3D 的主导骨干网。

**3.隐式密度。** 完全跳过密度；学习一个生成样本的生成器 `G(z)` 和一个辨别真假的鉴别器 `D(x)`。 GANs（古德费罗 2014 年）。推理速度很快（一次前向传递），但在训练过程中却不稳定。即使到 2026 年，StyleGAN 1/2/3 仍然是固定域照片写实（脸部、卧室）的最先进技术。

**4.基于分数/连续时间。** 直接学习对数密度 `∇_x log p(x)`（分数）的梯度。 Song & Ermon (2019) 表明，分数匹配将扩散推广到 SDE。流匹配（Lipman 2023）是 2024-2026 年的热点：无模拟训练、更直的路径、比 DDPM 快 4-10 倍的采样。 Stable Diffusion 3、Flux、AudioCraft 2都使用流匹配。

**5.基于词元的离散代码自回归。** 使用 VQ-VAE 或残差量化器将高亮度数据压缩为短的离散词元序列，然后使用 Transformer 对词元序列进行建模。 Parti、MuseNet、AudioLM、VALL-E、Sora 的 patch tokenizer 都使用这个。这是桶 1 加上学习的分词器。

## 五个问题分诊

当一篇新的生成模型论文发布时，在阅读方法部分之前回答这五个问题。

1. **正在建模什么？** 像素、潜伏、离散标记、3D 高斯、网格、波形？
2. **密度是显式的还是隐式的？** 他们写下`log p(x)`吗？
3. **采样：一次性还是迭代？** 迭代意味着推理速度较慢；一次性通常意味着对抗性或精炼。
4. **条件：无条件、类别、文本、图像、姿势？** 这决定了损失和架构脚手架。
5. **评估：FID、CLIP 分数、IS、人类偏好、任务准确性？** 每个都有已知的故障模式（请参阅第 14 课）。

您将在本阶段的每节课中重新回答这五个问题。到最后，他们就会条件反射。

## 构建它

本课程的代码是一个轻量级可视化：使用三种玩具方法（核密度、离散直方图和最近样本“GAN”生成器）拟合样本中的一维高斯混合，这样您就可以在一个屏幕上打印的问题上看到显式密度与隐式密度之间的差异。

运行`code/main.py`。它从二模高斯混合中提取 2000 个样本，然后打印：

```
explicit density (histogram): p(x in [-0.5, 0.5]) ≈ 0.38
approximate density (KDE):     p(x in [-0.5, 0.5]) ≈ 0.41
implicit (nearest-sample gen): 20 new samples printed, no p(x)
```

注意：前两个让你问“这一点的可能性有多大？”第三个不能。这是“显式与隐式”的区别，对于未来的每一堂课都很重要。

## 关键术语

| 学期 | 人们怎么说 | 它实际上意味着什么 |
|------|-----------------|-----------------------|
| 生成模型 | “它创造了新东西” | 学习 `p_data(x)` 的采样器，可选择公开 `log p(x)`。 |
| 显式密度 | “你可以评价一下” | 模型提供了封闭式或易处理的 `log p(x)`。 |
| 隐含密度 | “GAN式” | 只有采样器 - 无法评估给定点的 `p(x)`。 |
| 埃尔博 | “证据下限” | `log p(x)` 上易于处理的下界； VAEs 和扩散对其进行优化。 |
| 分数 | “对数密度梯度” | `∇_x log p(x)`；扩散和 SDE 模型学习这一领域。 |
| 流形假说 | “数据存在于表面” | 高维数据集中在低维流形上；为什么降维有效。 |
| 自回归 | “预测下一个作品” | 将联合因式分解为条件条件的乘积。 |
| 潜 | “压缩代码” | 解码器可以从中重建输入的低暗度表示。 |

## 制作笔记：五个族，五个推理形状

每个系列都映射到不同的推理服务器成本曲线。产生式推理文献将 LLM 推理定义为预填充 + 解码；同样的分解也适用于这里：

- **自回归（桶 1 和 5）。** 顺序解码主导延迟； KV-cache、连续批处理和推测解码都直接应用。
- **VAE /扩散/流量匹配（桶2和4）。** LLM意义上没有解码。成本 = `num_steps × step_cost`，`step_cost` 是全潜在分辨率下的变压器或 U-Net 前向。生产旋钮包括步数（DDIM/DPM-Solver/distillation）、批量大小和精度（bf16/fp8/int4）。
- **GAN（3号桶）。** 一次向前传球。没有时间表，没有 KV 缓存。 TTFT ≈ 总延迟。这就是为什么 StyleGAN 仍然在窄域 UX 上获胜。

当您在论文摘要中看到“比扩散更快”时，请将其翻译为“更少的步骤×相同的步骤成本”或“相同的步骤×更便宜的步骤成本”。其他一切都是营销。
