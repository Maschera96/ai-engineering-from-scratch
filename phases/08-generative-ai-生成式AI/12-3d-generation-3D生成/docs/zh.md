# 3D生成

> 3D 是 2D 到 3D 杠杆作用最强的模式。 2023 年的突破是 3D 高斯泼溅。 2024-2026 生成推动层多视图扩散+顶部的 3D 重建，从单个提示或照片生成对象和场景。

**类型：** 学习
**语言：** Python
**先修：** 第 4 阶段（视觉）、第 8 阶段·07（潜在 Diffusion）
**时间：** 约45分钟

## 问题

3D 内容很痛苦：

- **表示。** 网格、点云、体素网格、符号距离场 (SDF)、神经辐射场 (NeRF)、3D 高斯。每个都有权衡。
- **数据稀缺。** ImageNet 有 1400 万张图像。最大的干净 3D 数据集（Objaverse-XL，2023）有约 1000 万个对象，质量最低。
- **内存。** 512³ 体素网格为 128M 体素；一个有用的场景 NeRF 需要 1M samples/ray. 生成比重建更难。
- **监督。** 对于 2D 图像，您拥有像素。对于 3D，您通常有少量 2D 视图，并且必须提升到 3D。

2026 堆栈将这两个问题分开。首先，使用扩散模型生成*2D 多视图图像*。其次，对这些图像进行*3D 表示*（通常是高斯分布）。

## 概念

### 表示形式：3D 高斯泼溅（Kerbl 等人，2023）

将场景表示为约 1M 3D 高斯云。每个参数都有 59 个参数：位置 (3)、协方差（6 或四元数 4 + 比例 3）、不透明度 (1)、球谐颜色（3 阶为 48，0 阶为 3）。

渲染 = 投影 + Alpha 合成。快速（4090 上 1080p 时约 100 fps）。可微分。通过梯度下降与地面实况照片进行拟合。一个场景在消费级 GPU 上只需 5-30 分钟即可完成。

2023-2024 年的两项创新最重要：
- **生成高斯图。** LGM、LRM、InstantMesh 等模型直接从一张或多张图像预测高斯云。
- **4D 高斯泼溅。** 高斯与动态场景的每帧偏移。

### 多视角扩散

微调预训练的图像扩散模型，以根据文本提示或单个图像生成同一对象的多个一致视图。 Zero123（Liu 等人，2023）、MVDream（Shi 等人，2023）、SV3D（稳定性，2024）、CAT3D（Google，2024）。通常输出物体周围的 4-16 个视图，通过高斯溅射或 NeRF 提升为 3D。

### NeRF（上下文）

神经辐射场（Mildenhall 等人，2020）。一个微型 MLP 采用 `(x, y, z, view direction)` 并输出 `(color, density)`。通过沿光线积分进行渲染。质量优于基于网格的小说视图合成，但渲染速度慢 100-1000 倍。在大多数实时应用中被高斯泼溅法取代，但在研究中仍然占主导地位。

## 构建它

`code/main.py` 实现了玩具 2D“高斯分布”拟合：将合成目标图像（平滑梯度）表示为 2D 高斯分布的总和。通过梯度下降优化位置、颜色和协方差以匹配目标。您会看到两个核心操作：前向渲染（splat + alpha-composite）和梯度下降拟合。

### 第 1 步：2D 高斯分布

```python
def gaussian_at(x, y, gaussian):
    px, py = gaussian["pos"]
    sigma = gaussian["sigma"]
    d2 = (x - px) ** 2 + (y - py) ** 2
    return math.exp(-d2 / (2 * sigma * sigma))
```

### 第 2 步：通过求和 splats 进行渲染

```python
def render(image_size, gaussians):
    img = [[0.0] * image_size for _ in range(image_size)]
    for g in gaussians:
        for y in range(image_size):
            for x in range(image_size):
                img[y][x] += g["color"] * gaussian_at(x, y, g)
    return img
```

真正的 3D 高斯泼溅按深度和 alpha 复合按顺序对高斯进行排序。我们的 2D 玩具只是求和。

### 步骤3：梯度下降拟合

```python
for step in range(steps):
    pred = render(size, gaussians)
    loss = mse(pred, target)
    gradients = compute_grads(pred, target, gaussians)
    update(gaussians, gradients, lr)
```

## 常见陷阱

- **视图不一致。** 如果您独立生成 4 个视图，并且它们在对象结构方面存在分歧，则 3D 拟合会模糊。修复：具有共享注意力的多视图扩散。
- **背面幻觉。** 单图像 → 3D 必须发明看不见的一面。质量差异很大。
- **高斯 splat 爆炸。** 无约束训练增长到 10M splat 和过拟合。致密化 + 修剪启发式（来自 3D-GS 原始论文）至关重要。
- **拓扑问题。** 隐式场 (SDF) 的网格通常有孔或自相交。在发货前运行重新网格化程序（例如搅拌机的体素重新网格化）。
- **训练数据许可。** Objaverse 具有混合许可；商业用途因型号而异。

## 生产说明：3D尚无共享基板

与图像（潜在扩散 + DiT）和视频（时空 DiT）不同，3D 在 2026 年没有单一的主导运行时间。生产决策树在表示上分叉：

- **NeRF / triplane。** 推理是光线行进 + 每个样本的 MLP 前向。 512² 渲染需要数百万个 MLP 转发。积极地对射线样本进行批处理； SDPA/xformers适用。
- **多视图扩散+LRM重建。**两级管道。第 1 阶段（多视图 DiT）是一个扩散服务器，就像第 07 课一样。第 2 阶段（LRM 转换器）是对视图的一次性前向传递。总体延迟概况是“扩散+一次性”，相应地选择每阶段服务原语。
- **SD​​S / DreamFusion。** 每个资产优化，而不是推理。构建作业，而不是请求处理程序。

对于大多数 2026 年产品，正确的答案是“根据请求运行多视图扩散模型，异步重建为 3DGS，为 3DGS 提供实时查看服务”。这可以在 GPU 推理服务器（快速）和离线优化器（慢速）之间干净地分配工作负载。
