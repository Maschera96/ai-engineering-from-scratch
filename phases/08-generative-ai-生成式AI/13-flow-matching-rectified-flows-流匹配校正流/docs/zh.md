# 流量匹配和整流流量

> Diffusion 模型需要 20-50 个采样步骤，因为它们从噪声到数据走了一条弯曲的路径。流匹配（Lipman et al., 2023）和修正流（Liu et al., 2022）训练直线路径。更直的路径意味着更少的步骤意味着更快的推理。 Stable Diffusion 3、Flux.1和AudioCraft 2都在2024年切换到流匹配。

**类型：** 建造
**语言：** Python
**先修：** 第 8 阶段·06 (DDPM)，第 1 阶段·微积分
**时间：** 约45分钟

## 问题

DDPM 的逆过程是从 `N(0, I)` 回到数据分布的 1000 步随机行走。 DDIM 将其缩减为 20-50 个确定性步骤。您想要更少的步骤，最好是一步。阻碍因素是求解逆过程的 ODE 是僵硬的；路径是弯曲的。

如果您可以训练模型，使从噪声到数据的路径为“直线”，则从 `t=1` 到 `t=0` 的单个欧拉步骤就可以工作。流匹配直接构建此：定义从 `x_1 ∼ N(0, I)` 到 `x_0 ∼ data` 的直线插值，训练向量场 `v_θ(x, t)` 以匹配其时间导数，在推理时积分。

整流流 (Liu 2022) 更进一步：通过回流过程迭代地拉直路径，从而产生逐渐接近线性的 ODE。经过两次回流迭代后，2 步采样器可与 50 步 DDPM 质量相匹配。

## 概念

![流匹配：噪声和数据之间的直线插值](../assets/flow-matching.svg)

### 直线流

定义：

```
x_t = t · x_1 + (1 - t) · x_0,   t ∈ [0, 1]
```

其中 `x_0 ~ data` 和 `x_1 ~ N(0, I)`。沿这条直线的时间导数是常数：

```
dx_t / dt = x_1 - x_0
```

定义一个神经向量场 `v_θ(x_t, t)` 并训练它以匹配该导数：

```
L = E_{x_0, x_1, t} || v_θ(x_t, t) - (x_1 - x_0) ||²
```

这是**条件流匹配**损失（Lipman 2023）。训练无需模拟：您无需展开 ODE。只需采样 `(x_0, x_1, t)` 并回归即可。

### 采样

在推理时，及时*向后*积分学习到的向量场：

```
x_{t-Δt} = x_t - Δt · v_θ(x_t, t)
```

从 `x_1 ~ N(0, I)` 开始，欧拉步降到 `t=0`。

### 整流流（Liu 2022）

直线流有效，但学习的路径*实际上并不是直的* - 它们是弯曲的，因为许多 `x_0` 可以映射到相同的 `x_1`。修正流程的回流步骤：

1. 使用随机配对训练流模型 v_1。
2. 通过将来自 `x_1` 的 v_1 积分到其着陆 `x_0` 来采样 N 对 `(x_1, x_0)`。
3. 在这些配对示例上训练 v_2。因为这些对现在是“ODE 匹配的”，所以它们之间的直线插值确实更加平坦。
4. 重复。

在实践中，2 次回流迭代可以使您接近线性，从而实现 2-4 步推理。 SDXL-Turbo、SD3-Turbo、LCM都是从流量匹配中提取的模型。

### Why this won for images in 2024

三个原因：

1. **免模拟训练** - 训练期间无需展开 ODE，实现起来很简单。
2. **更好的损耗几何** - 直线路径具有一致的信噪比，而 DDPM ε-loss 在时间表边缘具有较差的 SNR。
3. **更快的推理** — SDXL-Turbo 质量下的 4-8 个步骤； 1 步稠度蒸馏。

## 流量匹配与 DDPM — 精确连接

与高斯条件路径匹配的流是扩散*具有特定的噪声调度*。选择 `x_t = α(t) x_0 + σ(t) x_1` 计划并进行流量匹配，以 `v = α'·x_0 - σ'·x_1` 恢复 Stratonovich 重新制定的扩散。对于高斯路径来说，两者在代数上是等价的。

流匹配增加了什么：目标的“清晰度”（普通速度）、更清晰的损失以及使用非高斯插值进行实验的许可。

    x1 = rng.gauss(0, 1)
    t = rng.random()
    x_t = t * x1 + (1 - t) * x0
    target = x1 - x0

### 第2步：多步推理

```python
def sample(net, num_steps):
    x = rng.gauss(0, 1)
    for i in range(num_steps):
        t = 1.0 - i / num_steps
        dt = 1.0 / num_steps
        x -= dt * net_forward(x, t)
    return x
```

## 常见陷阱

- **时间参数化。** 流量匹配在数据处使用 `t ∈ [0, 1]` 和 `t=0`，在噪声处使用 `t=1`。 DDPM 在数据处使用 `t ∈ [0, T]` 和 `t=0`，在噪声处使用 `t=T`。方向相同，尺度不同。论文经常犯这个错误。
- **计划选择。** 修正流的直线是“流匹配计划”，但您可以使用余弦或对数正态 t 采样（SD3 就是这样做的）以获得更好的规模覆盖。
- **回流成本。** 生成用于回流的配对数据集是每个样本的完整推理过程。仅当您确实需要 1-2 步推理时才进行回流。
- **无分类器指导仍然适用。** 只需将线性组合中的 ε 替换为 v：`v_cfg = (1+w) v_cond - w v_uncond`。

## 关键术语

| 学期 | 人们怎么说 | 它实际上意味着什么 |
|------|-----------------|-----------------------|
| 流量匹配 | “直线扩散” | 训练 `v_θ(x, t)` 以沿着插值匹配 `x_1 - x_0`。 |
| 整流流 | “回流” | 纠正学习流的迭代过程。 |
| 速度场 | “v_θ” | 模型的输出 — 移动 `x_t` 的方向。 |
| 直线插值 | “路径” | `x_t = (1-t)·x_0 + t·x_1`；微不足道的目标导数。 |
| 欧拉采样器 | “一阶 ODE 求解器” | 最简单的积分器；当路径笔直时效果很好。 |
| Logit-正态 t | “SD3采样” | 将 `t` 采样集中到梯度最强的中间值。 |
| 浓度蒸馏 | “1步采样器” | 训练学生将任何 `x_t` 直接映射到 `x_0`。 |
