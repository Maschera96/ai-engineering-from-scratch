# ControlNet、LoRA 和调节

> 单独的文本是一个笨拙的控制信号。 ControlNet 可让您克隆预训练的扩散模型，并使用深度图、姿势骨架、涂鸦或边缘图像来引导它。 LoRA 可让您通过训练 1000 万个参数来微调 2B 参数模型。他们共同将 Stable Diffusion 从玩具变成了每个机构都会提供的 2026 年图像管道。

**类型：** 建造
**语言：** Python
**先修：** 第 8 阶段·07（潜在 Diffusion），第 10 阶段（从头开始的法学硕士 — 用于 LoRA 基金会）
**时间：** 〜75分钟

## 问题

像“一个穿红裙子的女人在繁忙的街道上遛狗”这样的提示不会给模型提供有关狗在“哪里​​”、女人的“什么姿势”或街道的“视角”的信息。文本确定了指定图像所需内容的大约 10%。其余的都是视觉上的，无法用语言有效地描述。

从头开始为每个信号（姿势、深度、精明、分割）训练一个新的条件模型是令人望而却步的。您希望保持 2.6B 参数 SDXL 主干网冻结，附加一个读取调节的小型侧网络，并让它推动主干网的中间特征。即ControlNet。

您还想向模型传授新概念（您的脸、您的产品、您的风格），而不需要重新训练整个模型。您想要一个小 100 倍的增量。这就是 LoRA，插入现有注意力权重的低阶适配器。

ControlNet + LoRA + 文本 = 2026 年实践者工具包。大多数生产图像管道在 SDXL / SD3 / Flux 基座之上分层 2-5 LoRA、1-3 ControlNet 和 IP 适配器。

## 概念

![ControlNet 克隆编码器； LoRA 添加低阶增量](../assets/controlnet-lora.svg)

### ControlNet（Zhang 等人，2023）

采取预先训练的 SD。 *克隆* U-Net 的编码器一半。冻结原件。训练克隆接受额外的调节输入（边缘、深度、姿势）。使用*零卷积*跳跃连接将克隆连接回原始解码器的一半（1×1 卷积初始化为零 - 作为无操作开始，学习增量）。

```
SD U-Net decoder:   ... ← orig_enc_features + zero_conv(controlnet_enc(condition))
```

零转换初始化意味着 ControlNet 作为身份启动 - 即使在训练之前也没有坏处。使用标准扩散损失训练 1M（提示、条件、图像）三元组。

每个模态的 ControlNet 作为小型侧面型号发货（SDXL 约为 360M，SD 1.5 约为 70M）。您可以通过推理来组合它们：

```
features += weight_a * control_a(depth) + weight_b * control_b(pose)
```

### LoRA（Hu 等人，2021）

对于模型中的任何线性层 `W ∈ R^{d×d}`，冻结 `W` 并添加低秩增量：

```
W' = W + ΔW,  ΔW = B @ A,  A ∈ R^{r×d},  B ∈ R^{d×r}
```

与`r << d`。 4-16 级是注意力的标准，64-128 级是重度微调。新参数数量：`2 · d · r` 而不是 `d²`。对于 SDXL 注意 `d=640`、`r=16`：每个适配器 20k 参数，而不是 410k — 减少了 20 倍。整个型号：LoRA 通常为 20-200MB，而基础版为 5GB。

推理时，您可以缩放 LoRA：`W' = W + α · B @ A`。 `α = 0.5-1.5` 是正常的。多个 LoRA 相加堆叠（通常需要注意的是它们以非线性方式相互作用）。

### IP 适配器（Ye 等人，2023）

一个微型适配器，接受*图像*作为条件（与文本一起）。使用 CLIP 图像编码器生成图像标记，将它们与文本标记一起注入交叉注意力。每个基本模型约 20MB。允许您在没有 LoRA 的情况下“生成此参考样式的图像”。

## 可组合性矩阵

| 工具 | 它控制什么 | 尺寸 | 何时使用 |
|------|------------------|------|-------------|
| ControlNet | 空间结构（位姿、深度、边缘） | 70-360MB | 精确的布局、构图 |
| LoRA | 风格、主题、概念 | 20-200MB | 个性化、风格化 |

ControlNet ≈ 空间。 LoRA ≈ 语义。两者都用。

## 构建它

`code/main.py` 在一维上模拟两种机制：

1. **LoRA.** 预训练的线性层 `W`。冷冻它。训练低秩 `B @ A` 以使 `W + BA` 匹配目标线性层。表明 `r = 1` 足以完美学习 1 阶校正。

2. **ControlNet-lite。** 一个“冻结基础”预测器和一个读取额外信号的“侧网络”。侧网络的输出由初始化为零的可学习标量（我们的零转换版本）进行门控。训练并观察闸门的上升。

### 第 1 步：LoRA 数学

```python
def lora(W, A, B, x, alpha=1.0):
    # W is frozen; A, B are the trainable low-rank factors.
    return [W[i][j] * x[j] for i, j in ...] + alpha * (B @ (A @ x))
```

### 步骤 2：零初始化侧网络

```python
side_out = control_net(x, condition)
gated = gate * side_out  # gate initialized to 0
h = base(x) + gated
```

在第 0 步，输出与基数相同。早期训练缓慢地更新 `gate`，没有灾难性的漂移。

## 常见陷阱

- **过度缩放 LoRA。** `α = 2` 或 `α = 3` 是一种常见的“使其更强”黑客，会产生过度风格化/损坏的输出。保留`α ≤ 1.5`。
- **ControlNet 权重冲突。** 使用权重为 1.0 的姿势 ControlNet 和权重为 1.0 的深度 ControlNet 通常会出现过冲。权重总和 ≈ 1.0 是安全的默认值。
- **LoRA 的基础错误。** SDXL LoRA 在 SD 1.5 上默默无操作，因为注意力维度不匹配。扩散器将在 0.30+ 时发出警告。
- **文本反转漂移。** 在一个检查点上训练的词元在另一个检查点上严重漂移。 LoRA更加便携。
- **LoRA 权重合并和存储。** 您可以将 LoRA 烘焙到基本模型权重中以实现更快的推理（无运行时添加），但您失去了在运行时缩放 `α` 的能力。保留两个版本。

## 练习

1. **简单。** 在 `code/main.py` 中，将 LoRA 等级 `r` 从 1 更改为 4。LoRA 在什么等级时与等级 2 目标增量完全匹配？
2. **中。** 在两个目标变换上训练两个单独的 LoRA。将它们加载在一起并显示它们的附加相互作用。相互作用何时打破线性？
3. **硬。** 使用扩散器堆叠：SDXL-底座 + Canny-ControlNet（重量 0.8）+ LoRA（α 0.8）+ IP 适配器（重量 0.6）。随着堆叠重量的变化，衡量 FID 与提示依从性的权衡。

## 生产说明：LoRA 交换、ControlNet 通道、多租户服务

真正的文本到图像 SaaS 在同一基本检查点上为数百个 LoRA 和十几个 ControlNet 提供服务。服务问题看起来很像LLM多租户（生产文献涵盖了连续批处理和LoRAX / S-LoRA下的LLM案例）：

- **热插拔 LoRA，不要合并。** 将 `W' = W + α·B·A` 合并到基础中可以使每步推理速度加快约 3-5%，但会冻结 `α` 和基础。使 LoRA 在 VRAM 中保持热状态，作为 R 级增量；扩散器公开 `pipe.load_lora_weights()` + `pipe.set_adapters([...], adapter_weights=[...])` 以用于每个请求激活。交换成本是 `2 · d · r · num_layers` 权重 — MB 级，亚秒级。
- **ControlNet 作为第二个注意力通道。** 克隆编码器与底座并行运行。两个权重为 1.0 的 ControlNet = 每步两次额外的前向传球，而不是一次合并传球。批量大小的余量呈二次方下降。每个活动 ControlNet 的预算约为 1.5 倍步长成本。
- **也量化了 LoRA。** 如果您量化了基数（请参阅第 07 课，8GB 上的 Flux），LoRA 增量也会干净地量化为 8 位或 4 位。 QLoRA 式加载可让您在 4 位 Flux 基座上堆叠 5-10 个 LoRA，而不会耗尽内存。

Flux-特定：Niels 的 Flux-on-8GB 笔记本将基数量化为 4 位；在 `weight_name="pytorch_lora_weights.safetensors"` 的量化基础上堆叠样式 LoRA (`pipe.load_lora_weights("user/style-lora")`) 仍然有效。这是大多数 SaaS 机构在 2026 年推出的方案。
