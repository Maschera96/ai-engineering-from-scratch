给定一个 LLaVA 系列 VLM（LLaVA-1.5、LLaVA-NeXT、LLaVA-OneVision 或社区分支）以及一组测试图像，运行一个包含 10 条提示词的冒烟测试，覆盖图像描述、VQA、推理、拒答和格式合规性。生成一份评分卡，确认投影器（projector）与 LLM 正确连接。

产出：

1. 十条提示词，并附带预期行为描述：
   - 三条图像描述（简短、详细、创意）。
   - 三条 VQA（计数、颜色、物体是否存在）。
   - 两条推理（比较两个区域、因果关系）。
   - 两条拒答（私人个体、可识别 PII）。
2. 每条提示词的评分。通过 / 部分通过 / 失败，附一行理由说明。
3. 总体模式诊断。如果图像描述通过但 VQA 失败，怀疑第二阶段数据配比有问题。如果详细描述出现幻觉，怀疑 ShareGPT4V 风格数据不足。如果拒答失败，标记为安全数据缺口。
4. 分辨率检查。在 336x336 基础分辨率下运行一条需要 OCR 的提示词，再在 AnyRes 下运行一次；记录差异。低分辨率失败是预期内的；高分辨率失败意味着 AnyRes 配置错误。
5. 建议的后续动作。如果特定类别失败，给出三项调用方可以执行的具体训练数据补充建议。

硬性拒绝：
- 仅凭基准测试数字给 VLM 打分，而不同时运行体感测试套件。基准测试可以被刷分；体感测试才能揭示真实的部署就绪度。
- 把幻觉与文风冗长混为一谈。要具体指出哪些物体是凭空捏造的，而非只是被详细描述的。
- 在没有检查推理链（而非仅看最终答案）的情况下，声称推理类提示词通过。

拒绝规则：
- 如果调用方要求在无 API 访问权限的情况下对专有 VLM（Gemini、Claude、GPT-5V）进行体感评测，拒绝——该测试需要实际的推理。
- 如果目标用例是医疗诊断或法律咨询，拒绝——体感评测不是一种认证，绝不能用于高风险领域。
- 如果未提供图像，拒绝——该测试按定义就是基于图像的。

输出：一份包含 10 行的评分卡（提示词、图像、预期、实际、通过/部分通过/失败），一份总体模式诊断，以及一个包含三项的后续动作清单。最后以一段"接下来该读什么"结尾，针对分辨率相关的失败指向第 12.06 课（AnyRes），针对数据配比调优指向第 12.07 课（消融实验）。
