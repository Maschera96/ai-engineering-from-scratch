# Qwen-VL 家族与动态 FPS 视频

> Qwen-VL 家族——Qwen-VL（2023）、Qwen2-VL（2024）、Qwen2.5-VL（2025）、Qwen3-VL（2025）——是 2026 年最具影响力的开源视觉语言模型谱系。每一代都押注于一个决定性的架构选择，而开源生态的其余部分会在十二个月内复制它：通过 M-RoPE 实现的原生动态分辨率、带绝对时间对齐的动态 FPS 采样、ViT 中的窗口注意力，以及结构化的智能体输出格式。到 Qwen3-VL，这套配方已经稳定下来：一个采用原生宽高比输入的 2D-RoPE-ViT 编码器、一个接入大型 Qwen3 语言基座的 MLP 投影器，以及把 OCR、定位和智能体行为作为一等目标来强调的训练阶段。本课按时间顺序梳理整个家族，让你理解每个旋钮为什么在它现在的位置。

**类型：** 学习
**语言：** Python（标准库，M-RoPE 编码器 + 动态 FPS 采样器）
**前置：** Phase 12 · 06（patch-n'-pack）
**时长：** ~120 分钟

## 学习目标

- 计算 M-RoPE 的三轴旋转（时间、高度、宽度），并解释为什么三者都需要。
- 为一段视频选择动态 FPS 采样策略，并就每秒 token 数与事件检测准确率之间的权衡进行推理。
- 按顺序说出 Qwen-VL 的四代升级，以及每一代实现了什么。
- 接入 Qwen2.5-VL 风格的 JSON 智能体输出格式，并从 VLM 响应中解析结构化的工具调用。

## 问题

Qwen-VL 于 2023 年 8 月发布，是对 LLaVA-1.5 和 BLIP-2 的直接回应。Qwen 团队瞄准的差距有三方面：分辨率、视频和结构化输出。

分辨率：LLaVA-1.5 以 336x336 运行。这对照片足够，但对一张中文发票或密集的电子表格截图毫无用处。Qwen-VL 的第一个创新是 448x448 和带定位的边界框输出，让模型能够指向具体事物。

视频：Video-LLaMA 堆叠逐帧编码器，再把它们喂给 LLM。这对短片段有效，但对那种时间轴本身就是信号的多分钟视频则不行。Qwen 团队想要一个能理解时间的单一编码器。

结构化输出：LLaVA 产生自由格式文本。智能体需要 JSON。Qwen-VL 在显式的 JSON 输出格式上训练，包括把边界框坐标作为文本输出。

每一代 Qwen-VL 都在这三条轴中的某一条上扩展。

## 概念

### Qwen-VL（2023 年 8 月）

第一代：以 OpenCLIP ViT-bigG/14 作为编码器（2.5B 参数）、LLama 兼容的 Q-Former（1 步、256 个查询）、Qwen-7B 基座。贡献：

- 448x448 分辨率（当时开源 VLM 的 SOTA）。
- 定位：在带显式坐标 token 输出的图文对上训练。"The cat is at <box>(112, 204), (280, 344)</box>"。
- 从一开始就进行中英双语训练。

当时的基准测试：在英文上与 GPT-4V 竞争，在中文上占据主导。定位监督才是真正的亮点。

### Qwen2-VL（2024 年 9 月）—— M-RoPE 与原生分辨率

Qwen2-VL 用一个原生动态分辨率的 ViT 编码器替换了固定分辨率 + Q-Former 的栈。关键变化：

- 原生动态分辨率。ViT 接受任意能被 28 整除的 HxW（patch 14 配 2x 空间合并）。一张 1120x672 的图像（40x24 个合并后的 patch）产生 960 个视觉 token。无需缩放、无需切片、无需缩略图。
- M-RoPE（Multimodal RoPE）。每个 token 携带一个 3D 位置 (t, h, w) 而非 1D。图像 t=0，视频 t = frame_index。RoPE 按每个轴的频率旋转查询/键向量。没有位置嵌入表。
- MLP 投影器。去掉 Q-Former；在合并后的 patch token 上使用一个 2 层 MLP。
- 带动态 FPS 的视频。视频默认以 1-2 FPS 采样，但模型接受任意帧数。

结果：Qwen2-VL-7B 在多个多模态基准上追平了 GPT-4o，并在 DocVQA 上击败了它（94.5 对 88.4）。架构变化是决定性的一步。

### Qwen2.5-VL（2025 年 2 月）—— 动态 FPS + 绝对时间

Qwen2.5-VL 的重大转变在于视频。动态 FPS 不只是"需要时多采几帧"。论文将其形式化：

- 绝对时间 token。不用位置索引（帧 0、1、2……），而用真实时间戳。"在 0:04，猫跳了起来。"模型看到的是与帧 token 交错的 `<time>0.04</time>` token。
- 动态 FPS。对慢速画面以 1 FPS 采样，对动作画面以 4+ FPS 采样。由用户或训练者选择；M-RoPE 自适应。
- ViT 中的窗口注意力。空间注意力按窗口处理（块内局部）以提升吞吐；每隔几层做一次全局注意力。
- 显式 JSON 输出格式。在工具调用数据上训练："{\"tool\": \"click\", \"coords\": [380, 220]}"。开箱即用、面向智能体。
- MRoPE-v2 缩放。位置随最大输入尺寸缩放，使一段 10 分钟的视频不会耗尽频率范围。

基准测试：Qwen2.5-VL-72B 在大多数视频基准上击败 GPT-4o，在文档上追平 Gemini 2.0，并在 GUI 定位上确立了开源模型的 SOTA（ScreenSpot：84% 准确率，而 GPT-4o 为 38%）。

### Qwen3-VL（2025 年 11 月）

Qwen3-VL 是一次增量升级，重在巩固而非重新发明：更大的 LLM 主干（Qwen3-72B）、扩充的训练数据、改进的 OCR，以及通过 Qwen3 的"思考模式"获得的更强推理能力。ViT 和 M-RoPE 保持不变。论文聚焦于数据和训练改进，而非架构。

谱系的要点：到 2025 年，Qwen-VL 架构已经稳定。后续代际扩展的是算力和数据，而非基本组件。

### M-RoPE 的数学表述

经典 RoPE 用成对坐标按位置 `m` 旋转一个维度为 `d` 的查询 `q`：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE 把隐藏维度分成三个频段。假设 `d = 96`。把 32 维分给时间、32 维分给高度、32 维分给宽度。每个频段按自己的轴位置旋转。位于 (t=5, h=10, w=20) 的一个 patch 会对其三个频段分别施加旋转 `R_t(5)`、`R_h(10)`、`R_w(20)`。

文本 token 使用 `t = text_index, h = 0, w = 0`（或某种归一化选择），从而保持兼容。视频帧使用 `t = frame_time, h = row, w = col`。单张图像使用 `t = 0`。

好处是：一套位置编码就能处理文本、图像和视频，无需分支代码或不同的位置表。

### 动态 FPS 采样逻辑

给定一段时长为 `T` 秒的视频和一个目标 token 预算 `B`：

1. 计算你能承受的最大 FPS：`fps_max = B / (T * tokens_per_frame)`。
2. 从 `{1, 2, 4, 8}` 中挑一个满足 `fps <= fps_max` 的目标 FPS。
3. 如果运动量高（光流启发式或用户显式请求），选更高的 FPS。如果运动量低，选更低的。
4. 以选定的 FPS 均匀采样；在帧之间插入 `<time>t</time>` token。

Qwen2.5-VL 隐式地训练这套逻辑；在推理时用户通过 `fps` 参数控制。一段 60 秒的动作序列以 4 FPS、每帧 81 token 计 = 19440 token，在 32k 上下文中可控。

### 结构化智能体输出

Qwen2.5-VL 的智能体训练显式瞄准结构化工具调用：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析是确定性的：对模型输出执行 JSON.parse。相比之下，自由格式的 "click at (1024, 512)" 需要正则和歧义处理。这种转变正是 Qwen2.5-VL 的 ScreenSpot 分数从 Qwen2-VL 的 55% 跃升到 84% 的原因。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| M-RoPE | "Multimodal RoPE" | 在隐藏维度中带时间、高度和宽度频段的 3D 旋转位置嵌入 |
| 动态 FPS | "智能采样" | 根据运动量、时长和 token 预算为每段视频选择的帧采样率 |
| 绝对时间 token | "时间戳 token" | 在序列中交错的 `<time>t</time>`，使模型看到的是真实秒数而非帧索引 |
| 窗口注意力 | "局部注意力" | 为提速而限制在小窗口内的空间自注意力；周期性地加入全局注意力 |
| 结构化智能体输出 | "JSON 模式" | 训练数据监督，教 VLM 输出可解析的、带坐标和工具名的 JSON |
| min_pixels / max_pixels | "分辨率边界" | 每个请求的 Qwen2.5-VL 控制项，约束总像素数从而约束 token 数 |
| 定位（Grounding） | "指向它" | 把边界框坐标作为文本 token 输出；自 Qwen-VL v1 起使用 |
