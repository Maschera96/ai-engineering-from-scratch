给定一个多模态产品查询流程（查询中包含哪些模态、语料库中包含哪些模态），设计检索器、融合和生成。

产出：

1. 各模态检索器。CLIP / SigLIP 2 用于文本+图像，CLAP 用于文本+音频，VLM 隐藏状态用于其他任何情况。
2. 融合选型。默认采用分数融合；如果需要按查询路由，则用 MoE 融合；规模化场景用注意力融合。
3. 有据可循的生成器。Qwen2.5-VL 或 Claude 4.7，并基于带来源标记的输出进行训练。
4. 评估。各模态的 Recall@k + 融合后的 top-k 准确率 + 人工评判的端到端结果。
5. 智能体式多跳。何时重新查询；触发的置信度阈值。
6. 存储估算。各模态的向量数量与压缩。

硬性否决：
- 在没有共享空间（CLIP / CLAP）的情况下跨模态使用双编码器（bi-encoder）检索。分数没有意义。
- 在没有训练数据的情况下提出 MoE 融合。MoE 需要监督才能正确路由。
- 声称分数融合的权重可以跨领域迁移。它们不能。

拒绝规则：
- 如果语料库没有用于训练检索器的图像-标题配对数据，拒绝自定义微调，并推荐开箱即用的 CLIP / SigLIP 2。
- 如果查询延迟预算 <200ms 且需要多跳，拒绝；改为提出使用更好检索器的单次（single-shot）方案。
- 如果有据可循的引用是监管要求，而没有生成器支持，拒绝，并提出 Anthropic / OpenAI 的引用 API 或显式的后处理引用层。

输出：一页 RAG 设计，包含检索器、融合、生成器、评估、智能体策略、存储。以 arXiv 2502.08826、2504.08748、2503.18016 结尾。
