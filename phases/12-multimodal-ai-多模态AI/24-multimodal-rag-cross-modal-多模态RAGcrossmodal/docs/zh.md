# 多模态 RAG 与跨模态检索

> Vision-native 文档 RAG 只是其中一个切片。生产级多模态 RAG 涉及的范围更广——它在文本、图像、音频和视频之间检索，以支撑诸如行程规划（"帮我找一家安静、有自然光的纯素早午餐店"）、医疗分诊（"哪种伤情与这张照片加这些笔记相符"）、电商（"和这张自拍风格相似、符合我尺码的穿搭"）以及现场维修（"诊断这段发动机异响加这张零件照片"）之类的工作流。2025 年的三篇综述——Abootorabi 等人、Mei 等人、Zhao 等人——把这些子问题归纳成体系：跨模态检索、检索融合、生成接地（grounding）、多模态评估。本课将通读这些综述，并设计一条生产级流水线。

**类型：** 构建
**语言：** Python（标准库，带融合 + 接地生成器的跨模态检索器）
**前置：** Phase 12 · 23（ColPali）、Phase 11（RAG 基础）
**时长：** ~180 分钟

## 学习目标

- 设计跨模态检索：文本 → 图像、图像 → 文本、音频 → 视频，等等。
- 比较三种融合策略：分数融合、基于注意力的融合、MoE 融合。
- 解释生成接地：当来源是多种模态的混合时，"标注出处"是什么样子。
- 说出 2025 年三篇经典的多模态 RAG 综述及其子问题分类法。

## 问题

单模态 RAG 是一种已被解决的范式：嵌入查询、嵌入分块、检索、塞进 LLM。多模态 RAG 还需要：

1. 多个检索头（每种模态都需要在兼容空间中的嵌入）。
2. 跨模态的检索结果融合。
3. 能够跨模态标注出处的生成接地。
4. 覆盖跨模态信号的评估指标。

2025 年的这些综述最终都得出了同样的分类法。

## 概念

### 跨模态检索

给定模态 A 的查询，检索模态 B 的文档。三种模式：

1. 共享嵌入空间。CLIP 和 CLAP 在一个共享空间中生成文本 + 图像 / 文本 + 音频的嵌入。跨模态的余弦相似度可直接使用。仅限于 CLIP 训练过的配对。

2. 每模态编码器 + 翻译。文本编码器 + 图像编码器 + 一个在空间之间映射的小型翻译模块。如 Gupta 等人的 Sen2Sen 以及 2024 年的其他设计。灵活，但增加了复杂度。

3. VLM 作为编码器。把 VLM 的隐藏状态用作检索表示。VLM 支持的任意模态都适用。质量更高，成本更高。

选择：文本+图像用 CLIP / SigLIP 2；文本+音频用 CLAP；前沿质量的跨模态用 VLM 隐藏状态。

### 融合策略

你检索到了 10 条结果：5 张图像、3 段文本、2 段音频。如何合并？

分数融合（最便宜）。每种模态有自己的检索器，各自返回分数。在模态内部归一化分数，然后相加。简单，往往有效。

基于注意力的融合。把所有检索到的条目拼接起来，让一个小型注意力网络为它们赋权。需要训练。

MoE 融合。门控网络将查询路由到模态专属的专家。不同类型的查询路由方式不同——视觉类问题会给图像更高的权重。

生产默认方案：分数融合，并略微偏向查询的主导模态。如果 A/B 测试在你的领域显示出明显收益，再升级到 MoE。

### 生成接地

LLM 应当标注是哪条检索条目支撑了每个论断。对于多模态：

- 文本来源：标准引用 `[1]`。
- 图像来源：`[img 3]`，附一句简短说明。
- 音频：`[audio 2 at 0:34]`。

用具备接地意识的数据训练生成器：训练目标中的每个论断都标注上来源索引。在推理时，模型自然会发出引用。

### 2025 年的综述

Abootorabi 等人（arXiv:2502.08826，"Ask in Any Modality"）：多模态 RAG 的分类法。涵盖检索、融合、生成。覆盖面最广。

Mei 等人（arXiv:2504.08748，"A Survey of Multimodal RAG"）：聚焦子任务基准和失败模式。对评估设计很有用。

Zhao 等人（arXiv:2503.18016）：以视觉为中心的综述。在 ColPali 系列工作上很扎实。

通读这三篇能让你掌握截至 2025 年春季的最新进展。大多数子问题仍是开放的。

### MuRAG——奠基性论文

MuRAG（Chen 等人，2022）是第一个多模态 RAG。从多模态知识库中检索图像 + 文本，生成答案。在 VLM 浪潮到来之前就证明了可行性。现代系统（REACT、VisRAG、M3DocRAG）都建立在它之上。

### 一个生产级行程规划示例

查询："帮我找一家安静、有自然光的纯素早午餐店。"

流水线：

1. 拆解查询。"安静" → 音频/评论关键词；"纯素早午餐" → 菜单项；"自然光" → 图像特征。
2. 按模态检索：
   - 在评论上做文本检索："纯素早午餐，安静氛围。"
   - 在餐厅照片上做图像检索："自然光，通透。"
   - 在环境声片段上做音频检索："低分贝，无音乐。"
3. 融合分数。每家餐厅得到一个综合分数。
4. Top-k 餐厅 → 携带全部证据的 VLM 生成器 → 带引用的答案。

这远超文本 RAG。每种模态都补充了仅靠文本会遗漏的信号。

### 智能体式多模态 RAG

多跳：如果第一次检索没有返回高置信度的答案，LLM 会重新表述查询并再次检索。Phase 14 中的智能体式 RAG 模式在这里同样适用。示例：

- 检索初始 top-10 → LLM 提出"太吵了，过滤出 <40 dB 的" → 重新检索。
- 检索图像 → LLM 看到其中一张有菜单 → 检索菜单文本 → 作答。

增加了复杂度，但能处理单次检索无法应对的查询。

### 评估

跨模态评估仍不成熟。常见的代理指标：

- 各模态的 Recall@k。
- 融合后的 top-k 准确率。
- 人工评判的端到端满意度。
- 任务相关指标（完成的预订数、达成的购买数）。

没有横跨所有模态的标准基准。大多数论文在领域特定任务上做评估。

## 动手用

`code/main.py`：

- 三个模拟检索器（文本、图像、音频），运行在一个共享的餐厅语料库上。
- 分数融合，用可配置的权重组合各模态分数。
- 一个生成器桩，输出带引用的最终答案。
- 一个简单的智能体式循环，在置信度低时重新表述查询。

## 练习

1. 提出一个医疗分诊多模态 RAG：查询 = 伤情照片 + 文本症状。哪些模态分别从哪些知识库检索？

2. 分数融合是一个简单的加权求和。它有哪种 MoE 融合能够避免的失败模式？

3. 阅读 Abootorabi 等人的分类法（第 3 节）。三个经典子问题是什么，它们如何映射到你所选的产品？

4. 为一个行程规划多模态 RAG 设计一份评估规格。哪些指标能覆盖图像召回、音频召回和综合正确性？

5. 智能体式多跳 RAG 每次往返都有延迟开销。在何种查询难度下，准确率的提升才足以证明这份延迟开销是值得的？

## 关键术语

| 术语 | 人们怎么说 | 它实际指什么 |
|------|-----------------|------------------------|
| 跨模态检索 | "用一种模态查询，检索另一种模态" | 文本查询检索图像；图像查询检索文本；需要共享空间或翻译器 |
| 分数融合 | "组合分数" | 各模态检索分数的加权求和；最简单的融合 |
| MoE 融合 | "按模态路由的专家" | 门控网络按查询挑选信任哪种模态的分数 |
| 接地生成 | "标注你的出处" | 答案中的每个论断都标注上来源索引 |
| MuRAG | "第一个多模态 RAG" | 2022 年确立多模态 RAG 范式的论文 |
| 智能体式多跳 | "重新表述并重试" | 当首轮置信度低时，LLM 重新查询检索器 |
