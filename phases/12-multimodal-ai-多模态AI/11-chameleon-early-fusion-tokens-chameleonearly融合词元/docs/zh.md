# Chameleon 与早融合纯词元多模态模型

> 我们目前见过的每个 VLM 都把图像和文本分开处理。视觉词元来自视觉编码器，流经一个投影器，然后在 LLM 内部与文本相遇。视觉词表和文本词表从不重叠。Chameleon（Meta，2024 年 5 月）提出了一个问题：如果它们重叠会怎样？训练一个 VQ-VAE，把图像转换成来自共享词表的离散词元序列。现在每篇多模态文档都是一条序列——文本词元和图像词元交错排列，单一的自回归损失。副作用是：模型可以生成混合模态的输出——在一次推理调用中交替输出文本和图像词元。本课研读早融合的论点，并端到端构建一个玩具版本。

**类型：** 构建
**语言：** Python（标准库，VQ-VAE 分词器 + 交错解码器）
**前置知识：** 第 12 阶段 · 05、第 8 阶段（生成式 AI）
**时长：** 约 180 分钟

## 学习目标

- 解释为什么共享词表 + 单一损失会改变模型能做的事。
- 描述 VQ-VAE 如何把一张图像分词成与 transformer 下一词元目标兼容的离散序列。
- 说出 Chameleon 的训练稳定性技巧：QK-Norm、dropout 放置位置、LayerNorm 排序。
- 对比 Chameleon 与 BLIP-2 的 Q-Former 方法，并描述各自适用的场景。

## 问题

基于适配器的 VLM（LLaVA、BLIP-2、Qwen-VL）把文本和图像当作两种不同的东西。文本词元经过 `embed(text_token)`；图像经过 `visual_encoder(image) → projector → ... pseudo_tokens`。模型有两条输入路径，在中途某处汇合。

三个后果：

1. LLM 只能消费图像，不能产出图像。输出只有文本。
2. 混合模态文档（如文章中交替出现的段落和图像）很别扭——你要么在模型外部解析多模态输入，要么链式生成。
3. 分布不匹配。视觉词元和文本词元位于隐空间的不同区域，造成微妙的对齐问题。

Chameleon 否定了这个前提：图像不过是来自共享词表的离散词元序列。在交错文档上训练模型，单一损失、单一自回归解码器，你就免费解锁了混合模态生成能力。

## 概念

### 作为图像分词器的 VQ-VAE

分词器是一个向量量化变分自编码器。架构如下：

- 编码器：CNN + ViT，把图像映射成空间特征图，比如 32x32 个维度为 256 的特征。
- 码本：一个学习得到的 K 个向量的词表（Chameleon 使用 8192 个），维度同样为 256。
- 量化：对每个空间特征，按 L2 距离查找最近的码本条目。用整数索引替换连续特征。
- 解码器：CNN，把量化后的特征还原回像素。

训练：VAE 重构损失 + 承诺损失 + 码本损失。码本索引构成了图像的离散字母表。

对 Chameleon 而言：一张图像变成 32*32 = 1024 个词元，取自一个 8192 大小的词表。与文本词元（来自 LLM 的 BPE 词表，比如 32000）拼接。最终词表大小：40192。transformer 看到的是一条序列、一个损失。

### 共享词表

Chameleon 的词表合并了文本词元、图像词元和模态分隔符。每个词元有一个唯一 ID。输入嵌入层把每个 ID 映射成一个 D 维隐向量。输出投影把隐向量映射回词表 logits。Softmax 选出下一个词元，不论何种模态。

分隔符很重要：`<image>` 和 `</image>` 标签括住图像词元序列。在生成时，如果模型输出了 `<image>`，下游软件就知道接下来的 1024 个词元是要送进解码器渲染像素的 VQ 索引。

### 混合模态生成

推理就是在共享词表上做下一词元预测。示例提示词："Draw a cat and describe it."（画一只猫并描述它。）Chameleon 输出：

```
<image> 4821 1029 2891 ... (1024 image tokens) </image>
The cat is orange, sitting on a windowsill...
```

模型自主选择顺序——它可能先出图像后出文本，先文本后图像，或者交错出现。同一个解码器，同一个损失。

对比适配器式 VLM 的生成只能是纯文本。Chameleon 重新打开了模型输出模态的问题。

### 训练稳定性——QK-Norm、dropout、LayerNorm 排序

早融合训练在大规模下不稳定。Chameleon 的论文记录了三个技巧：

- QK-Norm。在注意力内部对 query 和 key 投影应用 LayerNorm，在点积之前。防止深层处 logit 幅值爆炸。2024 年之后多个大模型都采用。
- dropout 放置位置。在每个残差相加之后都做 dropout，不只是在注意力和 MLP 之后。当来自图像词元的梯度可能占主导时，需要更多正则化。
- LayerNorm 排序。残差分支上用 Pre-LN（标准做法），外加在最后一个块的跳跃连接上额外加一个 LN。稳定末层的梯度流。

没有这些技巧，340 亿参数的 Chameleon 训练在多个检查点处发散。有了它们，训练收敛。训练配方与架构本身一样是重要的贡献。

### 分词器的重构上限

VQ-VAE 是有损的。在 8192 个码本条目、每张 512x512 图像 1024 个词元的设置下，重构 PSNR 上限约 26-28 dB。这足以做出可辨认的图像生成，但明显比连续空间扩散更差（Stable Diffusion 3 达到 32+ dB）。

分词器是瓶颈。更好的分词器（MAGVIT-v2、IBQ、SBER-MoVQGAN）能抬高上限。Emu3（第 12.12 课）仅凭一个更好的分词器就达到了 SDXL 级别的生成质量。

### Chameleon 对比 BLIP-2 / LLaVA

Chameleon（早融合，共享词表）：
- 一个损失，一个解码器。
- 生成混合模态输出。
- 分词器是质量上限。
- 昂贵：推理路径上每生成一张图像都要跑一次 VQ-VAE 解码器。

BLIP-2 / LLaVA（晚融合，独立塔）：
- 只能图像进、文本出。
- 复用预训练 LLM。
- 理解任务没有分词器瓶颈。
- 便宜：单次前向传播。

按任务选择。如果你需要图像生成，选 Chameleon 家族。如果你只需要理解，适配器式 VLM 更简单，且复用了更多预训练算力。

### Fuyu 与 AnyGPT

Fuyu（Adept，2023）是一种相关方法：完全跳过独立的视觉编码器，把原始图像 patch 当作词元一样直接喂进 LLM 的输入投影，不用分词器。比 Chameleon 更简单，但失去了共享词表的输出生成能力。

AnyGPT（Zhan 等，2024）把 Chameleon 扩展到四种模态：文本、图像、语音、音乐。每种都用同样的 VQ-VAE 技巧，共享 transformer。任意到任意生成。在第 12.16 课有更多介绍。

## 动手用

`code/main.py` 构建了一个玩具级端到端早融合模型：

- 一个微型 VQ-VAE 风格的量化器，把 8x8 patch 映射成码本索引（K=16）。
- 一个共享词表，由（文本 id 0..31）+（图像 id 32..47）+（分隔符 48、49）组成。
- 一个玩具自回归解码器（二元组表），在合成的字幕 + 图像词元序列上训练。
- 一个采样循环，根据提示词输出交替的文本 + 图像词元。

代码刻意把 transformer 保持得很小（二元组），以便你端到端地追踪信号流。

## 练习

1. Chameleon 使用 K=8192 个码本条目、每张 512x512 图像 1024 个词元。估算相对于 24 位 RGB 图像的压缩比。它是有损的吗？有损到什么程度？

2. 一张 4K 图像（3840x2160）在同样的 VQ-VAE 密度下会产生多少个图像词元？Chameleon 风格的模型能在一次推理调用中生成一张 4K 图像吗？最先崩溃的是什么——上下文、分词器质量，还是 KV 缓存？

3. 用纯 Python 实现 QK-Norm。给定一个 64 维的 query 和 key，展示 LayerNorm 前后的点积。为什么在深层处控制幅值很重要？

4. 阅读 Chameleon 第 2.3 节关于训练稳定性的内容。描述论文在 34B 规模下不用 QK-Norm 时观察到的确切失败模式。"范数爆炸"的特征是什么？

5. 扩展玩具解码器，使其针对纯文本提示词输出混合模态响应。在训练数据分布为 60% 文本优先 / 40% 图像优先的情况下，测量模型选择图像优先 vs 文本优先的频率。

## 关键术语

| 术语 | 人们怎么说 | 它实际的意思 |
|------|-----------------|------------------------|
| 早融合 | "统一词元" | 图像从第一步起就转换成共享 transformer 词表的离散词元 |
| VQ-VAE | "图像分词器" | CNN + ViT + 码本，把图像映射成 transformer 可预测的整数索引 |
| 共享词表 | "一本字典" | 涵盖文本 + 图像 + 模态分隔符的单一词元 ID 空间 |
| QK-Norm | "注意力稳定器" | 在 query 和 key 点积之前对其应用 LayerNorm，防止范数爆炸 |
| 混合模态生成 | "文本 + 图像输出" | 在一次传播中自主产出交错文本和图像词元的推理 |
| 码本大小 | "K 个条目" | VQ-VAE 可量化到的离散向量数量；在压缩与保真之间权衡 |
| 分词器上限 | "重构极限" | 解码 VQ 词元可达到的最佳 PSNR；为模型的图像质量设界 |
