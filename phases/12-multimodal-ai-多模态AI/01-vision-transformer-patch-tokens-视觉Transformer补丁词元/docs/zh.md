# 视觉 Transformer 与补丁-词元基本单元

> 在涉及多模态之前，图像必须先变成 transformer 能够处理的词元序列。2020 年的 ViT 论文用 16x16 像素补丁、一次线性投影和一个位置嵌入回答了这个问题。五年之后，每个 2026 年的前沿模型（原生 2576px 的 Claude Opus 4.7、Gemini 3.1 Pro、Qwen3.5-Omni）仍然从这里起步——编码器从 ViT 换成 DINOv2 再换成 SigLIP 2，加入了 register 词元，位置方案变成了 2D-RoPE，但这个基本单元始终未变。本课从头到尾解读补丁-词元流水线，并用标准库 Python 把它搭建出来，让 Phase 12 的其余部分对“视觉词元”有一个具体的心智模型。

**类型：** 学习
**语言：** Python（标准库，补丁词元化器 + 几何计算器）
**前置要求：** Phase 7（Transformers）、Phase 4（计算机视觉）
**时长：** 约 120 分钟

## 学习目标

- 把一张 HxWx3 的图像转换成带有正确位置编码的补丁词元序列。
- 为给定（补丁大小、分辨率、隐藏维度、深度）的 ViT 计算序列长度、参数量和 FLOPs。
- 说出让 ViT 从 2020 年的研究走向 2026 年生产的三项升级：自监督预训练（DINO / MAE）、register 词元，以及原生分辨率打包。
- 为下游任务在 CLS 池化、均值池化和 register 词元之间做出选择。

## 问题

Transformer 处理的是向量序列。文本本身就是序列（字节或词元）。而图像是带有三个颜色通道的二维像素网格——并不是序列。如果你把每个像素展平，一张 224x224 的 RGB 图像会变成 150,528 个词元，而在这个长度上做自注意力根本行不通（在序列长度上是二次方复杂度）。

2020 年之前的方法是在前端拼接一个 CNN 特征提取器：ResNet 产出一个 7x7 的特征图，每个是 2048 维向量，把这 49 个词元喂给 transformer。这能用，但继承了 CNN 的偏置（平移等变性、局部感受野），也丢失了 transformer 对规模的胃口。

Dosovitskiy 等人（2020）提出了一个直白的问题：如果我们干脆跳过 CNN 会怎样？把图像切成固定大小的补丁（比如 16x16 像素），对每个补丁做线性投影得到一个向量，加上位置嵌入，然后把序列喂给一个普通的 transformer。在当时这是异端——没有卷积的视觉。只要有足够的数据（先是 JFT-300M，后来是 LAION），它就能在 ImageNet 上超过 ResNet，并且持续提升。

到了 2026 年，ViT 这一基本单元成了无可争议的基石。每个开放权重 VLM 的视觉塔都是它的某种后代（DINOv2、SigLIP 2、CLIP、EVA、InternViT）。问题不再是“我们该不该用补丁？”，而是“用多大的补丁、什么样的分辨率调度、什么样的预训练目标、什么样的位置编码”。

## 概念

### 补丁作为词元

给定一张形状为 `(H, W, 3)` 的图像 `x` 和补丁大小 `P`，你把图像切成一个 `(H/P) x (W/P)` 的非重叠补丁网格。每个补丁是一个 `P x P x 3` 的像素立方体。把每个立方体展平成一个 `3 P^2` 的向量。施加一个形状为 `(3 P^2, D)` 的共享线性投影 `W_E`，把每个补丁映射到模型的隐藏维度 `D`。

对于 ViT-B/16 的标准配置：
- 分辨率 224，补丁大小 16 → 网格 14x14 → 196 个补丁词元。
- 每个补丁是 `16 x 16 x 3 = 768` 个像素值，投影到 `D = 768`。
- 加上一个可学习的 `[CLS]` 词元 → 序列长度 197。

补丁投影在数学上与一个核大小为 `P`、步幅为 `P`、输出通道为 `D` 的二维卷积完全等价。这正是生产代码实际的实现方式——`nn.Conv2d(3, D, kernel_size=P, stride=P)`。“线性投影”的说法是概念性的；卷积核的说法是高效的。

### 位置嵌入

补丁本身没有固有顺序——transformer 把它们看成一个袋子。早期的 ViT 加入了一个可学习的一维位置嵌入（每个位置一个 768 维向量，共 197 个）。这能用，但把模型绑定在了训练分辨率上：推理时如果改变网格，你就必须对位置表做插值。

现代视觉骨干使用 2D-RoPE（Qwen2-VL 的 M-RoPE、SigLIP 2 的默认方案）或分解式二维位置。2D-RoPE 根据补丁的（行、列）索引旋转查询和键向量，从而让模型从旋转角度推断相对的二维位置。不需要位置表。模型在推理时能处理任意网格大小。

### CLS 词元、池化输出和 register 词元

图像级表示是什么？三种选择并存：

1. `[CLS]` 词元。在补丁序列前面前置一个可学习向量。经过所有 transformer 块之后，CLS 词元的隐藏状态就是图像表示。继承自 BERT。原始 ViT、CLIP 使用。
2. 均值池化。对补丁词元的输出隐藏状态取平均。SigLIP、DINOv2 和大多数现代 VLM 使用。
3. Register 词元。Darcet 等人（2023）观察到，未带显式汇聚词元训练的 ViT 会发展出高范数的“伪影”补丁，劫持自注意力。加入 4–16 个可学习的 register 词元能吸收这部分负载，并提升稠密预测的质量（分割、深度）。DINOv2 和 SigLIP 2 都自带 register。

这个选择对下游任务很重要。CLS 对分类来说够用。对于把补丁词元喂给 LLM 的 VLM，你会完全跳过池化——每个补丁都成为一个 LLM 输入词元。Register 在交接前会被丢弃（它们是脚手架，不是内容）。

### 预训练：监督、对比、掩码、自蒸馏

2020 年的 ViT 是在 JFT-300M 上用监督分类预训练的。很快就被以下方法取代：

- CLIP（2021）：在 4 亿对图文上做对比学习。见第 12.02 课。
- MAE（2021，He 等人）：掩盖 75% 的补丁，重建像素。自监督，适用于纯图像。
- DINO（2021）/ DINOv2（2023）：用学生-教师做自蒸馏，无标签、无字幕。2023 年的 DINOv2 ViT-g/14 是最强的纯视觉骨干，也是“稠密特征”用例的默认选择。
- SigLIP / SigLIP 2（2023、2025）：用 sigmoid 损失的 CLIP，并用 NaFlex 支持原生宽高比。是 2026 年开放 VLM（Qwen、Idefics2、LLaVA-OneVision）中占主导地位的视觉塔。

你对预训练的选择决定了骨干擅长什么：CLIP/SigLIP 用于与文本做语义匹配，DINOv2 用于稠密视觉特征，MAE 作为下游微调的起点。

### 缩放定律

ViT 缩放（Zhai 等人 2022）确立了 ViT 的质量在模型规模、数据规模和算力上服从可预测的定律。在固定算力下：
- 更大的模型 + 更多的数据 → 更好的质量。
- 补丁大小是序列长度与保真度之间的杠杆。补丁 14（DINOv2/SigLIP SO400m 的典型选择）比补丁 16 给出每张图更多的词元；对 OCR 和稠密任务更好，但速度更差。
- 分辨率是另一根大杠杆。从 224 升到 384 再到 512 几乎总能带来提升，代价是 FLOPs 的二次方增长。

ViT-g/14（10 亿参数，补丁 14，分辨率 224 → 256 个词元）和 SigLIP SO400m/14（4 亿参数，补丁 14）是 2026 年开放 VLM 的两台主力编码器。

### ViT 的参数量

完整计算见 `code/main.py`。对于分辨率 224 的 ViT-B/16：

```
patch_embed = 3 * 16 * 16 * 768 + 768  =  591k
cls + pos    = 768 + 197 * 768          =  152k
block        = 4 * 768^2 (QKVO) + 2 * 4 * 768^2 (MLP) + 2 * 2*768 (LN)
             = 12 * 768^2 + 3k          =  7.1M
12 blocks    = 85M
final LN    = 1.5k
total       ≈ 86M
```

在加载检查点之前先这样估算每个 ViT 的量级。骨干大小决定了任何下游 VLM 的 VRAM 下限。

### 2026 年生产配置

2026 年大多数开放 VLM 自带的编码器是原生分辨率（NaFlex）下的 SigLIP 2 SO400m/14。它具有：
- 4 亿参数。
- 补丁大小 14，默认分辨率 384 → 每张图 729 个补丁词元。
- 图像级任务用均值池化；VQA 时全部 729 个补丁流入 LLM。
- 4 个 register 词元，在交接给 LLM 之前丢弃。
- 带图像级缩放的 2D-RoPE，支持原生宽高比。

## 用起来

`code/main.py` 是一个补丁词元化器和几何计算器。它接收（图像 H、W、补丁 P、隐藏维度 D、深度 L）并报告：

- 补丁化后的网格形状和序列长度。
- 一张合成的 8x8 像素玩具图像的词元序列（走一遍展平 + 投影的路径）。
- 按补丁嵌入、位置嵌入、transformer 块和头部分解的参数量。
- 在目标分辨率下每次前向传播的 FLOPs。
- 跨 ViT-B/16 @ 224、ViT-L/14 @ 336、DINOv2 ViT-g/14 @ 224、SigLIP SO400m/14 @ 384 的对比表。

运行它。把参数量与公开数字对上。摆弄补丁大小和分辨率，体会词元数量的代价。

## 交付

本课产出 `outputs/skill-patch-geometry-reader.md`。给定一个 ViT 配置（补丁大小、分辨率、隐藏维度、深度），它会给出带理由的词元数、参数量和 VRAM 估算。每当你为 VLM 挑选视觉骨干时就使用这个技能——它能避免“词元爆炸、我的 LLM 上下文被塞满”这类意外。

## 练习

1. 计算 Qwen2.5-VL 在原生 1280x720 输入、补丁大小 14 下的补丁-词元序列长度。它与只用 CLS 的表示相比如何？

2. 一帧 1080p（1920x1080）在补丁 14 下产生多少词元？在一段 5 分钟视频上以 30 FPS 计算，总共有多少视觉词元？哪种代价节省最多：池化、帧采样还是词元合并？

3. 用纯 Python 实现对补丁词元的均值池化。验证对 DINOv2 输出的 196 个词元做均值池化的结果，与你向模型的 `forward` 请求一个池化嵌入时返回的结果是否一致。

4. 阅读《Vision Transformers Need Registers》（arXiv:2309.16588）的第 3 节。用两句话描述 register 吸收的是什么伪影，以及它为何对下游稠密预测重要。

5. 修改 `code/main.py` 以支持 patch-n'-pack：给定一组不同分辨率的图像，产出一个打包后的单一序列以及块对角注意力掩码。等你学到第 12.06 课时拿它来验证。

## 关键术语

| 术语 | 人们怎么说 | 它实际的含义 |
|------|----------------|------------------------|
| 补丁（Patch） | “16x16 像素方块” | 输入图像中固定大小的非重叠区域；成为一个词元 |
| 补丁嵌入（Patch embedding） | “线性投影” | 一个共享的学习矩阵（或步幅为 P 的 Conv2d），把展平的补丁像素映射为 D 维向量 |
| CLS 词元 | “类别词元” | 前置的可学习向量，其最终隐藏状态代表整张图像；在 2026 年是可选的 |
| Register 词元 | “汇聚词元” | 额外的可学习词元，吸收 ViT 在预训练中产生的高范数注意力伪影 |
| 位置嵌入（Position embedding） | “位置信息” | 让序列具有顺序感知能力的逐位置向量或旋转；2D-RoPE 是现代默认方案 |
| 网格（Grid） | “补丁网格” | 给定分辨率和补丁大小下，补丁的 (H/P) x (W/P) 二维数组 |
| NaFlex | “原生灵活分辨率” | SigLIP 2 的特性：单个模型无需重训练即可服务多种宽高比和分辨率 |
| 骨干（Backbone） | “视觉塔” | 预训练的图像编码器，其补丁-词元输出在 VLM 中喂给 LLM |
| 池化（Pooling） | “图像级摘要” | 把补丁词元变成一个向量的策略：CLS、均值、注意力池化或基于 register |
| 补丁 14 与 16 | “更细 vs 更粗的网格” | 补丁 14 每张图产生更多词元，对 OCR 保真度更好但更慢；补丁 16 是经典默认值 |
