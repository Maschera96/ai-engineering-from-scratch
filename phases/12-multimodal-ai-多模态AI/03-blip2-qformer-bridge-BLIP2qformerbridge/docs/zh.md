# 从 CLIP 到 BLIP-2 —— 作为模态桥梁的 Q-Former

> CLIP 能够对齐图像与文本，却无法生成图像描述、回答问题或进行对话。BLIP-2（Salesforce，2023）用一个小型可训练的桥梁解决了这个问题：32 个可学习的查询向量通过交叉注意力从冻结的 ViT 特征中取信息，然后直接接入冻结 LLM 的输入流。188M 参数的桥梁将一个 11B 的 LLM 与一个 ViT-g/14 连接起来。截至 2026 年的每一个基于适配器的 VLM —— MiniGPT-4、InstructBLIP、LLaVA 的同类们 —— 都是它的后代。本课解读 Q-Former 的架构，讲解它的两阶段训练，并构建一个玩具版本，将视觉 token 喂给一个冻结的文本解码器。

**类型：** 构建
**语言：** Python（标准库，交叉注意力 + 可学习查询演示）
**前置要求：** Phase 12 · 02（CLIP），Phase 7（Transformers）
**时长：** 约 180 分钟

## 学习目标

- 解释为什么在冻结的视觉编码器与冻结的 LLM 之间放一个可训练的瓶颈，在成本和稳定性上优于端到端微调。
- 实现一个交叉注意力块，让一组固定的可学习查询去关注外部图像特征。
- 走通 BLIP-2 的两阶段预训练：表示阶段（ITC + ITM + ITG），随后是生成阶段（在冻结解码器上使用 LM 损失）。
- 将 Q-Former 与 LLaVA 中使用的更简单的 MLP 投影器进行对比，并论证各自在什么情况下更优。

## 问题

你有一个冻结的 ViT，它为每张图像产生 256 个维度为 1408 的 patch token。你还有一个冻结的 7B LLM，它期望维度为 4096 的 token 嵌入。最直接的桥梁 —— 一个从 1408 到 4096 的线性层 —— 是可行的，但把全部 256 个 patch token 喂入 LLM 的上下文，每张图像要额外耗费 256 个 token。对于一个 32 张图像的批次，仅视觉模态就消耗了 8192 个 token。

BLIP-2 的问题是：你能否把 256-token 的图像表示压缩成少得多的 token（比如 32 个），同时保留足够的信息让 LLM 去描述、回答问题并对图像进行推理？以及，你能否在不触碰冻结骨干网络的前提下训练这个桥梁，把训练成本控制在仅桥梁参数的范围内？

答案是：一个 Q-Former。32 个可学习的“查询”向量交叉关注 ViT 的 patch token，产生一个 32-token 的视觉摘要供 LLM 消费。总共 188M 参数。在接触 LLM 之前，它先用对比、匹配和生成目标进行训练。

## 概念

### 可学习的查询

Q-Former 的核心技巧是：与其让 LLM 的文本 token 去关注图像 patch，不如引入一组新的 32 个可学习查询向量 `Q`，让*它们*去关注图像 patch。这些查询是模型的参数 —— 它们在训练期间被学习，并且对每张图像都使用同样的这 32 个查询。

在交叉注意力之后，每个查询持有图像的一份压缩摘要 —— “描述主要物体”“描述背景”“数物体的数量”等等。这些查询并不会真的专门对应到语义标签上；它们学习的是任何能让下游损失下降的编码方式。

### 架构

Q-Former 是一个小型 transformer（12 层，约 100M 参数），有两条路径：

1. 查询路径：32 个查询向量流经自注意力（在它们彼此之间），然后对冻结的 ViT patch token 做交叉注意力，再经过 FFN。
2. 文本路径：一个类似 BERT 的文本编码器与查询路径共享自注意力和 FFN 权重。文本路径上的交叉注意力被禁用。

训练时两条路径都运行。查询与文本通过共享的自注意力相互作用，这意味着对于需要的任务（ITM、ITG），查询可以基于文本进行条件化。在为 VLM 交接进行推理时，只有查询会流经，产出 32 个视觉 token。

### 两阶段训练

BLIP-2 分两个阶段进行预训练：

阶段 1：表示学习（无 LLM）。三个损失：
- ITC（图文对比）：在池化后的查询 token 与文本 CLS token 之间进行 CLIP 风格的对比。
- ITM（图文匹配）：二分类器 —— 这个图文对是否匹配？采用难负样本挖掘。
- ITG（基于图像的文本生成）：在文本上使用因果 LM 头，以查询为条件。迫使查询编码出可生成文本的内容。

只有 Q-Former 训练。ViT 被冻结。不涉及 LLM。

阶段 2：生成学习。接上一个冻结的 LLM（OPT-2.7B 或 Flan-T5-XL 等）。通过一个小型线性层把 32 个查询输出投影到 LLM 的嵌入维度。将它们前置到文本提示。仅在拼接的 提示 + 图像 + 描述 序列上，用 LM 损失训练这个线性投影和 Q-Former。

阶段 2 之后，Q-Former + 投影就是完整的视觉适配器。推理时：图像 → ViT → Q-Former → 线性投影 → 前置到文本 → 冻结的 LLM 产出输出。

### 参数经济学

BLIP-2 = ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（188M，训练）= 总共 8B，训练 188M。仅 Q-Former 就约占整个栈参数的 2.4%。训练成本反映了这一点：在少数几块 A100 上数天，相比端到端的数周。

质量：BLIP-2 在零样本 VQA 上达到或超过 Flamingo-80B，同时小 50 倍。这个桥梁奏效了。

### InstructBLIP 与指令感知的 Q-Former

InstructBLIP（2023）为 Q-Former 增加了一个额外输入：指令文本本身。在做交叉注意力时，查询现在同时能访问图像 patch 和指令。查询可以按指令专门化（“数汽车的数量”“描述氛围”），而不是学习一个单一固定的摘要。在留出任务上获得了基准提升。

### MiniGPT-4 与仅投影器的方法

MiniGPT-4 保留了 Q-Former，但只训练输出线性投影，同时冻结其余一切。便宜，但代价是质量 —— 那些查询是 BLIP-2 的，不是你的。适合快速迭代，但不是最优架构。

### 为什么 LLaVA 走了更简单的路

LLaVA（2023，课程 12.05）用一个普通的 2 层 MLP 替换了 Q-Former，它把每个 ViT patch token 投影到 LLM 空间 —— 对一个 24x24 的网格来说是每张图像 576 个 token，全部喂给 LLM。压缩更差，但让 LLM 能直接关注原始 patch。当时这有争议；到 2023 年底它占据了主导地位，因为视觉指令数据（LLaVA-Instruct-150k）证明了 MLP 可以被训练得保留足够的信号。权衡是：LLaVA 的上下文填得更快，但它能自然地扩展到多图像和视频。

到 2026 年，这个领域分化了：在 token 预算重要的地方（长视频、多图像）Q-Former 得以存续；在以每 token 的原始质量为优先的地方，MLP 投影器占据主导。

### 门控交叉注意力：祖先 Flamingo

Flamingo（课程 12.04）早于 BLIP-2，使用了同样的交叉注意力思想，但应用在每一个冻结的 LLM 层上，而不是作为单一桥梁。BLIP-2 表明你可以只压缩到输入层，仍然有效。Gemini 和 Idefics 结合了两者：交错的输入 token 加上可选的门控交叉注意力，用于上下文中的少样本。

### 2026 年的后代

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4，以及出于 token 预算原因的大多数视频-语言模型。
- Perceiver resampler：Flamingo 的变体（课程 12.04）；Idefics 系列、Eagle、OmniMAE。
- MLP 投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化（Attention pool）：VILA、PaliGemma。

这四种都成立。决定性的问题是你受限于 token 预算，还是每 token 的质量。

## 上手用

`code/main.py` 用标准库构建一个 Q-Former 风格的交叉注意力：

1. 模拟 256 个图像 patch token（维度 128）。
2. 实例化 32 个可学习查询（维度 128）。
3. 运行缩放点积交叉注意力（Q 来自查询，K/V 来自 patch）。
4. 通过一个线性层投影到 LLM 维度（512）。
5. 输出 32 个 LLM 就绪的视觉 token。

所有数学都用纯 Python（在向量上做嵌套循环）。是玩具，但形状正确。注意力权重矩阵会被打印出来，这样你就能看到每个查询从哪些 patch 中取了信息。

## 练习

1. 用 PyTorch 实现交叉注意力块。验证在 32 个查询和 256 个键/值的情况下，注意力权重矩阵是 32 x 256，且每一行在 softmax 之后求和为 1。

2. 在 BLIP-2 阶段 1，Q-Former 同时运行三个损失：ITC、ITM、ITG。用伪代码写出每个的前向签名。哪一个需要文本编码器路径处于激活状态？

3. 比较参数量：Q-Former（12 层，768 隐藏维）vs 一个 2 层 MLP 投影器（1408 → 4096，两层）。在多大的 LLM 规模下，188M 的 Q-Former 成本能在训练效率上回本？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| Q-Former | “查询 transformer” | 带有 32 个可学习查询向量的小型 transformer，这些向量交叉关注冻结的 ViT 特征 |
| 可学习查询 | “视觉的软提示” | 一组固定的参数，充当交叉注意力的查询侧；按模型学习，在所有输入间共享 |
| 交叉注意力 | “Q 来自这里，K/V 来自那里” | 查询、键、值来自不同来源的注意力；即查询从 ViT patch 中取信息的方式 |
| ITC | “图文对比” | 应用于 Q-Former 池化查询 vs 文本 CLS 的 CLIP 风格损失 |
| ITM | “图文匹配” | 在难负样本挖掘的样本对上的二分类器；迫使查询区分细粒度的不匹配 |
| ITG | “基于图像的文本生成” | 因果 LM 损失，文本以查询为条件生成；迫使查询编码可解码为文本的内容 |
| 两阶段预训练 | “先表示后生成” | 阶段 1 单独训练 Q-Former（ITC/ITM/ITG）；阶段 2 接上冻结的 LLM，只训练投影 + Q-Former |
| 冻结骨干 | “不要微调” | 视觉编码器和 LLM 权重是固定的；只有桥梁训练 |
| 投影头 | “线性到 LLM 维度” | 最后的线性层，把 Q-Former 输出映射到 LLM 的嵌入维度 |
| Perceiver resampler | “Flamingo 的版本” | 类似的可学习查询交叉注意力，Flamingo 在每一层使用它，而不是作为单一桥梁 |
