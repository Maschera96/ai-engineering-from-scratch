# MIO 与任意到任意流式多模态模型

> GPT-4o 推出了一款大多数开源模型无法复现的产品：一个能听到语音、看到视频并实时语音回应的智能体。到 2024 年底，开源生态的答案是 MIO（Wang 等人，2024 年 9 月）。MIO 将文本、图像、语音和音乐 token 化，在交错序列上训练单一因果 transformer，并实现任意模态到任意模态的生成。AnyGPT（Zhan 等人，2024 年 2 月）是概念验证；MIO 是规模化版本；Unified-IO 2（Allen AI，2023 年 12 月）则是带有视觉 + 动作接地的同类作品。本课解读任意到任意的范式——四个 tokenizer、一个 transformer、对流式友好的解码。

**类型：** 学习
**语言：** Python（标准库，四模态 token 分配器 + 流式解码循环）
**前置要求：** 第 12 阶段 · 11（Chameleon）、第 6 阶段（语音与音频）
**时长：** 约 120 分钟

## 学习目标

- 设计一套共享词表，在不冲突的情况下容纳文本、图像、语音和音乐 token。
- 在压缩 + 重建权衡上比较 SEED-Tokenizer（图像）和 SpeechTokenizer 残差 VQ（语音）。
- 解释构建任意到任意生成的四阶段课程。
- 说出三种开源任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题

统一多模态模型容易声称，却难以大规模构建。截至 2024 年，大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每一次跳转都会损失信息、增加延迟并使训练复杂化。GPT-4o 的演示视频展示了一种单一模型的替代方案，响应不到一秒；开源系统则落后数月。

工程上的挑战：

- 每种模态都必须有 tokenizer，压缩到足以无损重建，并以 transformer 能消费的速率产出 token。
- 单一词表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。最少四万多个条目。
- 训练数据必须覆盖每一对输入-输出（text→image、image→speech、speech→image 等），否则模型必须自行组合。
- 推理必须以足够快的速度流式输出 token 以满足对话延迟（首个音频字节时间 <500ms）。

## 概念

### 四种模态的四个 tokenizer

MIO 的 tokenizer 栈：

- 文本：标准 BPE，词表约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 个条目，每张图像 32x32 个 token。
- 语音：SpeechTokenizer 残差 VQ（2023）——将 16kHz 波形编码为 8 个层级码本；第一层是粗粒度内容，后续层加入韵律和说话人身份。
- 音乐：类似的残差 VQ（Meta 的 MusicGen / Encodec 系列），4-8 个码本。

每种模态产出整数 token。这些 token 在共享词表中获得不相交的 ID 区间：

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

总计：约 48k 词表。输入嵌入和输出投影覆盖全部范围。

### 流式解码

语音生成使用残差 VQ。transformer 预测基础（第 0 层）语音 token；一个并行解码的残差量化器预测后续层级。每个第 0 层 token 在 16kHz 下约对应 50ms 音频。

流式范式：

1. 用户对麦克风说话；实时音频 tokenizer 每 50ms 产出语音 token。
2. MIO 在 token 到达时即时消费（prompt 预填充 + 增量前向）。
3. 输出 token 随生成边流出；一个并行语音解码器以约 50-150ms 延迟将其转为音频采样。
4. 首个音频字节时间：MIO 论文中约 300-500ms，逼近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音 LLM 设计。Moshi 尤其在单张 GPU 上实现了 160ms 的往返延迟。

### 四阶段课程

MIO 的训练课程：

1. 第 1 阶段——对齐。大规模模态对语料：text-image、text-speech、text-music。每一对使用自己的 token 词表段。训练共享词表。
2. 第 2 阶段——交错。多模态交错文档（带图像 + 视频的博客、带转录稿的播客等）。训练跨模态上下文。
3. 第 3 阶段——语音增强。额外音频数据在不损失文本能力的前提下提升语音质量。
4. 第 4 阶段——SFT。跨模态的指令微调：VQA、字幕生成、旁白、语音到语音对话。

缺少某个阶段会削弱特定能力：跳过第 2 阶段模型会失去跨模态上下文；跳过第 3 阶段语音会很差。

### 视觉思维链

MIO 引入了视觉思维链：模型将中间图像 token 作为推理步骤输出。对于"猫是否在爬树？"，模型会：

1. 输出 `<image>` token，渲染场景（来自输入图像或草图）。
2. 输出分析该草图的文本。
3. 输出最终答案。

渲染出的中间图像充当草稿纸。在空间推理任务上基准成绩有所提升。这一思路对应于文本推理的思维链。

### 任意到任意的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），设计类似。
- Unified-IO 2（arXiv:2312.17172）：增加视觉动作输出、深度、法线。任务多样性更高，规模更小。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态专属扩散解码器。不是单一模型方案。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜空间实现任意到任意。

MIO 最接近纯 token 的任意到任意。AnyGPT 是其概念上的先驱。

### 延迟预算

对于对话产品，每个组件的延迟都很重要：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史）：8B 模型上约 100ms。
- 首个输出 token：约 50ms。
- 并行残差 VQ + 语音解码器：约 100-150ms。

首个音频字节时间总计：最少约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。根据公开基准，MIO/AnyGPT 在 400-600ms 区间。

### 为何任意到任意依然困难

即便到了 2026 年，开源任意到任意模型在两个维度上仍落后于闭源：

- 语音质量。残差 VQ tokenizer 是有损的；与 ElevenLabs 级别的语音相比，对话语音听起来很机械。
- 跨模态推理。让模型"唱出你所看到的"仍然比纯视觉任务更频繁地失败。

这些都是开放的研究问题。Qwen3-Omni（第 12.20 课）是 2025 年最先进的开源尝试。

## 动手用

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出哪些组件会耗费时间。

2. SpeechTokenizer 残差 VQ 使用 8 个码本。提出为什么并行解码残差层级是必要的（相对于顺序解码），以及它带来什么延迟节省。

3. 你的词表有 32k 文本 + 4k 图像 + 4k 语音。再加上 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数代价是多少？

4. 视觉思维链会输出一张中间图像。哪类问题会从中受益？哪类问题会因额外 token 而受损？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技术，并与 MIO 的视觉思维链比较。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| 任意到任意 | "多模态进/出" | 一个单一模型，能在任意方向上接受并产出文本、图像、语音和音乐 |
| 残差 VQ | "语音 tokenizer 栈" | 多码本 tokenization，每一层增加信息；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像编码" | MIO 使用的离散图像 tokenizer，码本含 4096 个条目 |
| 视觉思维链 | "视觉草稿纸" | 模型在最终答案之前生成一张中间图像作为推理步骤 |
| 首个音频字节时间 | "TTFAB" | 从用户语音到首个音频输出的延迟；对话感需 <500ms |
| 四阶段课程 | "训练配方" | 对齐 -> 交错 -> 语音增强 -> SFT，按此顺序 |
