# LLaVA-OneVision：单图、多图、视频统一于一个模型

> 在 LLaVA-OneVision（Li 等人，2024 年 8 月）之前，开放 VLM 世界有着各自独立的谱系：LLaVA-1.5 处理单图，Mantis 和 VILA 等多图模型处理多图，Video-LLaVA 和 Video-LLaMA 等视频模型处理视频。每个模型都在自己的基准上夺冠，却在其他场景上落败。LLaVA-OneVision 主张：单一课程就能训练出一个在三种场景上都占优的模型，并且涌现出的任务迁移效应（单图技能迁移到视频、多图推理迁移到单图）会胜过各路专家的总和。其配方看似简单：一个在各场景间保持恒定的视觉 token 预算，加上一条从单图到 OneVision（多图）再到视频的明确课程。本课将解读这个预算、这条课程，以及涌现行为。

**类型：** 构建
**语言：** Python（标准库，token 预算求解器 + 课程规划器）
**前置：** Phase 12 · 05（LLaVA），Phase 12 · 06（任意分辨率）
**时长：** ~180 分钟

## 学习目标

- 设计一个在单图、多图、视频输入间保持恒定的视觉 token 预算。
- 编排一条训练课程，将技能从单图迁移到视频且不发生灾难性遗忘。
- 解释为什么在课程编排得当时，单一模型能在相同参数量下胜过专家模型。
- 说出 LLaVA-OneVision 报告的三种涌现能力：多摄像头推理、set-of-mark 提示、iPhone 截图智能体。

## 问题

图像、多图和视频各自以不同的方式给模型施加压力。

单图需要高分辨率 token（AnyRes，约 2880 个视觉 token）以捕捉 OCR 和细节。每个样本的预算：一张图像，2880 个 token。

多图需要若干张中等分辨率图像（每张约 576 个 token），以便跨图像推理能装进上下文。每个样本的预算：4-8 张图像，每张 576 个，共 2300-4600 个 token。

视频需要许多低分辨率的帧（池化后每帧约 196 个 token）以捕捉时间动态。每个样本的预算：8-32 帧，每帧 196 个，共 1600-6200 个 token。

如果你训练独立的模型，你只挑选一个预算。如果你训练一个模型，你需要让预算在各场景间合理伸缩，又不撑爆上下文。

在 OneVision 之前，默认答案是"训练一个场景，忽略其他场景"。Video-LLaVA 通过额外的训练阶段把视频改装到图像模型上。LLaVA-NeXT 通过分块（tiling）增加了多图支持。没有一个能干净地处理全部三种场景。

## 概念

### OneVision token 预算

LLaVA-OneVision 选定了一个约 3000-4000 个 token 每样本的统一视觉 token 预算，按场景以不同方式分配：

- 单图：AnyRes-9（3x3 块 + 缩略图），每块在 384 分辨率下有 729 个 patch，激进的 2x2 双线性池化 → 每块 182 个。总计：9 * 182 + 182 = 1820 个 token。或 AnyRes-4 每块 729 个 = 2916 + 729。
- 多图：每张图像在中等分辨率（384，不分块）下有 729 个 token，不做池化。预算 6 张图像 → 4374 个 token。
- 视频：32 帧在 384 分辨率下经激进的 3x3 双线性池化 → 每帧 81 个 token。总计：32 * 81 = 2592 个 token。

这种分配使总 token 数大致保持恒定。LLM 永远不会看到撑爆其上下文的批次。编码器为每个场景产出不同的几何形状，但 LLM 消费的是相同的预算。

### 三阶段课程

LLaVA-OneVision 分三个阶段训练：

1. 单图 SFT（SI 阶段）。所有数据都是单图加文本。在高分辨率 AnyRes 输入上训练。这教会感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加 OneVision 专用的单图数据。
2. OneVision SFT（OV 阶段）。混合单图 + 多图 + 视频（均匀采样的帧）。在统一 token 预算上训练。这教会模型处理异构的批次形状。不重置权重——从 SI 阶段继续。
3. 任务迁移（TT 阶段）。继续使用目标任务混合，通常根据产品偏重多图或视频。可选的部署微调。

关键：课程顺序很重要。视频优先或多图优先训练得到的图像性能比单图优先更差，即使使用相同的数据。论文对此做了明确的消融实验。

### 为什么课程有效

单图训练构建感知基础。Patch token 携带细粒度的视觉特征；LLM 学会将它们与文本整合。多图和视频引入了结构性挑战（哪张图是哪张、先发生了什么），缺乏强大的感知基础就难以学会。

如果你从头一起训练所有场景，模型会欠拟合感知（每批次单图数据有限）并过拟合结构（大量多图/视频数据）。结果：一个能遵循跨图像推理模式但视觉上很浅的模型。

课程排序让你从 SI 阶段获得感知强度，再从 OV 阶段获得组合/时间推理能力，二者皆不丢失。

### 涌现的跨场景技能

LLaVA-OneVision 论文报告了三种涌现能力：

1. 多摄像头推理。在多图 + 视频上分别训练；推理时，被要求对多摄像头驾驶场景进行推理。尽管训练中从未见过那种确切格式，模型仍能正确整合各视图。
2. Set-of-mark 提示。用户用带编号的标记标注图像中的对象；模型推理"标记 3 相对于标记 7 在做什么"。既未在标记上训练，也未在标注上训练；从空间接地 + 多图引用的组合中学会。
3. iPhone 截图智能体。用户提供一张 iPhone 屏幕截图并要求规划下一次点击。在 UI 截图、用户工作流视频和多图前后对比上训练。泛化到了智能体用例。

这些都不是被训练的任务；它们从课程的组合结构中涌现。

### 视觉 token 池化

token 预算需要池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 个 patch 变为 12x12 = 144（2x 因子）或 8x8 = 64（3x 因子）。池化在 patch 网格空间中完成，而非 token 空间，以保留局部性。

每个场景的池化因子选择本身就是一个超参数。池化越少 = token 越多 = 表示越丰富。池化越多 = token 越少 = 能装下越多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续工作（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上"完全开放"。在部分基准上追平了与专有模型的差距，并将配方民主化。相同课程，更多数据，更好的基座 LLM。架构无变化。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（第 12.09 课）做出了不同的选择。它使用 M-RoPE 和动态 FPS 而非固定池化。其预算随输入伸缩——1 分钟视频比 5 秒视频使用更多 token。LLaVA-OneVision 固定预算并伸缩池化。两者都有效；它们以可配置性换取可预测性。

## 动手用

`code/main.py` 是一个面向 OneVision 风格 VLM 的课程与预算规划器。给定每样本的 token 预算和目标场景混合（比如 40% 单图、30% 多图、30% 视频），它会：

- 为每个场景分配分辨率、池化因子和帧数。
- 检查每个场景是否都能装进共享预算。
- 报告预期 token 数、LLM FLOPs，以及哪些场景 token 不足。
- 打印逐阶段的训练计划表。

用它来规划一次 OneVision 微调，或对 VLM 部署的每请求成本做合理性检查。

## 交付

本课产出 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每样本预算，它会输出 AnyRes 因子、每帧池化、视频帧数和课程阶段权重。在你训练或微调统一场景 VLM 时都可使用它。

## 练习

1. 你的产品支持 80% 单图、10% 多图（2-4 张图像）、10% 视频（8-16 帧）。设计 token 预算。你会把不做重度多图所省下的额外预算放到哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出一种课程可能解锁、但论文未报告的第四种涌现技能。

3. 调换课程顺序——先训练多图，再单图，再视频。预测哪些基准会退化以及为什么。

4. 论文报告的视频基准只在每样本 8 帧上训练。这能泛化到推理时的 30 秒视频吗？最先崩溃的是什么——token 预算还是时间推理？

5. 将 24x24 个 patch 双线性池化到 12x12 是每维 4x 的缩减。用标准库 Python 实现该池化，并验证每个 2x2 块的均值与双线性输出相匹配。

## 关键术语

| 术语 | 人们怎么说 | 它实际的含义 |
|------|-----------------|------------------------|
| OneVision 场景 | "单图、多图或视频" | 统一 VLM 处理的三种输入形状之一；预算在各场景间保持恒定 |
| Token 预算 | "每样本多少个 token" | LLM 在每个训练/推理样本中看到的视觉 token 总数，通常 3000-4000 |
| 课程 | "训练顺序" | 为涌现迁移而选择的阶段排序（单图 → 多图 → 视频） |
| 双线性池化 | "token 收缩" | 对 patch 网格（2D）应用双线性插值，在保留局部性的同时减少 token 数 |
| 涌现技能 | "没训练，照样能用" | 由于课程组合，在推理时出现的、无匹配训练数据的能力 |
| AnyRes-k | "k 块配置" | k 个固定分辨率的子块加一张缩略图，典型 k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 在单图上学到的技能，经由共享骨干网络应用于视频（反之亦然） |
