# InternVL3：原生多模态预训练

> 在 InternVL3 之前，每个开源 VLM 都遵循同样的三步配方：拿一个在数万亿文本 token 上训练的文本 LLM，外接一个视觉编码器，再对接缝处做微调。这样做行得通，但会留下对齐债务——文本 LLM 已经把全部预训练预算花在了纯文本上，并不会原生地理解视觉 token。当你事后添加视觉时，LLM 必须重新学习如何把视觉输入与它的文本推理关联起来，同时还不能遗忘文本。InternVL3（Zhu 等人，2025 年 4 月）摒弃了这种事后处理（post-hoc）的做法：一次预训练，文本与多模态从第一步起就交错（interleaved）在一起。结果是在 78B 参数的开源规模上，在 MMMU-Pro 上追平了 Gemini 2.5 Pro。本课将解读支持原生预训练的论据，以及当你这样做时会改变什么。

**类型：** 学习
**语言：** Python（标准库，训练语料混合器）
**前置：** Phase 12 · 05、Phase 12 · 07（配方）
**时间：** ~120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会累积对齐债务，并列举三个可测量的症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料混合，以及文本 : 交错 : caption 的比例为什么重要。
- 将 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE 进行比较。
- 说出 Visual Resolution Router（ViR）和 Decoupled Vision-Language（DvD）这两个部署优化。

## 问题

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——都是拿一个已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）再添加视觉。训练阶段通常如下：

1. 冻结的 LLM + 冻结的视觉编码器 + 可训练的投影器（projector），在 caption 配对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选的任务特定微调。

对齐债务表现出三个症状：

- 灾难性遗忘。事后 VLM 会遗忘纯文本技能。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本智能体出现退化。
- 答案漂移。同一个视觉问题用不同措辞会得到不同答案。视觉编码器与 LLM 的连接绑定，比 LLM 自己的 token 之间的绑定更弱。
- 视觉-文本不一致。VLM 可以正确描述一张图像，然后又给出一个与自己描述矛盾的回答。视觉 token 不像文本那样以相同方式参与 LLM 的内部一致性检查。

这些症状有充分记录。MM1.5 第 4 节对它们做了量化。LLaVA-OneVision 的消融实验也有所暗示。原生预训练就是答案。

## 概念

### 原生多模态预训练

InternVL3 在一个从第一步起就是原生多模态的语料上从头训练。混合比例为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 配对图像-caption 数据
- 5% 视频-文本数据

视觉 token、文本 token 以及跨模态交互，从第一个梯度步起就全部参与同一个损失。没有对齐预训练，没有投影器冻结阶段，也没有需要恢复的灾难性遗忘。

基础模型的训练是单阶段的。之后会进行指令微调，但基础模型已经把视觉 token 当作一等公民来理解。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用具有固定轴分配的 M-RoPE。InternVL3 引入了 V2PE：位置编码按模态类型（文本、图像、视频）变化，并带有可学习的缩放。具体而言：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享相同的 RoPE 频率基底，但每个频带的隐藏维度分配是一个可学习参数，而非固定划分。这带来了在预训练期间权衡时间频率与空间频率分辨率的自由度。

V2PE 的消融结论：在相同算力下，在视频基准上比 M-RoPE 高 1-2 分。算不上革命，但更干净。

### Visual Resolution Router（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张只有一个低细节物体的照片，在以 1280px 原生分辨率编码时会浪费 token。ViR 是一个小型分类器，在编码之前预测回答问题所需的最低分辨率。

路由分三档：低分辨率（256 token）、中（576）、高（2048+）。在生产流量中，60% 的查询用低或中分辨率就够了。净效果：在同等质量下吞吐提升 2-3 倍。

### Decoupled Vision-Language 部署（DvD）

当你部署一个大型 VLM 时，视觉编码器每张图像只运行一次，但 LLM 会为每个输出 token 自回归运行。这两个组件有不同的瓶颈（视觉 = 卷积 + 注意力的 GPU 显存带宽；LLM = KV 缓存）。DvD 将它们拆分到不同的 GPU 上，并在两者之间进行流式传输。

对于一个 8B + 400M 编码器的模型，DvD 相比同机部署大致能把单节点吞吐翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准结论：在 78B 参数下，追平 Gemini 2.5 Pro 的 MMMU-Pro。在 38B 下，追平 GPT-4o。在 8B 下，领跑开源 8B 排行榜。全部基于单阶段预训练 + 指令微调的配方。

对齐债务假设是可测量的：在每单位视觉基准增益下，InternVL3-8B 损失的文本基准分数（MMLU、GSM8K）比 Qwen2.5-VL-7B 更少。这个模型更像一个通才，因为训练是一整块，而非两块。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）扩展了这套配方。同样的原生预训练方法，更多数据，更多参数。MMMU 的提升是渐进式的。

InternVL-U（2026）加入了统一生成——通过在同一主干上叠加 MMDiT 头实现图像输出。其中的 "U" 代表 "Understanding + generation"（理解 + 生成），追逐 Transfusion 风格的统一模型（第 12.13 课）。同一个原生预训练主干同时支持理解头和生成头。

### 原生预训练的取舍

原生预训练不是免费的：

- 算力。从头训练一个新 VLM 的成本与训练一个文本 LLM 相同——数百万 GPU 小时。事后适配复用现有 LLM 权重，节省了大部分成本。
- 数据。大规模的交错图文语料很稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。仅文本就有 15T token。多模态预训练数据的稀缺是一个硬约束。
- 基础 LLM 复用。原生预训练放弃了之后换入一个新 LLM 的选项。事后做法让你只需重训适配器，就能把 Llama-3.1 换成 Llama-4。

InternVL3 押的注是：对齐债务比复用损失更糟。基准支持这一论断。而生产成本则让未来的实验室无法低成本复制。事后 VLM 会继续存在，因为对大多数项目而言它们仍然更便宜。

## 动手用

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器。它会：

- 接收一个目标语料混合（%text、%interleaved、%caption、%video），并计算每个模态的预期步数。
- 在一批查询上模拟 ViR 路由（分布：50% 低细节、30% 中、20% 高细节），并报告平均 token 数。
- 在给定编码器与 LLM FLOPs 的情况下报告 DvD 吞吐估计。
- 并排打印事后 vs 原生预训练在参数、算力、数据和预期对齐债务症状上的对比。

## 交付

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定一个拟定的 VLM 训练计划，它会审计应该走原生还是事后路线，标记对齐债务风险，并推荐一个语料混合。当你在为一个新的开源 VLM 项目做规模评估、需要选择训练策略时使用它。

## 练习

1. 估算 InternVL3-8B（原生预训练）与 LLaVA-OneVision-7B（事后）之间的算力差异。GPU 小时的比例大约是多少？是什么解释了这个差距？

2. InternVL3 报告的比例是 40% 文本 / 35% 交错 / 20% caption / 5% 视频。如果你的目标任务偏重视频，提出一个新比例，并论证为什么基础模型仍然需要大量的文本和 caption 数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练表现出最大退化的那个确切基准。这个退化付出了多大代价？

4. ViR 把 60% 的流量路由到低分辨率编码。它会误路由哪些类型的查询（在需要高分辨率时却发往低分辨率）？提出三种路由器失效模式。

5. DvD 把视觉和 LLM 拆分到不同的 GPU 上。在什么样的流量模式下，DvD 会损害吞吐而非帮助吞吐？

## 关键术语

| 术语 | 人们怎么说 | 它实际是什么 |
|------|-----------------|------------------------|
| 原生多模态预训练 | “从头一起训” | 文本 + 图像 + 视频 token 从第 1 步起就参与损失，而非之后外接 |
| 对齐债务 | “事后惩罚” | 把视觉外接到冻结 LLM 上所带来的、文本技能和答案一致性上可测量的退化 |
| V2PE | “可变视觉位置编码” | 按模态可学习的位置编码分配；InternVL3 的 M-RoPE 继任者 |
| ViR | “分辨率路由器” | 在编码前为每个查询挑选所需最低分辨率的小型分类器，节省推理 token |
| DvD | “解耦部署” | 视觉编码器在一块 GPU、LLM 在另一块，带流式交接；为大型 VLM 把吞吐翻倍 |
| InternVL-U | “统一理解 + 生成” | 2026 年的后续工作，为原生预训练主干加上图像生成头 |
| 交错语料 | “OBELICS / MMC4” | 文本与图像按自然阅读顺序排布的文档；原生预训练的原材料 |
