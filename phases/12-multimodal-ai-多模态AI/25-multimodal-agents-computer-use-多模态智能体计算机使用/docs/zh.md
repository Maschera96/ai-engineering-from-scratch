# 多模态智能体与计算机使用（综合实战）

> 2026 年的前沿产品是一个多模态智能体：它能读取截图、点击按钮、浏览网页 UI、填写表单，并端到端地完成工作流。SeeClick 和 CogAgent（2024）证明了 GUI 接地（grounding）这一基础能力。Ferret-UI 加入了移动端。ChartAgent 引入了面向图表的视觉工具使用。VisualWebArena 和 AgentVista（2026）是前沿模型所追逐的基准——即便是 Gemini 3 Pro 和 Claude Opus 4.7，在 AgentVista 的高难度任务上也只能拿到约 30%。这场综合实战把 Phase 12 的每一条线索都串了起来：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、接地（坐标输出）、长程记忆和评估。

**类型：** 综合实战
**语言：** Python（标准库，动作模式 + 智能体循环骨架）
**前置知识：** Phase 12 · 05（LLaVA）、Phase 12 · 09（Qwen-VL JSON）、Phase 14（智能体工程）
**时间：** 约 240 分钟

## 学习目标

- 设计一个多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建一个 GUI 接地输出模式（点击坐标、输入文本、滚动、拖拽），让 VLM 能以 JSON 形式输出。
- 比较纯截图智能体、可访问性树（accessibility-tree）智能体和混合智能体。
- 在一个小型 VisualWebArena 子集上搭建多模态智能体的基准评估。

## 问题

一个订票网站的工作流：“帮我找一张 4 月 15 日去东京、靠过道、800 美元以下的航班，并订下来。”

一个多模态智能体需要：

1. 对浏览器截图。
2. 把截图 + URL + 目标解析成一份计划。
3. 输出一个结构化动作：在 (x,y) 处点击、在元素 E 处输入“Tokyo”、向下滚动、选中（单选按钮）。
4. 把该动作应用到浏览器上。
5. 观察新状态（下一张截图）。
6. 重复，直到任务完成。

每一步都是一次多模态 VLM 调用。VLM 的输出必须是可解析的 JSON。错误会跨步骤累积，因此恢复能力很重要。

## 概念

### GUI 接地——基础能力

GUI 接地是指：给定一张截图和一条自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是首个规模化的开源成果：在合成 + 真实 GUI 数据上微调 VLM，把坐标作为纯文本 token 输出。可行。

CogAgent（arXiv:2312.08914）加入了 1120x1120 的高分辨率编码，以应对密集的 UI。得分：网页导航约 84%。

Ferret-UI（arXiv:2404.05719）专注于移动端 UI，与 iOS 可访问性数据集成。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc` 有助于恢复：如果坐标在截图之间发生漂移，这个语义提示能让系统重新接地。

### 动作模式

一个典型的动作模式有 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

智能体每一步输出一个动作。浏览器封装层执行该动作并返回新状态。

### 纯截图 vs 可访问性树

两种输入模式：

- 纯截图：完整图像，无结构信息。最通用；适用于任何应用。
- 可访问性树：结构化的 DOM / iOS 可访问性信息。在接地上可靠得多；在有该树可用的地方适用。
- 混合：两者兼用，把树作为原子动作的可靠接地器，把截图作为语义上下文。

生产环境的智能体在可能时使用混合模式。浏览器自动化（Selenium + 可访问性）总是有树可用；桌面应用则不一定。

### 长程记忆

一个 20 步的工作流会产生 20 张截图。VLM 的上下文会很快被填满。三种压缩策略：

- 摘要链（summary-chain）：每 5 步之后，总结发生了什么，丢弃旧截图。
- 跳帧（skip-frame）：保留第一张、最后一张以及每隔 3 张的截图。
- 工具记录日志：执行动作，保留一份记录所做操作的文本日志；不再回看旧截图。

Claude 的计算机使用 API 采用日志模式。更简单、更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）引入了用于图表理解的视觉工具使用：裁剪、缩放、OCR、调用外部检测。智能体可以输出“裁剪到区域 (100, 200, 300, 400) 然后调用 OCR”这样的工具调用。工具返回文本；VLM 继续推理。

这一模式可推广：set-of-mark 提示、区域标注和外部检测工具都符合同一个“输出一个工具调用、接收一个结构化响应”的模式。

### 2026 年的基准

- ScreenSpot-Pro。在约 1k 张网页截图上的 GUI 接地。开源 SOTA Qwen2.5-VL-72B 约 85%。前沿约 90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类信息）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）。2026 年最难的基准。涵盖 12 个领域的真实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- WebArena / WebShop。较老的基准；已被前沿模型刷满。

### 为什么它依然很难

智能体性能的瓶颈：

1. 细粒度视觉接地。“点击那个小 X”在移动端分辨率下经常失败。
2. 长程规划。在 10 个动作之后，智能体会偏离目标。
3. 错误恢复。当一次点击失败（点错按钮）时，检测 + 恢复很少出现在训练数据里。
4. 跨页面上下文。在标签页之间跳转或处理长表单时会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（用截图匹配来判断动作是否成功）。

### 综合实战的动手部分

综合实战任务：构建一个计算机使用智能体，它能：

1. 读取订票网站模拟页面的 HTML + 截图。
2. 规划一个多步序列：搜索 → 选择 → 填写表单 → 提交。
3. 输出符合动作模式的 JSON 动作。
4. 在一个固定的 10 个任务的子集上评估。

本课提供易于扩展为真实浏览器的脚手架代码。

## 用起来

`code/main.py` 是综合实战的脚手架：

- 动作模式 JSON 定义（10 个动作）。
- 以字典形式表示的模拟浏览器状态。
- 智能体循环骨架：接收状态、输出动作、应用、循环。
- 10 个任务的迷你基准（合成页面），用于衡量端到端成功率。
- 当某个动作失败时的错误恢复钩子。

## 交付它

本课产出 `outputs/skill-multimodal-agent-designer.md`。给定一个计算机使用产品（领域、动作集、评估目标），它会设计出完整的智能体循环、记忆策略、接地模式和预期的基准分数。

## 练习

1. 用一个 `screenshot_region` 工具（裁剪 + 缩放）扩展动作模式。哪些任务会从中受益？

2. 阅读 AgentVista（arXiv:2602.23166）。描述最难的任务类别，以及前沿模型为何仍然失败。

3. 长程记忆压缩：设计一个摘要链，活跃保留 ≤4 张截图，日志数量不限。

4. 构建一个错误恢复钩子：动作失败（找不到按钮）时，智能体下一步该做什么？

5. 在 10 个网页任务上比较纯截图的 Claude 4.7 与混合截图 + 可访问性树的 Qwen2.5-VL。哪一个在哪些任务上胜出？

## 关键术语

| 术语 | 人们怎么说 | 它实际指什么 |
|------|-----------------|------------------------|
| GUI 接地 | “点击坐标” | 模型在截图上为指令的目标输出 (x,y) |
| 动作模式 | “工具定义” | 对有效动作（点击、输入、滚动、拖拽）的 JSON 描述 |
| 可访问性树 | “结构化 DOM” | 来自浏览器/iOS API 的机器可读 UI 层级 |
| 混合智能体 | “截图 + 树” | 同时使用图像和结构信息；比单独任一种都更可靠 |
| 视觉工具使用 | “缩放/裁剪/检测” | 智能体在计划中途调用外部视觉工具（OCR、检测） |
| 摘要链 | “记忆压缩” | 用周期性文本摘要替换冗长的截图历史 |
| VisualWebArena | “端到端网页基准” | 2024 年的端到端网页任务基准 |
| AgentVista | “2026 年高难度基准” | 12 个领域的真实工作流；即便 Gemini 3 Pro 也只有约 30% |
