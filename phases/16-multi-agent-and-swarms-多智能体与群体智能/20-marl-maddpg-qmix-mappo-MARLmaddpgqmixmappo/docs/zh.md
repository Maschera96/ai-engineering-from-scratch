# MARL — MADDPG、QMIX、MAPPO
> 多智能体协调的强化学习传统，到 2026 年仍然影响着 LLM 智能体系统。**MADDPG**（Lowe 等人，NeurIPS 2017，arXiv：1706.02275）引入了集中训练、分散执行（CTDE）：每个批评者在训练期间都会看到所有智能体的状态和动作；在测试时只有本地演员运行。适用于合作、竞争和混合环境。 **QMIX** (Rashid et al., ICML 2018, arXiv:1803.11485) 是使用单调混合网络进行值分解；每个智能体的 Q 合并为联合 Q，因此 `argmax` 分布干净——在星际争霸多智能体挑战赛 (SMAC) 上占据主导地位。 **MAPPO**（Yu et al., NeurIPS 2022, arXiv:2103.01955）是具有中心化价值函数的 PPO；只需最少的调整即可在粒子世界、SMAC、Google Research Football、Hanabi 上“出奇地有效”。这些为必须分散行动的代理团队的培训政策奠定了基础。 MAPPO 是**默认的 2026 年合作 MARL 基线**。本课程从一个小型网格世界玩具构建每个想法，并在接触 LLM 代理训练之前将这三个想法植入肌肉记忆中。
**类型：** 学习
**语言：** Python（stdlib、小型无 NumPy 实现）
**先决条件：** 第 09 阶段（强化学习）、第 16 · 09 阶段（并行群网络）
**时间：** ~90 分钟
＃＃ 问题
LLM 代理系统越来越多地训练代理间协调策略：何时推迟、何时采取行动、呼叫哪个对等点。告诉您如何训练此类策略的文献是多智能体强化学习 (MARL)，它早于 LLM 浪潮，并拥有一小部分主导算法。
阅读没有模式词汇的 MARL 论文是痛苦的。集中式训练与分散式执行（CTDE）、价值分解和集中式批评并不是流行语——它们是针对特定问题的具体答案：
- 独立强化学习（每个智能体单独学习）从每个智能体的角度来看是非平稳的。坏的。
- 集中式强化学习（一个代理控制所有代理）无法扩展并且违反了执行约束。
- CTDE 充分利用了两者的优势：利用全球信息进行训练，利用本地政策进行部署。
＃＃ 概念
### 论文使用的三种环境
- **粒子世界（多代理粒子环境）。** 具有 cooperative/competitive 任务的简单 2D 物理。 MADDPG 的原始测试平台。
- **星际争霸多智能体挑战赛（SMAC）。** 合作微观管理，局部观察。 QMIX 的测试平台。离散动作，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO 基线。
不同的环境具有不同的 action/observation 类型。算法会相应地进行选择。
### MADDPG (2017) — CTDE 模式
每个代理 `i` 都有一个参与者 `mu_i(o_i)`，将其自己的观察映射到操作。每个智能体还有一个评论家 `Q_i(x, a_1, ..., a_n)`，可以查看训练期间的所有观察结果和所有操作。演员根据批评者的评估通过政策梯度进行更新。
```
actor update:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
critic update:   TD on Q_i(x, a_1..n) given next-state joint estimate
```

为什么选择CTDE：在训练时，我们知道每个人的行为；我们用它来减少每个批评者的方差。在部署时，每个代理仅看到 `o_i` 并调用 `mu_i(o_i)`。
失败模式：批评者随着N个代理的增长而增长（输入包括所有动作）。如果没有近似值，则无法扩展到超过约 10 个代理。
### QMIX (2018) — 价值分解
仅限合作。全局奖励是每个智能体 Q 值的单调函数之和：
```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证了 `argmax_a Q_tot` 可以由每个独立选择 `argmax_{a_i} Q_i` 的代理来计算。这**正是您需要的去中心化执行属性**。在训练时，混合网络从每个代理的 Q 中生成 `Q_tot`。
为什么 QMIX 在 SMAC 上获胜：合作星际争霸微观管理具有同质代理、本地 obs、全球奖励 - 完美适合价值分解。
失效模式：单调性约束是限制性的；有些任务的奖励结构不是单调可分解的（一名代理人为团队做出牺牲）。扩展（QTRAN、QPLEX）放松了这一点。
### MAPPO (2022) — 被忽视的默认值
多Agent PPO：具有集中价值功能的PPO。每个代理都有自己的政策；所有代理共享（或具有每个代理）查看完整状态的价值函数。于等人。 2022 年，我们在五个基准测试中对 MAPPO 与 MADDPG、QMIX 及其扩展进行了基准测试，结果发现：
- MAPPO 在粒子世界、SMAC、Google Research Football、Hanabi、MPE 上匹配或击败非策略 MARL 方法。
- 需要最少的超参数调整。
- 稳定的训练；可在种子间重现。
在本文发表之前，社区低估了按策略 MARL。 2026年，MAPPO是合作MARL的默认基线；任何新方法都必须击败它。
### 为什么LLM代理工程师应该关心
三种直接用途：
1. **路由器训练。** 元代理选择哪个子代理处理任务。这是一个 MARL 问题，具有 N 个分散的子代理和一个集中式路由器。 MAPPO 适合。
2. **角色出现。** 在生成智能体模拟中，训练智能体随着时间的推移采用互补角色是一个变相的 MARL 问题。 QMIX 式的价值分解通过构建强制互补。
3. **多代理工具的使用。** 当代理共享工具并竞争预算时，通过 CTDE 培训它们会产生尊重资源限制的可部署本地策略。
实际警告：到 2026 年，大多数生产 LLM 代理系统都会促进他们的政策，而不是培训他们。当您拥有 (a) 大量交互数据、(b) 明确的奖励信号以及 (c) 愿意投资培训基础设施时，MARL 就会出现。
### CTDE 作为超越 RL 的设计模式
即使没有经过培训，CTDE 也是一种有用的架构模式：
- 在*设计*期间，承担整个团队的可见性。
- 在*运行时*，强制分散执行：每个代理只能看到 `o_i`。
该模式迫使您保持每个代理的状态明确，并预先考虑部分可观察性。许多生产多智能体系统默默地假设各处共享状态——CTDE 规则阻止了这种情况的发生。
### 非平稳性问题
当多个智能体同时学习时，每个智能体的环境（包括其他智能体的策略）都是非平稳的。经典的单智能体强化学习证明被打破。本课中的 MARL 算法都解决了这个问题：
- MADDPG：全局评论家看到所有操作，因此其价值估计是固定的。
- QMIX：价值分解将学习转移到明确定义最优性的联合 Q 空间。
- MAPPO：中心化的价值函数抑制了其他人政策变化的差异。
在 LLM 代理系统中，非平稳性表现为“我的代理上个月工作了，现在其他上游代理发生了变化，我的行为不当”。使用 CTDE 训练 MARL 是原则性的解决方案；提示级修复速度更快，但持久性较差。
### 本课不涵盖的内容
训练实际网络是第 09 阶段的主题。本课程构建脚本化策略版本，演示 CTDE、值分解和集中值模式，无需梯度更新。目标是在获取完整的 MARL 库（PyMARL、MARLlib、RLlib 多代理）之前内化模式。
## 构建它
`code/main.py` 实现了三种模式演示，全部在一个微小的 2 代理协作网格世界上：
- 环境：4x4 网格上有 2 个智能体，一个奖励颗粒。如果任何代理到达颗粒，则奖励 = 1；任务完成。
- `IndependentAgents` — 每个代理将其他代理视为环境。基线。
- `MADDPGStyle` — 中心化批评家计算联合值；演员政策从中更新。脚本化的政策改进。
- `QMIXStyle` — 使用单调混合器进行值分解。
- `MAPPOStyle` — 中心化价值函数；根据共享基线更新策略。
所有四个都运行相同的剧集并报告平均目标步数。 CTDE 变体收敛到比独立基线更短的路径。
跑步：
```
python3 code/main.py
```

预期输出：独立代理平均采取~6步； CTDE 变体收敛到约 3.5 步（4x4 网格的最佳步数是 3）。尽管有脚本化的政策，但模式差异仍然存在。
## 使用它
`outputs/skill-marl-picker.md` 是一项为给定的多智能体任务选择 MARL 算法的技能：合作与竞争、同质与异质、动作空间类型、规模、奖励信号。
## 发货
MARL 的生产很少见。当你使用它时：
- **从 MAPPO 开始。** 2022 年的论文将此作为基线；首先复制它可以节省数周的寻找更奇特方法的时间。
- **记录每个代理的观察和操作流。** 在没有每个代理跟踪的情况下调试 MARL 是没有希望的。
- **将训练代码与执行代码分开。** CTDE 是一门学科；让执行路径真的只看到`o_i`。
- **奖励塑造警告。** MARL 对奖励设计极其敏感。塑造中的一个协调错误，代理学习如何利用它。运行对抗性测试。
- **对于LLM代理**，首先考虑提示级别的政策。仅当交互数据+奖励信号+基础设施全部存在时才投资 MARL 培训。
