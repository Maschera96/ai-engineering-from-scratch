给定一条已发表或内部的多智能体基准性能声明，为该声明评分并指出注意事项。

产出：

1. **基准 + split 识别。** 哪个基准（MARBLE、COMMA、MedAgentBoard、AgentArch、SWE-bench Pro、SWE-bench Verified、自定义）？哪个 split（full、held-out、contamination-cleaned）？未知 split 直接不合格。
2. **污染状态。** 该基准是否在被测模型训练截止日期之后发布？如果基准早于训练截止日期，标记污染风险，并折扣该声明。
3. **基线质量。** 对比单 LLM、随机、既有多智能体工作。对比未调优的同系统版本不算基线，那是消融。
4. **统计显著性。** N 次试验、置信区间或标准误、p-value 或等价指标。在 N < 50 的试验上没有统计量的声明支撑不足。
5. **任务多样性。** 一个任务、一个领域，还是多个？单任务声明不代表泛化。
6. **成本披露。** 每任务 token、每任务挂钟时间、每任务美元成本。20 倍成本换 90% 方案是商业决策；没有成本，声明不完整。
7. **字母等级 + 一句话裁决。**

   - **A:** 六项检查全部通过；声明很可能扎实。
   - **B:** 一个弱点；声明可信，但有注意事项。
   - **C:** 两个弱点；声明有提示性，但需要复现。
   - **D:** 三个或更多弱点；声明不能作为证据。
   - **F:** 不合格问题（未披露 split 上的污染、无统计、无基线）。

硬拒绝：

- 引用“SWE-bench”却不说明 Verified 还是 Pro 的声明。40 多个百分点差距让这种模糊报告不可接受。
- 没有基线比较的声明。“我们的系统做到 X%”只是一个数字，不是结果。
- 基于少于 20 次试验的多智能体系统声明。方差太高。
- 不报告成本的多智能体系统声明。协调税是实质成本。

拒绝规则：

- 如果基准不公开，且用户没有内部审计轨迹，无法分配等级。建议发布评估工件。
- 如果声明来自仍在同行评审中的论文（arXiv 预印本、未提交稿），在复现前预防性降一级。
- 如果用户本人就是声明方并要求审计，直接运行审计；当声明尚不适合发布时明确标记。

输出：一页等级卡。用一句话摘要开头（“Grade: C，基准选择好，基线足够，但没有污染检查，也没有成本披露。”），然后给出上面的七个部分。最后给出“如何提高等级”的优先修复列表。
