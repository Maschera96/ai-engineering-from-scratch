# 心理理论和紧急协调
> 李等人。 （arXiv：2310.10701）表明，LLM 智能体在合作文本游戏中表现出**新兴的高阶心智理论**（ToM） - 推理另一个智能体对第三个智能体的信念的看法 - 但由于上下文管理和幻觉而无法进行长期规划。 Riedl (arXiv:2510.05174) 测量了整个群体的高阶协同作用，发现**只有** ToM 提示条件才会产生身份相关的分化和目标导向的互补性；能力较低的法学硕士仅显示虚假出现。也就是说，协调出现是即时条件和模型依赖的，而不是自由的。本课程实现了一个最小的 ToM 感知代理，在有或没有 ToM 提示的情况下运行协作任务，并根据 Riedl 2025 协议测量协调增量。
**类型：** 学习 + 构建
**语言：** Python (stdlib)
**先决条件：** 第 16 · 07 阶段（心灵与辩论学会），第 16 · 17 阶段（生成代理）
**时间：** ~75 分钟
＃＃ 问题
多代理协调通常看起来很神奇：代理分工、相互预测、避免冗余。通常这种“出现”是快速工程的产物——有人告诉特工们要“协调”。去掉提示，去掉协调。
Riedl 的 2025 年发现更为严格：在受控条件下，只有当智能体被提示推理**其他智能体的想法** (ToM) 时，才会出现协调。如果没有 ToM 提示，即使是强大的模型也会显示出无法通过统计控制的协调模式。这对于生产来说很重要：团队提供的“多代理协调”功能依赖于提示且脆​​弱。
本课将 ToM 视为一种特定能力（对信念的信念进行推理），构建一个最小的 ToM 感知代理，并衡量真正的协调是什么样子与即时着装是什么样子。
＃＃ 概念
### ToM 是什么意思
发展心理学：一个三岁的孩子认为任何人的内心世界都与他们的相符。一个 5 岁的孩子明白其他人有不同的信仰。一个 7 岁的孩子对信念进行推理（“她认为我认为球在杯子下面”）。它们是零阶、一阶和二阶 ToM。
对于 LLM 代理，ToM 订单映射到：
- **零阶：**没有其他模型。代理仅根据自己的观察采取行动。
- **一阶：** 智能体拥有每个智能体信念的模型。 “爱丽丝相信X。”
- **二阶：** 代理建模递归信念。 “爱丽丝相信鲍勃相信X。”
李等人。 2023 发现一阶和二阶 ToM 出现在合作博弈中的 LLM 智能体中，但会因长视界和不可靠的通信而退化。
### Sally-Anne 测试，简述
1985 年的错误信念测试：莎莉将一颗弹珠放入篮子 A 中，然后离开。安妮将其移至篮子 B。莎莉回来后会看向哪里？具有一阶 ToM 的孩子说篮子 A（莎莉的信念与现实不同）。没有的孩子说篮子 B.
GPT-4 时代的法学硕士在坦率地提出时可以通过 Sally-Anne 式的测试。当叙述很长、场景多次变化或者问题措辞间接时，它们就会失败。这就是 2026 年 ToM 在生产法学硕士中的实际状态。
### Riedl 的协调测量
Riedl (arXiv:2510.05174) 构建了一个群体规模的测试：N 个代理、一个合作目标、可变的提示条件。措施：
1. **与身份相关的差异。** 随着时间的推移，代理是否会形成稳定的角色区别？
2. **目标导向的互补性。** 智能体的行为是否相互补充（不同的子任务）而不是重复？
3. **高阶协同作用。** 衡量该群体是否实现了子集无法实现的目标的统计指标。
结果：只有在 ToM 提示条件下，所有三个指标才会产生高于基线的信号。如果没有 ToM 提示，中等容量模型的指标就接近机会。大型模型在没有显式 ToM 提示的情况下表现出一定的协调性，但效果小于显式提示的效果。
### 协调错觉
如果没有统计控制，演示中的“紧急协调”通常会反映出：
- 提示工程协调一致（系统提示说“一起工作”）。
- 观察者偏见（我们看到我们期望的模式）。
- 成功运行的事后选择。
在没有可测量信号的情况下推销“紧急协调”的生产系统应被视为营销。索赔前先测量。
### 最小的 ToM 感知代理
结构：
```
agent state:
  own_beliefs:    {facts the agent believes}
  other_models:   {other_agent_id -> {beliefs_the_agent_attributes_to_them}}
  actions_last_N: [history of others' actions]

observation update:
  - update own_beliefs from direct observation
  - update other_models[agent_id] from their action + prior beliefs

action selection:
  - enumerate candidate actions
  - for each, predict what each other agent will do next given their modeled beliefs
  - pick action that maximizes joint outcome under those predictions
```

`other_models` 属性是 ToM 状态。一阶 ToM 仅保留一个级别。二阶添加了 `other_models[i][other_models_of_j]` — 我认为代理 i 认为代理 j 相信的内容。
### 为什么长期看伤人
李等人。文档：上下文限制导致代理忘记哪个信念属于谁。幻觉给其他智能体模型添加了错误的信念。两者都会产生“我以为他认为 X”的错误，并随着时间的推移而复合。
该文件和 2024-2026 年后续行动中记录的缓解措施：
- **提示中显式的 ToM 状态。** 结构化格式：`{agent_id: belief_list}`。强制检索以保留身份信念绑定。
- **推理链较短。** 每回合的 ToM 更新较少，可减少复合幻觉。
- **外部 ToM 存储。** 在 LLM 上下文之外维护模型；每转仅注入相关零件。
### ToM 在生产中失败的地方
- **对抗性设置。** 具有良好 ToM 的代理更容易操纵（你可以模仿他们对你的模仿，然后利用）。
- **异构团队。** 当模型不同时，适用于一个对手的 ToM 模型无法泛化。
- **依赖于事实真相的任务。** ToM 是关于信念；如果正确性取决于事实，ToM 可能会分散注意力。
### 您可以实际测量的协调性
团队的协调是真实的而不是匆忙着装的三个实际信号：
1. **随着时间的推移具有互补性。** 在多回合任务中，智能体的操作是否涵盖不相交的子任务？
2. **预期。** 特工 A 在 T+1 回合的行动是否取决于对 B 在 T+2 回合的行动的预测（结果是正确的）？
3. **纠正。** 当 A 在 T 轮误读了 B 的信念时，A 会在 T+2 轮纠正吗？
这些可以在记录的多代理系统中进行测量。它们是“协调”叙事的实质版本。
## 构建它
`code/main.py` 实现：
- `ToMAgent` — 跟踪自己的信念和每个其他代理的信念模型。
- 合作任务：三个代理必须从三个盒子中收集三个代币；每个盒子可以容纳一个令牌。代理无法通信；他们从彼此的行为中推断出意图。
- 两种配置：`zeroth_order`（无 ToM）和 `first_order`（具有一级信念模型的 ToM）。
- 超过 200 项随机试验的测量：完成率、重复率（两种药物针对同一个盒子）、平均完成次数。
跑步：
```
python3 code/main.py
```

预期输出：零阶智能体以约 35% 的速度重复工作，并在 10 轮内完成约 60% 的试验。一阶 ToM 代理的复制率约为 5%，完成度约为 95%。 Delta 是可测量的协调效应。

`outputs/skill-tom-auditor.md` 是一种审核多代理系统声称的“紧急协调”的技能。检查是否及时着装、对照对照的统计显着性以及测量的互补性。
## 发货
协调索赔清单：
- **控制条件。** 没有协调提示的系统版本。测量两者。
- **统计测试。** 在您的指标中，`p < 0.05` 系统和控制之间的差异是否显着？
- **互补性衡量标准。** 随着时间的推移，行动脱节，而不仅仅是最终的成功。
- **失败案例日志。** 当代理不协调时，ToM 状态是什么样的？
