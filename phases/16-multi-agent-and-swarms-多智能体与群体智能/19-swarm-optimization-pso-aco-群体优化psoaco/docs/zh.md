# LLM 的群优化（PSO、ACO）
> 仿生优化法正在卷土重来。 **LMPSO** (arXiv:2504.09247) 使用 PSO，其中每个粒子的速度是一个提示，LLM 生成下一个候选粒子；适用于结构化序列输出（数学表达式、程序）。 **Model 群体s** (arXiv:2410.11163) 将每个 LLM 专家视为模型权重流形上的 PSO 粒子，并报告在 9 个数据集上仅包含 200 个实例的 12 个基线上的 **13.3% 平均增益**。 **群体Prompt** (ICAART 2025) 混合 PSO + Gray Wolf 以进行提示优化。 **AMRO-S** (arXiv:2603.12933) 是受 ACO 启发的信息素专家，用于多代理 LLM 路由 — **4.7 倍加速**、可解释的路由证据、将推理与学习分离的质量门控异步更新。本课程在提示参数空间上实现 PSO，在代理路由上实现 ACO，衡量这些经典算法为何适合 LLM 时代，何时不适合。
**类型：** 学习 + 构建
**语言：** Python (stdlib)
**先决条件：** 第 16 · 09 阶段（并行群网络）、第 16 · 14 阶段（共识和 BFT）
**时间：** ~75 分钟
＃＃ 问题
您的任务评估得分为 62%。你想改进它。最幼稚的做法是无梯度手动调整，但扩展性很差。强化学习需要奖励信号和足够的部署来进行训练。通过提示进行反向传播实际上是不可能的——提示是一个离散的字符串，而不是一个可微分的参数。
经典的仿生优化——用于连续搜索空间的 PSO，用于路径选择的 ACO——正是为这种机制而设计的：无梯度、基于群体、每次评估成本低廉。将它们与法学硕士配对以进行无梯度搜索步骤，您将获得一个令人惊讶的实用优化器。
相同的模式适用于多代理系统中的代理*路由*。 ACO 风格的信息素踪迹记录了哪个代理在哪种任务类型上表现最好，让路由器利用该踪迹，并衰减信息素，以便可以重新发现路由。
＃＃ 概念
### PSO 复习（Kennedy & Eberhart 1995）
粒子群优化：连续搜索空间中的粒子群。每个粒子都有位置 `x_i` 和速度 `v_i`。每次迭代：
```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中 `p_best` 是粒子自身的最佳值，`g_best` 是群体的最佳值，`w, c1, c2` 是惯性+认知+社会权重，`r1, r2` 是随机因素。
LLM 输出上的 ### PSO — LMPSO
arXiv:2504.09247 针对 LLM 生成的结构化输出（数学表达式、程序）调整 PSO。每个粒子都是一个候选输出。 Velocity 是一个*提示*，描述如何将当前输出修改为 personal/global 最佳值。 LLM 根据速度提示生成新的输出。速度的“惯性”是一种类似于“进行微小的增量改变”的提示。
这在以下情况下效果很好：
- 输出是结构化的（可解析、可评估）。
- 健身是自动的（测试运行、算术评估）。
- 人数较少（约 10-30 个粒子），因此 LLM 呼叫总数保持在可控范围内。
当健身需要人工审查时，它就不能很好地工作——每次迭代的成本变得令人望而却步。
### 模型群
arXiv:2410.11163 将 PSO 从输出层取出并放入 *model* 层。每个“粒子”都是一位专家LLM（参数）。群通过无梯度更新将参数移向集体最佳值。据报告：9 个数据集的 12 个基线的平均增益为 13.3%，每次迭代仅 200 个实例。
关键的见解是 LLM 专家模型已经在共享参数流形（适配器权重、LoRA 增量）中附近。在这个低维子空间上的 PSO 既便宜又有效。
### ACO 复习 (Dorigo 1992)
蚁群优化：蚂蚁遍历一张图；每条路径都有一条信息素踪迹。蚂蚁移动的概率按信息素强度加权。完成任务的蚂蚁会沉积与溶液质量成正比的信息素。信息素会随着时间的推移而衰减。
### AMRO-S — ACO 用于代理路由
arXiv:2603.12933 使用 ACO 进行多代理路由。每个任务类型都是一个“目的地”；每个代理都是一条可能的路线。信息素加强了产生良好产出的路线。主要贡献：
- **可解释的路由证据。** 信息素强度是人类可读的信号。
- **质量门控异步更新。** 信息素仅在质量检查通过后更新，从而将推理与学习分离。
- 在多代理路由基准上 **4.7 倍加速**。
质量门很重要：没有它，快速但错误的代理会积累信息素，并且系统会锁定错误的路线。
### 法学硕士何时使用 PSO / ACO
**在以下情况下使用 PSO：**
- 搜索空间是连续的或映射到连续参数（提示嵌入、LoRA 权重、数字生成参数）。
- 健身既便宜又自动化。
- 人口可能很少（10-30）。
**在以下情况下使用 ACO：**
- 您有路由或路径选择问题。
- 决策随着时间的推移而强化（相同的任务类型再次出现）。
- 您需要可解释的证据来做出路由决策。
**在以下情况下请勿使用：**
- 健身需要人工审查（每次迭代成本太高）。
- 搜索空间是离散和组合的，PSO 没有覆盖（使用遗传算法代替）。
- 实时决策需要严格的延迟（PSO/ACO 相对于单遍启发式收敛缓慢）。
### 为什么仿生学仍然获胜
基于梯度的方法需要可微信号。 LLM 的输出和路由决策不是微分的。伪梯度方法（强化学习路由器、DPO 式提示调谐器）有效，但需要昂贵的培训。
PSO 和 ACO 仅需要一个 *evaluator* 函数。如果您可以对候选输出或路由决策进行评分，则可以对空间进行优化。这使得适用性的门槛大大降低。
### 实际限制
- **群体预算。** N 个粒子 × T 次迭代 × 每次评估成本。对于每次调用约 0.02 美元的 LLM 评估，运行 50 次迭代的 20 粒子 PSO 的成本约为 20 美元。相应地计划。
- **探索与利用。**信息素衰减率和 PSO 惯性权衡；衰减太快→忘记解决方案；太慢→陷入早期局部最优。
- **灾难性漂移。** 如果适应度景观发生变化（新数据分布），两种算法都会收敛然后发散。监控最佳健身稳定性。
## 构建它
`code/main.py` 实现：
- `LMPSO` — PSO 超过数字提示参数（温度、top_k 权重）。每个粒子的“LLM 生成”都被模拟为脚本化的适应度函数。运行算法 30 次迭代并显示 g_best 收敛。
- `AMRO_S` — ACO 式路由。 3 个代理，4 种任务类型，信息素矩阵，100 个路由任务。打印（任务类型→代理选择）随时间的分布以显示踪迹形成。
- 比较：同一任务流上的随机路由与 ACO 路由。测量质量和延迟。
跑步：
```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适应度在 30 次迭代中从随机改进到接近最优。
- AMRO-S：信息素表稳定在每个任务类型的正确代理上； ACO 路由在质量上优于随机路由约 30-40%，并且还减少了延迟（重试次数更少）。
## 使用它
`outputs/skill-群体-optimizer.md` 有助于在 PSO、ACO、遗传算法和基于梯度的优化器之间进行选择，以解决 LLM/代理优化问题。
## 发货
- **从小处开始。** 10-20 个粒子，20-50 次迭代。仅当收敛曲线显示出明显增益时才扩大规模。
- **每次迭代记录信息素或 g_best。** 在没有线索的情况下调试群优化器是痛苦的。
- **质量门更新。** 特别是对于 ACO 路由：快速且错误的代理不得累积信息素。
- **重置分布变化的衰减。** 当您的评估分布发生变化时，老化的信息素就会失效；暂时重置或加倍衰减率。
- **限制每次迭代成本。** 发出每次迭代成本指标。 PSO 的成本为 500 美元/迭代，收益为 0.5%，不可发货。
## 练习
1.运行`code/main.py`。观察 LMPSO 收敛。改变群体大小 5、10、20、50。收敛时间在多大时达到饱和？
2. 实施“灾难性漂移”实验：迭代 30 次后，更改适应度函数。 PSO 的适应速度有多快？重置 `p_best` 有帮助吗？
3. 向 AMRO-S 添加质量门：仅在评估分数 > 0.7 的运行中沉积信息素。与非门控版本相比，这如何改变收敛性？
4. 阅读 LMPSO (arXiv:2504.09247)。将纸张的“速度作为提示”映射回您的数字速度。模拟中丢失了什么以及保留了什么？
5. 阅读 AMRO-S (arXiv:2603.12933)。通过异步信息素更新实现解耦的“推理快速路径”。这如何改变持续负载下的系统延迟？
