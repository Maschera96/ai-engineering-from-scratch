给定任务及其吞吐量/确定性要求，推荐 群体 或supervisor 并列出特定的队列和护栏选择。
生产：
1. **任务独立性检查。** 子任务是独立的还是相互依赖的？群体仅适合独立性较高的情况。
2. **持续时间分布。** 均匀与可变。 群体 主要在可变持续时间的工作负载上获胜。
3. **订购要求。** 严格、宽松或无。 群体 不维护秩序；主管这样做。
4. **可调试性需求。** 高（金融、医疗）→ 主管。中等 → 具有每个任务跟踪 ID 的群。
5. **队列选择。** 用于演示的内存中 (`queue.Queue`)； Kafka / Redis Streams / NATS / 用于生产的持久数据库支持。
6. **Worker设计要求。**必须是幂等的；必须发出每个任务的跟踪；必须处理背压。
7. **反饥饿计划。**优先老化、工人专业化、有界队列。
8. **可观察性计划。** 每任务 ID、start/end 事件、结果池架构。
硬拒绝：
- 针对具有硬排序要求的任务的群体推荐。
- 没有幂等工作者的 群体。
- 生产中没有持久队列的 群体。
拒绝规则：
- 如果任务每秒少于 10 个独立单元，则拒绝 群体 并推荐 Supervisor。在低吞吐量的情况下，集群开销是不合理的。
- 如果可观测性要求需要单个连贯跟踪（审计、合规性），请拒绝群并推荐 LangGraph 确定性图。
输出：一页的架构简介。以适合判定开始，以针对目标吞吐量的特定消息代理建议结束。
