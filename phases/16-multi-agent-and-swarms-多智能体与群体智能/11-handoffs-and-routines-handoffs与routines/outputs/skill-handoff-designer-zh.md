给定面向用户的任务（通常是分类或基于技能的路由），生成准备映射到 OpenAI 群体 或 OpenAI Agents SDK 的切换拓扑。
生产：
1. **代理名册。** 每个代理：姓名、一句话目的、工具以及可以移交给哪些其他代理。
2. **切换功能。** 每个代理的工具签名。每个切换函数返回一个目标代理。
3. **上下文传输策略。** 在每个切换边缘：完整历史记录、最后 N 条消息或汇总快照。证明合法。
4. **Guardrails.** 每个代理的输入验证（允许哪些提示触发向敏感专家的移交），在需要时进行移交验证。
5. **循环检测。** 检测乒乓球的规则（例如，“A 切换到 B；B 切换回 A”连续发生多次）。
6. **回退行为。** 如果缺少切换目标（已删除代理、身份验证失败），则由哪个代理处理会话。
7. **会话/内存计划。** 是否使用 Agents SDK 会话、调用者管理的内存或根本不使用内存。
硬拒绝：
- 任何没有环路检测的切换设计。
- 将完整历史记录传递给具有不同工具权限的专家的移交功能（安全风险）。
- 假设 群体 的无状态行为但需要多轮内存的设计 — 请改用 Agents SDK 会话。
拒绝规则：
- 如果任务需要并行执行，请拒绝群体并推荐supervisor（第05课）。
- 如果任务需要确定性 audit/replay，则拒绝并推荐 LangGraph 静态图。
- 如果任务是一个简单的 DAG 阶段（研究 → 代码 → 审查），建议改为 CrewAI Sequential。
输出：一页的交接摘要。最后附上安全说明，说明即时注入如何触发不必要的切换以及哪些护栏会阻止它。
