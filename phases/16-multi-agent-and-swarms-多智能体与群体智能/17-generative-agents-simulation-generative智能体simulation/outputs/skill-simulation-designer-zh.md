给定一个需要代理群体紧急行为的场景（社会模拟、游戏 NPC、政策预演、市场动态），设计模拟。
生产：
1. **种群规模和异质性。** N 因子；共享基本模型与不同模型；及时家属；角色分配。 《超人前传》使用了 25 名具有个性化角色的同类代理人；更多的人群受益于异质性。
2. **内存架构。** 每个条目的字段：`(ts, kind, content, importance, embedding_ref, source_ids)`。新近度衰减常数；重要性评分程序；相关性度量（带有嵌入模型 X 的余弦）。压缩的保留策略。
3. **反射节奏。** 触发：未处理的重要性总和 > 阈值，或每 N 个观察值，或周期性刻度。每个触发器的反射次数。反思提示模板。
4. **计划范围。** 天/小时/行动水平。哪些是强制性的；哪个是可选的。修订触发器：与活动计划相矛盾的重要性 > 阈值的新观察。
5. **世界模型。** 空间网格、社交图、资源约束。什么构成了观察（视线、对话、通知）。哪些规范性约束是架构无法学习且必须明确编码的（容量限制、关闭时间、私人空间）。
6. **种子目标。** 哪些代理具有哪些优先级。可能存在竞争的重叠目标；应该共存的非竞争性目标。
7. **预算。** 每个代理的每次 LLM 调用（观察 + 检索 + 反映 + 计划 + 行动）。每个代理每次报价的预期代币。 T 个刻度的总模拟成本。
8. **评估指标。** 可信度（人工评估者）、目标实现率、协调事件计数、空间规范违规作为失败信号。
硬拒绝：
- 没有明确的空间/社会规范编码的设计。该建筑将违反它们（2023 年公园的封闭商店、单浴室失败）。
- 具有可变内存的设计。内存必须是仅追加的；更正是新条目。
- 每一次运行反射的设计。这是预算效率低下的；反射的成本很高，触发器应该基于阈值。
- 在没有内存压缩策略的情况下进行大 N (> 50) 的模拟。检索成本随着流长度的增加而增加。
拒绝规则：
- 如果场景需要紧急*任务执行*而不是紧急*社交行为*，则推荐主管/角色/原语模式（阶段 16 · 05-08）。 《超人前传》是一款社交模拟游戏。
- 如果预算允许每个时间点总共 < 100 个 LLM 调用，建议 N = 3-5 进行密集交互而不是较大群体。
- 如果场景不能从涌现（紧密脚本化的任务）中受益，建议使用单代理+工具。
输出：一页设计简介。从单句摘要开始（“超人前传式模拟：15 个异构代理，重要性总和 > 120 的反射，3 级计划范围，具有容量约束的空间网格，通过可信度 + 协调事件来衡量。”），然后是上面的八个部分。以预期的紧急行为和需要注意的前三种故障模式结束。
