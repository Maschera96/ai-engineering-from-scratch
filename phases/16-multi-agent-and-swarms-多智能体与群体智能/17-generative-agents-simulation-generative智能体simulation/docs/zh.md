# 生成代理和紧急模拟
> 帕克等人。 2023 年（UIST '23，arXiv：2304.03442）填充 **Smallville**，一个由 25 个代理组成的沙箱，具有三部分架构：**内存流**（自然语言日志）、**反射**（代理生成有关其自己的流的更高级别综合）和 **计划**（日级行为，然后是子计划）。具有里程碑意义的结果是情人节派对的出现：一名特工在没有进一步编写脚本的情况下，植入了“想要举办情人节派对”的种子，制作了在人群中传播的邀请函，协调了日期，然后派对就举办了——来自 24 名在开始时对此一无所知的特工。消融表明所有三个组成部分都是可信度所必需的。记录的失败是空间规范错误（进入关闭的商店、共用单人浴室）。这是 2026 年智能体模拟和多智能体社会评估的参考架构。
**类型：** 学习 + 构建
**语言：** Python (stdlib)
**先决条件：** 第 16·04 阶段（原始模型）、第 16·13 阶段（共享内存）
**时间：** ~75 分钟
＃＃ 问题
大多数多代理系统都是脚本严密的团队：规划者计划、编码器代码、审阅者评论。这适用于明确定义的任务。它无法捕捉当智能体拥有记忆、优先级和开放世界时出现的突发的、无脚本的行为。研究、社会模拟和日益增长的游戏人工智能都需要第二种。
《超人前传》的架构是它的基准。在 Park 2023 之前，最好的代理模拟是浅层脚本追随者；之后，该模式成为开放世界中生成代理的默认模式。如果你在 2026 年构建一个代理模拟，你要么使用《超人前传》的三个组件，要么明确说明为什么不使用。
＃＃ 概念
### 三个组成部分
**内存流。** 观察、行动、反思和计划的仅附加日志。每个条目都有时间戳、类型、描述（自然语言）和派生元数据：**新近度**、**重要性**（代理自评 1-10）和 **相关性**（与当前查询的余弦相似度）。
```
[2026-02-14 09:12:03] observation: Isabella Rodriguez asked me if I like jazz
[2026-02-14 09:14:22] reflection:   I enjoy long conversations about music
[2026-02-14 10:05:00] plan:         Attend Isabella's Valentine's Day party tonight
```

内存检索结合了三个分数：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。 Top-k 条目进入当前提示符。
**反思。** 代理会定期（每 N 个记忆或重要事件）从最近的记忆中生成高阶合成。反射条目返回到流中并且可以像任何其他内存一样检索。这就是代理建立“理解”的方式——该架构相当于长期信念。
**计划。** 自上而下的分解。首先，制定一个大致的日常计划（“去上班，和克劳斯一起吃晚饭”）。然后是小时级计划。然后是行动层面的计划。计划是可修改的：当观察结果与计划相矛盾时，代理会重新计划受影响的部分。
### 为什么这三者都很重要（消融）
帕克等人。进行消融，放弃每一项观察、反思和计划。每次消融都会损害可信度：
- 如果没有**观察**，代理就会错过背景并按照陈旧的信念行事。
- 如果没有**反思**，主体就无法形成更高阶的信念；互动保持浅层。
- 没有**计划**，行为就会变成反应性噪音；目标消散。
人类评分者的可信度得分在这三者中最高；删除任何一个都会产生可测量的回归。
### 情人节的出现
一位经纪人伊莎贝拉·罗德里格斯 (Isabella Rodriguez) 的目标是“2 月 14 日下午 5 点在霍布斯咖啡馆举办情人节派对”。其他 24 名特工没有收到此类种子。模拟天数：
1.伊莎贝拉的计划包括邀请人。
2. 每个邀请都会成为邻居记忆流中的一个观察结果。
3. 邻居的反映产生了信念：“伊莎贝拉正在举办派对。”
4. 邻居的计划包含“2 月 14 日参加聚会”。
5. 邻居告诉其他邻居。邀请在没有中央协调的情况下传播。
6. 2月14日下午5点，几名特工聚集在Hobbs Cafe。
这是技术意义上的出现：系统级行为（一方）源于本地交互（双边邀请+个人规划），没有中央协调者。
### 记录的故障模式
帕克等人。明确记录：
- **空间范数错误。** 特工走进关门的商店。特工们尝试使用同一个单人浴室。特工在不适合吃饭的房间里吃饭。该模型并不仅仅从环境中推断出社会物理规范。
- **内存溢出。**深度模拟运行会导致内存检索成本增加。实用的补救措施：定期内存压缩（总结和修剪）和低重要性条目的衰减。
- **反射幻觉。** 反射可以发明内存流中不存在的关系。缓解措施：在反射提示中包含源内存 ID，并在检索时进行验证。
这些是与生产相关的故障模式：任何 2026 年代理模拟都会继承它们。
### 三部分实施细则
1. **内存只能追加。** 切勿改变内存条目。更正是新条目。
2. **重要性分数很便宜。** 在撰写时致电法学硕士将重要性评级为 1-10。缓存分数。
3. **检索是排名的，而不是过滤的。** 按综合得分Top-k；不要使用硬过滤器（这会丢失上下文）。
4. **反射定期运行。** 当未处理记忆的重要性总和超过阈值（例如 150）时触发。
5. **计划是可以修改的。** 当新的观察结果与计划相矛盾时，仅重新生成受影响的部分，而不是整个计划。
### 超越超人前传的生成代理
2024-2026年后续文献扩展了该架构：
- **Multi-agent social simulation for policy / market research.** Smallville-like populations simulate user behavior in response to features.比 A/B 测试更快；准确性有争议。
- **NPC AI for games.** RPGs with Smallville agents produce emergent storylines instead of scripted quests.
- **生成代理评估基准。** 该指标不再是任务准确性，而是长期运行的可信度+行为的连贯性。
架构是参考。扩展交换组件（记忆向量存储、检索增强反射、神经符号计划），但保留三部分结构。
### 为什么这对于多代理工程很重要
《超人前传》证明了当组件正确时，多智能体的出现成本低廉。该架构现已在开源模型上复制（较小的法学硕士会优雅地而不是急剧地失去可信度）。任何需要**紧急社会行为**的生产系统都使用这种形状。任何需要**严格任务执行**的系统都使用此阶段早期的主管/角色/原语模式。
## 构建它
`code/main.py` 使用脚本化代理策略（没有真正的 LLM）实现 stdlib Python 中的三个组件。该演示再现了情人节派对的缩影：
- `MemoryStream` — 具有 recency/importance/relevance 检索的仅附加日志。
- `reflect(stream)` — 对最近的高度重要记忆的脚本化反思。
- `plan(agent_state)` — 基于当前信念的日级和小时级计划。
- 场景：5 名特工。特工 1 以“下午 5 点举办派对”开始。在模拟的蜱虫中，邀请传播并且代理聚集。
跑步：
```
python3 code/main.py
```

预期输出：逐笔追踪。到最后一次，5 名特工中至少有 3 人在他们的计划中显示了聚会，并且他们在聚会地点集合。单一种子在没有任何协调者的情况下产生了协调到达。
## 使用它
`outputs/skill-simulation-designer.md` 设计了一个生成代理模拟：代理数量、内存模式、反射节奏、计划范围和评估指标。
## 发货
生产模拟规则：
- **内存是数据库。** 大规模选择一个真实的存储（向量 DB、Postgres）。内存中的 stdlib 用于原型。
- **记录检索跟踪。** 对于每个操作，记录驱动该操作的前 k 个记忆。这就是你的调试能力。
- **每个代理代币的预算。** 每个代理的检索 + 反映 + 计划每个周期是 O(k) LLM 调用。 N 个座席 × T 个刻度 × 每个刻度的呼叫次数可能会让您的预算相形见绌。
- **定期压缩内存。** 总结并修剪低重要性条目。保留策略是一个设计决策，而不是一个细节。
- **明确检测空间/社会规范违规行为**。架构不学习它们。
## 练习
1.运行`code/main.py`。确认 3 名以上特工聚集在聚会上。将特工增加到 10 名——这种现象还会发生吗？
2.去掉反射步骤。行为是什么样的？映射到 Park 2023 的消融发现。
3. 引入一个竞争性种子目标（“Klaus 想在下午 5 点进行研究演讲”）。特工们会分裂，还是只有一个目标占主导地位？是什么决定了它？
4. 添加空间限制：Hobbs Cafe 最多可容纳 4 个特工。模拟手柄是否优雅地溢出，还是撞上了“单人浴室”的失败模式？
5. 阅读 Park 等人的著作。 (arXiv:2304.03442) 第 6 节（突发行为实验）。确定一种在你的缩影中无法重现的行为。您需要增强架构的哪些组件？
