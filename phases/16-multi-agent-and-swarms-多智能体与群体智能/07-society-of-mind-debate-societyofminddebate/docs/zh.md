# 心灵社会和多智能体辩论
> 明斯基 1986 年的前提——智能是一个专家社会——每十年都会被重新发现。 2023 年，杜等人。将其转化为具体的算法：多个LLM实例提出答案，阅读彼此的答案，批评和更新。经过 N 轮，他们达成了共识，击败了零样本 CoT 以及对六项推理和事实性任务的反思。有两个发现很重要：**多个代理**和**多轮**都独立做出贡献。社会击败了单人独白；多轮交换胜过一次性投票。
**类型：** 学习 + 构建
**语言：** Python (stdlib)
**先决条件：** 第 16 阶段·04（原始模型）
**时间：** ~60 分钟
＃＃ 问题
自我一致性——对一个模型进行多次采样并采用大多数答案——是你可以采用的最便宜的推理改进方法。它有效，但很快就会饱和。您可以将样本加倍，而不会看到另一个有意义的跳跃。
辩论打破了饱和状态。 N 个智能体不是来自一个模型的 N 个独立样本，而是互相阅读彼此的推理并进行修改。样本之间的相关性下降（它们不再是独立同分布），并且收敛点通常是正确的（独立同分布）。投票肯定是错误的。
＃＃ 概念
### 杜等人。 2023算法
来自 arXiv：2305.14325（ICML 2024）：
1. N 个代理中的每一个都对问题给出一个初始答案。
2. 对于第 r = 2..R 轮：向每个代理显示其他代理的第 r-1 轮答案，并询问“考虑这些，给出您更新的答案”。
3. R 轮后，以多数票选出最终答案。
该论文对 MMLU、GSM8K、传记、数学和事实性基准进行了测试。辩论始终胜过 CoT 和自我反思。
### 两个独立的旋钮
来自同一篇论文的消融：
- **单独的代理计数**（1 轮，N 的多数票）在大多数任务上击败了单一代理，但处于稳定状态。
- **单独的轮数**（1个代理看到自己先前的推理）几乎没有帮助 - 反射的已知弱点。
- **两者一起**产生大跳跃。多个智能体之间的多轮交换驱动增益。
### 为什么它有效
两种机制：
1. **暴露于分歧。** 当一个智能体看到另一个智能体的推理链有不同的结论时，它必须证明或更新。不管怎样，r+1 轮的上下文比 r 轮更丰富。
2. **相关误差减少。** 在自洽中，所有样本都来自同一模型，因此误差是相关的——你可以平均得出一个自信的错误答案。不同的模型或不同的种子去相关。不同的“有争议的观点”进一步去相关。
### 异类辩论
A-HMAD 和相关后续产品对不同的代理使用“不同的基础模型”。 Llama + Claude + GPT 辩论减少了单一栽培的崩溃（第 26 课），因为一个模型家族的相关错误不会被其他模型家族共享。
缺点：参与辩论的弱模型可能会将共识拖向错误的答案（参见“我们应该发疯吗？”，arXiv：2311.17371）。
### NLSOM — 129 个代理扩展
诸葛等人。 （“Mindstorms in Natural Language-Based Societies of Mind”，arXiv:2305.17066）将这个想法扩展到了 129 个成员的社会。结果是：专业化和自组织随着规模的扩大而出现，系统在视觉问答等任务上的表现优于单智能体。
### 故障模式
- **阿谀奉承级联。** 所有特工都会听从听起来最有信心的特工。争论最终以最大声的声音瓦解。提示扮演对抗性角色（“一名代理人必须提出相反的立场”）会有所帮助。
- **主题漂移。** 经过多轮辩论，偏离了最初的问题。缓解措施：每轮重新注入问题。
- **计算爆炸。** N 个代理 × R 轮 = N·R LLM 调用，每个调用都有一个不断增长的上下文。 5 名代理人、5 轮辩论是在不断增长的背景下进行 25 次呼吁。每个问题的成本可能超过单个 CoT 调用的 10 倍。
## 构建它
`code/main.py` 对一个数学问题进行 3 主体 × 3 轮辩论，其中每个主体以不同的（可能是错误的）答案开始。代理是有脚本的——每个“更新”都是通过对邻居的答案进行平均，并根据脚本置信度进行加权。在逐轮日志中可以看到收敛情况。
该演示展示了两个关键效果：
- 单轮交换使代理更接近正确答案。
- 第 2 轮之后的额外轮次显示收益递减（与 Du 等人的稳定期相匹配）。
跑步：
```
python3 code/main.py
```

## 使用它
`outputs/skill-debate-configurator.md` 为一项新任务配置辩论：代理数量、轮数、异质性（相同模型与混合模型）、角色分配（对称与单对抗）。它还会在您运行之前估算令牌成本。
## 发货
如果您进行辩论：
- **上限为 3 轮。** Du 等人。显示 3 轮获得了大部分收益。更多的是成本，而不是质量。
- **代理上限为 5。** 超过 5，上下文膨胀和成本占主导地位。
- **默认情况下是异构的。** 池中至少有两个不同的基本模型。
- **对抗性时段。** 一名特工提示不同意。打破阿谀奉承。
- **记录每一轮。**隐藏中间轮的辩论系统无法调试或审核。
## 练习
1. 运行 `code/main.py`，然后将轮数设置为 5 并观察收益递减。额外收敛在哪一轮停止？
2. 添加第四个具有对抗性角色的代理人：始终不同意当前的多数意见。这会破坏还是改善收敛？
3. 绘制（打印）每轮的一致性分数（多数答案中代理的分数）。什么时候达到1.0，这相当于“正确”吗？
4. 阅读杜等人的著作。第 4 节消融。使用此代码复制“仅代理”与“仅轮次”与“两者”结果。
5. 阅读“我们应该发疯吗？” (arXiv:2311.17371) 并列出了循环赛之外的两种辩论变体——例如，法官主导、链式辩论、对抗性。
