给定一个多代理框架（或使用该框架的代码库），生成四基元映射，以便读者可以在一个段落中理解该框架。
生产：
1. **Agent定义。** Agent是如何构建的？什么参数？它携带什么状态？命名确切的类或工厂。
2. **切换机制。** 它使用三种切换模式中的哪一种 - 函数返回、图形边缘或扬声器选择？如果是混合体，哪个是主要的？显示触发一次切换的最少代码。
3. **共享状态模型。** 完整消息池还是投影视图？内存中还是持久（检查点）？对于并发编写者来说它是线程安全的吗？谁来调解矛盾？
4. **协调器类型。** 静态、LLM 选择、切换驱动还是队列驱动？如果选LLM，默认什么型号？如果是静态的，图是循环图还是 DAG？
5. **跨轴权衡。** 各一句话：确定性、可扩展性上限、可调试性、典型故障模式。
硬拒绝：
- 任何声称抽象是“新的”而不显示它的映射都不会折叠为四个基元之一。如果你无法减少它，请准确地命名间隙，而不是发明第五个基元。
- 仅引用营销文档的框架比较。始终引用框架存储库或官方食谱中的具体代码示例。
- 诸如“框架 X 更适合代理”之类的陈述，但未指定框架优化哪个原语。
拒绝规则：
- 如果框架是闭源的并且公共文档不公开代理切换状态协调器表面，请声明没有内部结构就不可能进行映射。
- 如果用户提供代码库但没有框架（手动代理），则映射自定义实现并标记哪个原语设计不足。
- 如果框架早于 2024 年（原始 AutoGen v0.2，群体 之前）并且不再维护，请包含一行注释，说明其后继者是否保留映射。
输出：一页的框架简介。从一句话总结开始（“框架 X 将切换修复为图形边缘并通过化简器公开共享状态。”），然后是上面的五个部分，然后是结束段落，命名该框架的原语最适合哪个生产项目。
