# 代理的共识和拜占庭容错
> 经典分布式系统 BFT 满足随机 LLM。 2025-2026 年出现了三个研究方向： **CP-WBFT** (arXiv:2511.10400) 通过置信度探测来衡量每张选票； **DecentLLMs** (arXiv:2507.14928) 通过并行工作提案和几何中值聚合实现无领导； **WBFT** (arXiv:2505.05103) 将加权投票与层次结构聚类相结合，以分割核心节点和边缘节点。 “AI 代理能同意吗？”的诚实实证结果(arXiv:2603.01213) 的一个问题是，即使是标量协议在今天也是脆弱的——单个欺骗性代理可以损害代理混合。 BFT 是必要的，但还不够。本课程构建了一个最小的 BFT 协议，注入三种特定于代理的攻击（拜占庭谎言、阿谀奉承、相关错误单一文化），并测量每个共识变体如何应对。
**类型：** 学习 + 构建
**语言：** Python (stdlib)
**先决条件：** 第 16 · 07 阶段（心灵与辩论学会）、第 16 · 13 阶段（共享内存）
**时间：** ~75 分钟
＃＃ 问题
您有 N 个 LLM 代理，每个代理都给出一个答案。他们不同意。多数票选择了错误的一个，因为两个代理是相关的（相同的基础模型、相同的训练数据、相同的故障模式）。第三个代理人碰巧以一种新颖的方式犯了错误——所以多数是虚假多数。
现在添加一个欺骗性代理人：它是故意说谎的。或者是一个阿谀奉承的代理人：它同意最后发言的人。在经典 BFT 中，假设拜占庭节点是 `f < n/3` 的一部分并且行为任意。 2026 年的现实是，即使是诚实的，LLM 节点也是随机的，跨模型相关，并且受到彼此输出的影响。你不能将他们视为独立的伯努利选民。
经典的 BFT（PBFT，1999）并没有错——它是不完整的。它处理任意位翻转。它不处理“三个诚实的代理因为共享训练数据而共享幻觉”。本课程以 PBFT 的基础为基础，并以 2025-2026 年的三项调整为基础。
＃＃ 概念
### 经典的 BFT 为您提供什么
实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 拜占庭节点。该协议具有三个阶段（预准备、准备、提交）和两个原语（签名消息、仲裁证书）。 `n >= 3f + 1` 诚实或恶意节点之间就单个值达成一致。
这些保证是强有力的，但假设：
1. **独立的错误。** 拜占庭不协调。
2. **诚实的节点是真正诚实的。** 诚实输出的正确性不是问题；该协议仅协调分歧。
3. **这个问题有一个真实的答案。** 对错误事实的共识仍然是共识。
LLM 代理人违反了这三项规定。运行相同基本模型的两个代理共享故障。一个“诚实的”法学硕士仍然会产生幻觉。对于模棱两可的问题，“真相”是由特工决定的——没有外部的预言。
### 三种 LLM 特定攻击
**拜占庭谎言。** 一名特工输出了故意错误的答案。如果 `f < n/3`，经典的 BFT 会处理这个问题。
**阿谀奉承。** 一名特工在投票前阅读其他人的答案，并与最后发言的人保持一致。并非恶意，但与最大的声音相关。经典 BFT 不会阻止这种情况，因为代理通过了每个签名检查。
**相关错误单一培养。** 三个代理共享一个基本模型。他们产生了同样的错误答案的幻觉。大多数人都是错误的。经典的 BFT 没有帮助，因为所有三个“诚实地”都同意。
### 2025-2026 年的回应
**CP-WBFT** (arXiv:2511.10400) — 置信度探测加权 BFT。每个投票者都将置信度探针附加到其答案（自我报告的概率，或单独的校准模型的预测）。充满信心地投票权重。据报告，完整图表的 BFT 改进为 +85.71%。缓解措施：阿谀奉承（顺从者往往对其自愿立场缺乏信心）。
**体面的法学硕士** (arXiv:2507.14928) — 无领导者。工作代理并行提出建议，评估代理对建议进行评分，最终答案是评分位置的几何中位数。当 `f < n/2` 时具有鲁棒性。缓解措施：拜占庭谎言和相关错误（几何中位数对异常值具有鲁棒性，并且会拉向密集簇，而不是模型偏差平均值）。
**WBFT** (arXiv:2505.05103) — 具有层次结构聚类的加权 BFT。投票权重是根据响应质量加上从历史记录中学到的信任分数来分配的。将代理集群分为核心和边缘；核心代理必须首先达成共识，边缘代理紧随其后。缓解措施：可扩展性（核心共识小而快）和部分单一文化（可以选择核心以实现多样性）。
### 经验：“人工智能代理能同意吗？” （arXiv：2603.01213）
该论文衡量了多个前沿模型的标量一致性（LLM 代理就单个数值达成一致）。这个发现令人不舒服：
- 即使没有对手，LLM 代理人在许多基准上对标量问题的分歧率也超过 30%。
- 采用欺骗性角色的单个代理可以使混合代理共识偏离诚实基线 40 个以上百分点。
- 分歧率与模型多样性相关——异质集合比同质集合的分歧更多（好：不相关的错误），但漂移也更慢（坏：达成一致的时间更长）。
结论：BFT 为您提供了对齐输出的机制，但它不会告诉您对齐的输出是否正确。结合验证（第 16·08 阶段角色专业化）、多样性（第 16·15 阶段辩论变体）和评估代理（第 16·24 阶段基准）。
### 核心协议，精简
LLM 代理的最小 BFT 轮：
```
1. task arrives; each agent i produces answer a_i
2. each agent attaches confidence probe c_i in [0, 1]
3. aggregator collects (a_i, c_i) from all n agents
4. aggregator groups by semantic cluster (equivalent answers)
5. aggregator computes weight for each cluster C:
     w(C) = sum_{i in C} c_i
6. winner = cluster with max weight, if max > threshold * sum(c_i)
   else: retry or escalate
7. minority clusters logged with provenance for post-hoc audit
```

语义聚类步骤是法学硕士特有的扭曲。两个答案“研究报告 4.2%”和“4.2% 改善”是同一簇。幼稚的字符串相等性检查会错过这一点。在生产中，使用廉价的嵌入模型或显式规范化。
### 阈值调整
`threshold` 参数决定何时接受以及何时重试。太低：你接受微弱的多数。太高了：你永远不会接受任何东西。经验范围：`n=5-7` 代理为 0.5-0.67，较小的 `n` 则更高。低于阈值，升级到人类或不同的代理团队。
### 共识无济于事的地方
- **模棱两可的问题。** 如果问题没有基本事实，共识就是一种意见。就这么称呼吧。
- **复合问题。**“编写代码并解释它”——两个答案。对每个人独立投票。
- **对抗性多轮**。如果智能体可以观察前几轮并模仿（Du 2023 辩论），他们就会开始彼此达成一致，无论真相如何。限制回合（通常为 2-3 轮）。
## 构建它
`code/main.py` 实现：
- `AgentVoter` — 具有（答案，信心）的脚本化策略。
- `MajorityVote` — 经典复数。
- `CPWBFT` — 具有语义聚类的置信加权投票。
- `DecentLLMs` — 评分提案的几何中值聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。
实施的攻击模式：
1. `byzantine`：一名代理人的谎言具有很高的可信度。
2. `sycophancy`：一个代理复制它看到的第一个答案，并具有匹配的置信度。
3. `monoculture`：三个代理共享一个具有中等置信度的错误答案（相关错误）。
跑步：
```
python3 code/main.py
```

预期输出：（攻击，聚合器）->最终答案表，突出显示正确答案。多元化无法解决单一文化的问题。 CPWBFT 的置信权重减少了阿谀奉承。当单一文化少于人口的一半时，DecentLLM 的几何中位数会拉向诚实集群。
## 使用它
`outputs/skill-共识-designer.md` 为多智能体集成设计了一个共识协议：聚类方法、权重、阈值和亚阈值回合的升级策略。
## 发货
在发布任何共识机制之前：
- **至少使用上述三种模式**进行攻击测试。你的协议应该可以预见地失败，而不是默默地失败。
- **记录每个少数群体**及其来源。少数集群是相关错误的预警系统。
- **执行有界回合。** 没有“继续辩论直到达成协议”——这会奖励阿谀奉承。
- **将协议与正确性分开。** 共识输出发送给验证者；验证者独立于整体。
- **监控同意率。** 急剧上升意味着符合偏差；急剧下降意味着模型漂移。
## 练习
1.运行`code/main.py`。确认多元化无法应对单一文化攻击，但当单一文化置信度低于 0.7 时，CPWBFT 可以部分缓解该攻击。
2. 添加第四种攻击模式：**沉默弃权** - 一名特工拒绝回答（“我不知道”）。每个聚合器应该如何对待弃权？实施您的选择。
3. 将语义聚类从字符串规范化转换为嵌入相似性（使用任何开源嵌入模型）。谄媚攻击会发生什么？
4. 阅读 CP-WBFT (arXiv:2511.10400)。实施置信度探测校准步骤（单独的校准模型检查每个代理的自我报告置信度）。测量单一栽培场景的准确性增益。
5. 阅读《AI 代理能同意吗？》 （arXiv：2603.01213）。重现一个简化的标量协议实验：三个代理、一个标量问题、欺骗性角色提示。 CPWBFT 或 DecentLLM 能抓住这一点吗？
## 关键术语
|术语 |人们怎么说|它实际上意味着什么 ||------|----------------|------------------------|
| BFT | “拜占庭容错”| Castro-Liskov 1999 协议，用于与 `f < n/3` 任意故障达成共识。 |
|拜占庭| “任何不良行为” |一个节点可以撒谎、丢弃消息、默默地失败——除了安全崩溃之外什么都可以。 |
|置信度探针| “你有多大把握？” |投票所附的自我报告或校准器预测概率。 |
|语义聚类| “相同的答案，不同的措辞” |在计票之前对等效答案进行分组。 |
|几何中位数| “稳健的中心” |最小化到样本点的距离总和的点。与均值不同，对异常值具有鲁棒性。 |
|单一栽培 | “同样的型号，同样的失败”|代理共享训练数据或基础模型时的相关错误。 |
|阿谀奉承| “同意大声”|代理的投票偏向于使用 first/loudest. 的人 |
| Core/Edge | “分层BFT”| WBFT 分裂：小核心共识优先，边缘节点跟随。限制延迟。 |
