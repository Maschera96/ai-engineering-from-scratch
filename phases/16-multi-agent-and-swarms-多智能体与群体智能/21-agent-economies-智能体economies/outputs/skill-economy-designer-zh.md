给定需要激励调整的多代理场景（开放网络、异构运营商、代币化奖励或基于声誉的路由），设计经济层。
生产：
1. **身份层。*​​ 用于便携式身份的 W3C DID，或平台内部 ID（如果系统是封闭的）。通过网络的开放性来证明其合理性。
2. **信用归属。** 均等分割、最后贡献者通吃、贡献加权、Shapley（精确或抽样）或无（按调用付费）。当联盟很重要时，建议使用 Shapley 抽样；简单的按次付费的平等分配。
3. **支付机制。** 任务分配的第二价格拍卖（单调聚合下真实），第一价格为了速度，公布价格为了简单。如果收益取决于质量验证，则进行托管。
4. **声誉规则。** 指数衰减常数、削减政策、最低下限、最高上限。信誉读取成本较低（路由为 O(1)）并在验证后写入。
5. **验证。** 谁验证贡献质量？单独的代理、人工审核、链上预言机、跨代理证明？如果没有验证，信用归属就是猜测。
6. **女巫缓解。** 什么可以阻止一名操作员启动 N 个假代理？声誉的锻造成本、人性证明、股权要求或每个 DID 的声誉上限。
7. **法律和司法管辖区检查。** 代币计价的支付涉及大多数司法管辖区的金融监管。如果适用，请标记并建议进行法律审查。
硬拒绝：
- 任何未经贡献质量验证的设计。最快但最错误的代理人将获得荣誉。
- 声誉不衰。陈旧的声誉会奖励那些几年前表现出色但现在已经崩溃的代理商。
- N > 6 时的 Shapley 精确计算。计算时间随着 N 的增加而增长！；样品代替。
- 第二价格拍卖，其中聚合函数不是单调的。真实性是站不住脚的。
- 无需监管检查的代币分配。许多司法管辖区将其视为证券活动。
拒绝规则：
- 如果系统完全是内部的（一家公司，一名运营商），建议更简单的分配（经理分配，指标是内部的）。经济机制是矫枉过正的。
- 如果无法验证贡献质量，建议在经济设计之前添加验证。没有它，经济就只是装饰品。
- 如果用户想要一个代币化系统但没有法律团队，请标记风险并建议从声誉（非代币）开始。
输出：一份两页的简介。从一句话总结开始（“具有 DID 的纯信誉系统、3 代理管道上的 Shapley 采样信用、槽分配的第二价格拍卖、验证失败的削减。”），然后是上面的七个部分。以为期 30 天的试点计划结束：预热阶段、验证管道设置、声誉加权推出、审核计划。
