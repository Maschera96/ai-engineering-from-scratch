# 代理经济、代币激励、声誉
> 长视野自主代理（METR 的 1 小时到 8 小时工作曲线）需要经济代理。新兴的**5层堆栈**是：**DePIN**（物理计算）→**身份**（W3C DID + 声誉资本）→**认知**（RAG + MCP）→**结算**（账户抽象）→**治理**（代理 DAO）。生产代理激励网络包括 **Bittensor**（TAO 子网奖励特定于任务的模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于变压器的 PoW，将计算重新分配给高效的 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用**Shapley 值信用归因**来公平奖励贡献代理；谷歌研究“大型语言模型的机制设计”提出了单调聚合下二次价格支付的**代币拍卖**。本课程构建了一个最小的代理市场，将 Shapley 价值信用归因应用于多代理管道，并运行第二价格代币拍卖，以便博弈论机制具体落地。
**类型：** 学习
**语言：** Python (stdlib)
**先决条件：** 第 16 · 16 阶段（谈判和讨价还价）、第 16 · 09 阶段（并行群网络）
**时间：** ~75 分钟
＃＃ 问题
当代理共同创造价值但需要单独奖励时，多代理系统就会变得复杂。经典机制——平等分配、最后贡献者通吃——是不公平的或可博弈的。通过 Shapley 值进行的基于联盟的奖励在构造上是公平的，但计算成本昂贵。 2025-2026 年的文献提出了有用的近似方法：沙普利采样、单调聚合拍卖以及从已确认的贡献中获得的链上声誉。
除了信用归属之外，该领域还转向了实际的经济主体：Bittensor TAO 奖励挖矿计算以微调特定于子网的模型，Fetch.ai/ASI 使用 FET 代币奖励 ASI-1 Mini LLM 使用，Gonka 将变压器工作量证明重新分配给高效的 AI 任务。自主交易的代理如今已经存在；问题是如何调整激励措施。
本课将代理经济视为一个特定的问题系列——信用归因、机制设计和声誉——并用最少的数学知识构建每个问题，以便使想法得以坚持。
＃＃ 概念
### 5 层代理经济堆栈
1. **DePIN（物理计算）。** 租用 GPU、存储、带宽的去中心化基础设施。 Bittensor 子网、渲染网络、Akash。不针对特定代理；代理使用它。
2. **身份。** W3C 去中心化标识符 (DID) 为每个代理提供独立于任何平台的持久 ID。 DID 会获得声誉。代理网络协议 (ANP) 使用 DID 作为发现层。
3. **认知。** 代理的推理循环：LLM + RAG + MCP。这就是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）允许代理从自己的余额中支付 Gas，而无需持有 ETH。代理可以相互支付服务费用或计算费用。
5. **治理。** 代理 DAO：人类*和*代理对协议变更进行投票的治理结构，投票权与声誉挂钩。
并非每个生产系统都使用这五种。 Bittensor 使用 1、2、部分 3、部分 4、5 中的任何一个。OpenAI 代理不使用除 3 之外的任何一个。堆栈是参考映射，而不是要求。
### Bittensor、Fetch.ai、Gonka — 运行什么
**Bittensor (TAO)。** 子网是专门的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对它们进行排名；权益加权评分分配 TAO 奖励。每个子网都有自己的评估。经济教训：为特定任务的输出质量付费，而不是为使用的计算付费。
**Fetch.ai / ASI 联盟。** ASI-1 Mini LLM 在 Fetch.ai 的网络上运行；用户支付 FET 代币进行推理。这里，代理对等的叙述更加有力：Fetch 上的代理可以呼叫另一个代理执行任务并用 FET 付款。
**Gonka.** Transformer 工作量证明：“工作”是 Transformer 的前向传递。矿工通过运行已知正确输出（来自训练数据）的推理任务来赚钱。资源生产性 PoW，而不是基于哈希的 PoW。
截至 2026 年 4 月，这三者均为生产级。收益分配不同。 Bittensor 奖励相对于子网验证器的质量；获取付费用户衡量的奖励效用； Gonka 奖励可验证的推理工作。
### Shapley 值信用归因
三个代理协作完成一项任务。输出得分 0.8。谁贡献了什么？
Shapley 值：满足四个公理（效率、对称性、线性、零）的唯一信用分配。对于代理 `i`：
```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 时 `i` 之前的代理集。实践中：枚举所有排列，记录每个排列中每个代理的边际贡献，平均值。
对于 N=3 个代理，有 6 种排列。对于 N=10，3.6M — 因此在实践中您会抽样排序而不是枚举。
### 聚合的第二价格拍卖
谷歌研究（“大型语言模型的机制设计”）提出了用于聚合 LLM 输出的第二价格代币拍卖。设置：N个代理每个提出一个完成；每个都有被选择的私人价值。拍卖师选择价值最高的提案并支付“第二高”的价值。在单调聚合下（价值取决于选择的提案，而不是出价的数量），这是真实的——代理出价了他们的真实价值。
为什么这对法学硕士系统很重要：您可以将完成任务外包给多个具有不同定价的代理；拍卖会挑选最好的+公平支付，代理商没有动机误报。
### 声誉资本
与 DID 绑定的声誉分数是根据已确认的贡献累积的。一个简单的更新规则：
```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。 声誉：
- 路由决策的阅读成本很低（“将困难任务发送给高代表代理”）。
- 锻造成本高昂（随着时间的推移积累，与 DID 绑定）。
- 可以削减：验证失败的贡献会被扣除。
### AAMAS 2025 去中心化 LaMAS
LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 值信用归属和简单的拍卖机制。关键主张：分散信用归因步骤使系统可审计并且不受单点操纵的影响。
### 经济崩溃的地方
- **价格预言机操纵。** 如果信用功能可以被操纵，代理商就会对其进行操纵。每个机制都需要对抗性测试。
- **女巫攻击。** 一名操作员启动 N 个假代理来夸大自己的贡献。 DID 会减慢速度，但并不能阻止这种情况；建立声誉的成本是缓解措施。
- **验证成本。** 信用归属仅与验证者一样公平。如果验证成本低（小型LLM），则可以进行游戏；如果昂贵（人工小组），系统就无法扩展。
- **监管悬而未决。** 代理经济体与金融监管相互交叉。截至 2026 年，Bittensor、Fetch 和 Gonka 都在某些司法管辖区的法律灰色地带运营。
### 代理经济何时有意义
- **具有异构运营商的开放网络。** 没有一个团队控制所有代理。
- **可验证的输出。** 如果没有验证，信用归属只是猜测。
- **长期工作流程。** 一次性任务不会从声誉积累中受益。
- **代币化支付在您所在的司法管辖区是合法可行的**。
在封闭的公司系统中，经济让位于更简单的分配（经理分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 发货
2026年运行代理经济：
- **从声誉开始，而不是代币。** 声誉的实施成本低廉，但单独存在就有价值；代币增加了法律和经济的复杂性。
- **在奖励之前进行验证。** 切勿在没有独立验证步骤的情况下分配积分。自我报告的质量会增加女巫游戏。
- **Shapley 样本，而非精确 Shapley。** 样本 100-1000 个订单；精确枚举不具有规模性。
- **上限衰减因子和底层声誉。** 无限制的衰减会抹去合法的贡献者；衰减太慢会奖励陈旧的高代表代理。
- **对抗性审核机制。** 在开放网络之前运行红队场景。每个机制都有一个博弈论；你想要找到漏洞，而不是攻击者。

1.运行`code/main.py`。确认 Shapley 值总和等于总值（效率公理）。改变价值函数； Shapley 分配是否按预期方向变化？
2. 实施 Shapley *采样*（基于 K 排序的蒙特卡罗）。 K 如何影响近似精度？与 N=4 的精确值进行比较。
3. 拍卖前实行结盟步骤：代理商可以合并成团队，作为一个单位竞标。形成哪些联盟？帕累托结果是否比单独投标更好？
4. 阅读 Google Research 机制设计帖子。确定一个假设，如果违反该假设，就会破坏真实性。 LLM 设置中的失败模式是什么样的？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在一项综合任务中实施超过 10 个智能体的 Shapley 步骤。精确计算需要多长时间？ 100 次抽检的采样结果有多接近？
