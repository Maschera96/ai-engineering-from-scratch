# 谈判和讨价还价
> 代理协商资源、价格、任务分配和条款。 2026 年的基准设定很明确：NegotiationArena (arXiv:2402.05863) 显示法学硕士可以通过角色操纵（“绝望”）将收益提高约 20%； “衡量讨价还价能力”（arXiv：2402.15813）显示买家比卖家更难，而且规模也无济于事——他们的 **OG-Narrator**（确定性报价生成器 + LLM 解说员）将交易率从 26.67% 推高至 88.88%；大规模自主谈判竞赛（arXiv：2503.06416）进行了约 18 万次谈判，发现**隐藏思想链**代理通过向对手隐藏推理而获胜；巴塔查里亚等人。 2025 年，哈佛谈判项目指标将 Llama-3 列为最有效，Claude-3 激进，GPT-4 最公平。本课程实现 Contract Net 协议（FIPA 祖先，第 02 课），连接 LLM 风格的 buyer/seller，运行 OG-Narrator 风格的分解，并测量交易率如何随每个结构选择而变化。
**类型：** 学习 + 构建
**语言：** Python (stdlib)
**先决条件：** 第 16 阶段·02（FIPA-ACL 遗产）、第 16 阶段·09（并行群网络）
**时间：** ~75 分钟
＃＃ 问题
两个代理商需要就价格达成一致。在纯语言提示下，2024-2026 年法学硕士以惊人的低利率完成交易（arXiv:2402.15813 中严格参数化交易的约 27%）。规模并不能解决这个问题：GPT-4 在结构上并不比 GPT-3.5 更擅长讨价还价；它更擅长讨价还价的“语言”。
根本问题是法学硕士将两项工作混为一谈——决定录取通知书和描述录取通知书。 OG-Narrator 将这些分开：确定性报价生成器计算数字移动； LLM仅叙述。交易率跃升至约 89%。
这反映了一个经典的多代理发现：将机制与通信层解耦会获胜。合约网络协议（FIPA，1996；Smith，1980）是参考任务市场机制。将法学硕士插入旁白槽，您就会获得一个由法学硕士驱动的现代任务市场。
＃＃ 概念
### Contract Net，一段话
史密斯 1980 年合同网络协议：**经理**广播**提案征集 (cfp)**； **投标人**以包含其报价的**提议**消息进行响应；经理选出获胜者并向获胜者发送**接受提案**，向失败者发送**拒绝提案**。获胜者完成工作。可选消息：**拒绝**（投标人拒绝提议）。 FIPA 将其编码为 `fipa-contract-net` 交互协议。
### 为什么 OG-Narrator 获胜
“测量语言模型的讨价还价能力”（arXiv：2402.15813）观察到：
- 法学硕士经常违反讨价还价规则（以荒谬的价格报价，忽略对方的 ZOPA）。
- 他们的锚定能力较差（接受糟糕的首次报价；以象征性而非战略性的金额还价）。
- 仅靠规模并不能解决这些问题。更大的模型会产生更合理的语言，但也会出现类似的策略错误。
OG-Narrator 分解：
```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ offer generator  │ price → │  LLM narrator    │ → message
           │  (deterministic) │        │  (writes the     │
           │                  │        │   human-style    │
           └──────────────────┘        │   accompaniment) │
                                       └──────────────────┘
```

报价生成器是一种经典的谈判策略：鲁宾斯坦讨价还价模型、Zeuthen 策略或简单的针锋相对的价格策略。法学硕士叙述。该消息包含确定性价格和自然语言框架。
交易率上升的原因是：
- 价格保持在议价区间。
- 锚是战略性的，而不是情感性的。
- 法学硕士做它擅长的事情：写作。
### NegotiationArena 调查结果
arXiv:2402.05863 提供了规范基准。标题调查结果：
- 法学硕士可以通过采用角色来提高收益约 20%（“我非常想在周五之前卖掉这个”）——角色操纵是一种真正的策略。
- Fair/cooperative 代理被敌方利用；防御需要明确的反姿势。
- 在大约 40% 的基准情景中，对称配对会导致不公平的结果。
这并不是“法学硕士是糟糕的谈判者”。这是“法学硕士的谈判太像人类了，包括可利用的部分。”
### 思想链隐藏
大规模自主谈判竞赛 (arXiv:2503.06416) 在许多 LLM 策略中进行了约 18 万次谈判。获胜者向同行隐瞒了他们的理由：
- 如果代理人将“我只会出 75 美元；我的保留价是 70 美元”打印到公开可见的便签本上，对手就会读取它。
- 获胜者私下计算策略；输出通道仅包含报价和最低要求的旁白。
这是 2026 年经典博弈论的回响（Aumann 1976 年关于理性和信息）：揭示你的私人估值成本回报。法学硕士并没有凭直觉意识到这一点，他们很乐意在推理痕迹中输入他们的保留意见，这些痕迹对对方来说是可见的。
工程要点：将私有便签本上下文与公共消息上下文分开。不是可选的。
### 巴塔查亚等人。 2025 — 模特排名
关于哈佛谈判项目指标（有原则的谈判、BATNA 尊重、利益互惠）：
- **Llama-3** 在讨价还价方面最有效（交易率+回报）。
- **Claude-3** 是最具侵略性的谈判者（高锚，迟到的让步）。
- **GPT-4** 是最公平的（配对之间的回报差异最小）。
这是 2025 年的快照。关键不在于哪个模型会在 2026 年 4 月获胜，而在于不同的基础模型具有持久的谈判风格。异质集成（第 15 课）将此作为多样性来源。
### 通过 Contract Net + LLM 进行任务分配
LLM 多代理的 Contract Net 的现代重用：
1. 管理代理将任务分解为多个单元。
2. 向工作代理广播带有任务描述的 `cfp`。
3. 每个工作人员返回一个报价：`(price, eta, confidence)`，其中价格可以是代币、计算单元或美元。
4. 经理选出获奖者（单个或多个，具体取决于任务）和奖项。
5.被拒绝的工人可以自由投标其他任务。
其规模远远超过 100 名工作人员，因为协调是广播和响应，而不是同步聊天。在生产中使用：Microsoft Agent Framework 的编排模式、一些 LangGraph 实现。
### LLM-利益相关者互动谈判
NeurIPS 2024 (https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf) 引入了具有**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者都有私人公用事业；法学硕士必须从消息中推断出它们。这是两党讨价还价到N党联盟形成的概括。与具有异构工人能力的生产任务市场相关。
### 叙述与机制规则
在所有 2024-2026 年谈判基准中，一致的工程规则是：
> 让法学硕士来讲述。不要让法学硕士计算offer。
如果报价需要是一个数字（价格、预计到达时间、数量），请根据协商状态确定性地生成报价，并让法学硕士生成框架。如果要约需要是提案结构（任务分解、角色分配），请让法学硕士起草它，但在发送之前根据模式和约束检查对其进行验证。
## 构建它
`code/main.py` 实现：
- `ContractNetManager`、`ContractNetTask`、`Bid` — 经理 + 投标人、广播 cfp、收集提案、奖励。
- `og_narrator_bargain(state, rng)` — OG-Narrator 买家：确定性的 Zeuthen 式向中点让步。
- `seller_response(state, rng)` — 确定性卖家还价政策（两种风格的结构性基本事实）。
- `naive_llm_bargain(state, rng)` — 模拟全 LLM 讨价还价者：选择差异较大的价格，通常在 ZOPA 之外。
- 衡量：超过 1000 次试验的成交率，每次试验都采样新鲜的预订价格。
跑步：
```
python3 code/main.py
```

预期产出：naive-LLM 成交率~65-75%； OG-Narrator 成交率~85-95%； 15-25分的差距是从叙述中分解报价生成的结构优势。加上一个包含三个投标者和一项任务的合同网任务市场分配示例。

`outputs/skill-bargainer-designer.md` 设计了一个讨价还价协议：谁生成报价（确定性或 LLM）、谁叙述、私人暂存器如何与公共消息分开以及如何监控交易率。
## 发货
生产议价清单：
- **单独的暂存器。** 私有状态永远不会到达对方的上下文。这是没有商量余地的。
- **确定性报价生成。** 价格、数量、预计到达时间：计算，不提示。
- **根据模式验证所有传入的报价**。在协议边界拒绝 ZOPA 之外的要约。
- **绑定回合。** 最多 3-5 回合；陷入僵局时升级为调解员。
- **连续衡量交易率和支付差异**。交易率下降是一种症状——通常是一种迅速的漂移或对手方的攻击。
- **记录所有被拒绝的提案**以及确定性的理由。对于合同网管理者来说，失败的投标人需要了解原因。
## 练习
1.运行`code/main.py`。确认 OG-Narrator 在交易率上击败了 naive-LLM。多少？
2. 实施**基于角色的支付改进** (arXiv:2402.05863) - 买家仅在旁白中采用“本周迫切需要购买”的角色，报价生成器不变。交易率或回报会改变吗？
3. 实现思想链**隐藏**：维护一个不传递给对方的私有便笺字符串。如果你不小心泄漏了会发生什么（通过交换通道进行模拟）？
4. 将Contract Net扩展到N个投标人的底价拍卖。当出价全部超过保留价时，经理如何在最低价格和最高质量之间做出决定？您选择哪种奖励规则？为什么？
5. 阅读 Bhattacharya 等人的著作。 2025 年哈佛谈判项目指标。实施两个不同风格的讨价还价者（激进与公平）。测量对称和非对称配对下的收益方差。
